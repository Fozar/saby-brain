# Звонок 2026-06-30: Мусохранов — Тимошенко (сдача задачи корешков)

Дата: 2026-06-30 12:17  
Участники: [[Мусохранов-Андрей-Владиславович]] (тимлид), Тимошенко Александр  
Тема: Демонстрация и сдача задачи по корешкам для существующих сделок  
Длительность: ~5 минут

---

## Контекст

Тимошенко демонстрирует Мусохранову реализацию задачи по корешкам. Задача имеет два компонента (читаются из ВНТ):
1. **Отправление корешков при создании обычных реферальных сделок** — реализовано (см. [[zvonok-musohranov-timoshenko-2026-06-24]], [[ReferralProgram-Stub-Implementation]])
2. **Корешки для существующих сделок** — миграция существующих сделок на корешки; реализована, но проверка не выполнена и **не может быть проведена сейчас**

---

## Вопрос производительности

Мусохранов спрашивает, зачем выбирать все сделки («все миллион»). Тимошенко поясняет: итерация идёт **по программам**, для каждой программы выбираются её сделки — тот же подход, что в реестре сделок. Мусохранов принял объяснение.

---

## Текущее состояние задачи

| Компонент | Статус |
|---|---|
| Создание корешка при новой сделке | Реализовано, тесты зелёные |
| Корешки для существующих сделок (миграция) | Реализовано, проверка **не выполнена** |
| GUI-часть | **Не реализована** — только консольные вызовы |
| Боевой запуск миграции | Зависит от версии **400** (согласовано с Пиреновым) |

**Ключевое ограничение:** боевая миграция корешков для существующих сделок возможна только начиная с версии 400. Это ограничение уже согласовано с Пиреновым.

---

## Формат сдачи (решение Мусохранова)

Мусохранов: *«Сдавай как есть и декларируй планы по этой задаче»*.

Требования к сдаче:
1. Написать **обзор по каждому пункту** задачи (что сделано, что нет)
2. Объявить **планы**:
   - Когда задача выйдет на бой
   - Когда после отпуска будет доделана
   - Когда GUI-часть заработает (пользователи смогут увидеть результат)
3. От Мусохранова требуется только: **посмотреть схему конвертации**
