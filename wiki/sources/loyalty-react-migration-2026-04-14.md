# Source: Loyalty React Migration — Project Docs

3 документа: цель проекта, ТЗ (полное), план работ.

## Ключевые факты

- **Проект:** «[Технический] Перевод страниц лояльности на React»
- **Цель:** ускорить построение страниц лояльности, создать запас производительности
- **Дедлайн:** 27.02.2026; тестирование 01.08.2025–27.02.2026
- **Бюджет:** 147,5 человеко-дней; выпуск на бой: 25.5100/25.6100 (внедрение не требуется)
- **Аудитория:** ~6 500 аккаунтов (Q1 2025)

## Страницы в scope

| Страница | Основная работа |
|---|---|
| Скидки и акции | React-таблицы, карусель→React, оптимизация зависимостей, курсорная навигация |
| Дисконтные карты/Клиенты | Порционная загрузка, оптимизация GetListWithStats, React-карточка |
| Бонусы/Клиенты | Вынос фильтра в отдельный слайс, новый лёгкий API настроек |
| Бонусы/Покупки | React-таблицы, порционная загрузка |
| Промокоды | React-реестр партнёрских промокодов, множественная навигация |
| Реферальная система/Покупки | Порционная загрузка |

## Новое API

- **`Bonus.GetBaseSettings`** → `{IsActive: bool, FranchiseRole: 1/2/null}` — замена тяжёлого `Bonus.ViewSettings` на страницах Бонусы/Клиенты и Бонусы/Покупки

## Методы BL под оптимизацию

| Метод | Тип работ |
|---|---|
| `Promotion.GetList` | Переход на курсорную навигацию |
| `Promotion.GetSaleList` | Порционная загрузка (мин. время выполнения 250 мс) |
| `DiscountCard.GetListWithStats` | Оптимизация + порционная загрузка |
| `Bonus.GetClientListWithStats` | Порционная загрузка |
| `Bonus.GetSaleList` | Порционная загрузка |
| `PromoCode.GetList` | Порционная загрузка + множественная навигация |
| `ReferralBonus.GetSaleList` | Порционная загрузка |
