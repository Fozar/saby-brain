## Что описывают документы

**Описание.pdf** — основной документ. Содержит:
- Краткое описание: сервис в Kubernetes, автомасштабирование, Redis-only (без собственной БД)
- 7 архитектурных компонентов: API-слой, очередь Redis, CustomEntryPoint, служебный пул, модуль отчётности, мониторинг задач, автомасштабирование
- Задачи сервиса: конвертация БД, разворот данных, конвертация эластиков, население индексов, отчёты анализа БД
- Схема взаимодействия: version manager → Redis → database-converter (Pods) → jinnee → db-hosts; параллельно: cloud control api, file-transfer, distribution storage, fs-public

**База данных.pdf** — одна строка: «Нет базы данных». Сервис работает исключительно с Redis.

**Организация кода.pdf** — сервис наследуется от `basic_platform_no_db`, единственный модуль `DatabaseConverter`. Зависимости: DiskClient, EntryPoint, Offline DB Converter, PermissionChecker, UserServiceCloud. Репозиторий: `https://git.sbis.ru/orchestration/admin.git`.

**Параметры облака.pdf** — 7 облачных параметров в разделе `Управление версиями → Конвертеры`:
задержка после масштабирования, использование Redis для отчётов, потоковая передача логов, процент воркеров, размер буфера логов, таймаут масштабирования, подробные сообщения jinnee.
