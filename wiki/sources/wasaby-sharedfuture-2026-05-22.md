# Эффективная работа с удаленными запросами. Shared future

Статья технического писателя Сизоненко Р.В. из сообщества `wasaby.Backend` (01.12.2025). Разбирает механизм shared future для параллельного выполнения удалённых запросов к БД в Wasaby Framework.

## Ключевые тезисы

- Наибольший прирост производительности достигается **распараллеливанием удалённых запросов** к БД
- Суммарное время = время **самого долгого** запроса (вместо суммы всех)
- API доступен на Python (`FutureInvoke`) и C++ (`Invoke<sbis::shared_future<T>>`)
- `ParallelTasks` — класс для тонкого управления пулом задач; поток освобождается сразу, не ожидая текущего вызывающего потока

## Три ограничения

1. **Лимит потоков**: по умолчанию 16; превышение → синхронный fallback
2. **Ожидание при выходе**: платформа дожидается незавершённых FutureInvoke перед отдачей ответа
3. **Отдельная транзакция**: нет доступа к незафиксированным данным основного потока
