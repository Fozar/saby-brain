## Алгоритмы и процессы

### Копирование дистрибутива (fix → prod)

1. distribution storage (prod) запрашивает полную структуру у distribution storage (fix)
2. Получает ссылки на файлы и образы
3. Загружает файлы из filestorage(fix) → перегружает в filestorage(prod)
4. Через datacenter(prod) запускает перенос образов из registry(fix) → registry(prod)
5. Получив новые ссылки, сохраняет структуру с обновлёнными ссылками в distribution storage (prod)

## Архитектура

### Генерация saby-пакетов

Выполняется внутри сервиса distribution storage. Компоненты:

| Компонент | Роль |
|-----------|------|
| Хранилище мета-описаний | Дистрибутив типа «saby-packet-meta»: zip с .sabypack файлами (JSON) |
| Парсер мета-описаний | Извлекает описания пакетов, приводит к внутреннему представлению |
| Источники текущих версий | Запросы к API cloud-ctrl: облачные приложения, кластеры, пакеты данных |
| Компонент вычисления версий | По версиям из облака определяет диапазоны (`>=текущая <следующая_мажорная`) |
| Компонент версионирования пакетов | Версия пакета = max(версии входящих приложений), сборка = инкремент |
| Модуль валидации | Проверяет домены, типы зависимостей, дубли |
| Модуль сохранения | Упаковывает JSON → zip → filestorage; создаёт запись в БД |
| Обработчик событий загрузки | При добавлении файла: если saby-пакет — обновляет зависимости; если мета — валидирует |

### Публикация дистрибутива

Перевод из внутреннего контура → публичный (public distribution storage).

Участники:
- **Публичный API** (`DistrVersion.Publish`) — входная точка; только внутри контура
- **Копирование образов в публичный registry** — Docker-образы через datacenter; заполняет `Image.PublicHost`
- **Установка признака публичности** — флаг `Public = True` в ВерсияПродукта
- **Механизм heartbeat** — периодические сигналы в cloud-ctrl о ходе длительной публикации
- **Механизм callback** — уведомление cloud-ctrl о результате (успех/ошибка)

Всё в рамках единой транзакции: ошибка при копировании образов или обновлении БД → публикация не фиксируется.

Файлы НЕ копируются — при запросе генерируются временные персональные ссылки.

### Загрузка дистрибутива из публичного хранилища

Инициатор: локальный distribution storage автономного облака. Источник: public distribution storage мастер-облака.

Процесс:
1. Локальный DS запрашивает структуру через `DistrVersion.Copy()`
2. Файлы скачиваются через временные ссылки, образы копируются из публичного registry в локальный через datacenter
3. Идентификаторы файлов и URL образов заменяются на локальные
4. Компонент сохранения создаёт в локальном хранилище полностью независимую версию

Поддерживаются heartbeat и callback.

### Задачи планировщика

| Задача | Периодичность |
|--------|--------------|
| Очистка репозиториев | Ежедневно — удаление устаревших дистрибутивов |
| Очистка образов | Ежедневно — анализ хранилища образов |

### Параметры облака (группа «Управление версиями.Хранилище дистрибутивов»)

| Параметр | Тип | Описание |
|---------|-----|---------|
| Очистка.Период очистки старых версий | число дней | Через сколько дней удаляются неиспользуемые сборки |

| Очистка.Переопределение | JSON | Переопределение параметров для конкретных дистрибутивов |
| Очистка.Исключения | массив строк | Дистрибутивы, исключённые из очистки |
| Очистка.Срок хранения скриптов | число дней | Срок удаления архивов скриптов |
| Хранилище файлов.Имя хранилища | строка | Namespace в filestorage для основных файлов |
| Хранилище файлов.Мобильные дистрибутивы.Имя хранилища | строка | Namespace для мобильных дистрибутивов |
| Хранилище файлов.Сайт | строка | Системное имя сайта для загрузки/отдачи дистрибутивов |
| Хранилище файлов.СрокЖизниФайла | число секунд | TTL файла в filestorage при загрузке — защита от «потерянных» файлов |

## Сервис public distribution storage

**Задачи:**
- Публичный доступ к saby-пакетам и дистрибутивам для автономных облаков
- Выдача метаданных только авторизованным клиентам с проверкой прав на конкретные пакеты

**Концепт:**
- Надстройка над основным distribution storage, **общая БД**
- Публикация: образы копируются в публичный registry, файлы — нет (временные ссылки)
- Вынесен в отдельный компонент — публикуется в управляющем домене (registry.saby.ru)
- Основной distribution storage остаётся внутренним, недоступным снаружи

**Ключевые API:**
- `SabyPacketList` — список saby-пакетов
- `SabyPacketComposition` — полный состав пакета (дистрибутивы и версии)
- `DistrVersionGetFullStructure` — структура дистрибутива со ссылками на файлы и образы

**Взаимодействие:**
- Автономное облако → запрашивает структуру, скачивает файлы по временным ссылкам
- Внутренний distribution storage → хранит признак публичности; через него проходят команды публикации
- filestorage → физическое хранение; временные ссылки ведут напрямую (с ограничением по времени и клиенту)
- registry (Harbor / Gitea) → Docker-образы
