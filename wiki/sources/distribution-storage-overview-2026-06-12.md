## Содержание

### Участок и задачи

Хранилище дистрибутивов — **middleware-сервис** управления метаданными и ссылками, не файловое хранилище. Физически файлы хранятся в filestorage, образы контейнеров — в registry (Harbor).

Задачи:
- Приём, учёт и предоставление по ссылкам дистрибутивов приложений всех типов для сервиса «Управление версиями»
- Копирование дистрибутивов и образов между стендами, поиск модулей, сравнение сборок, удаление неиспользуемых артефактов
- Публикация утверждённых версий и образов в публичный контур, контролируемая выдача автономным облакам
- Специализированный API для мобильного магазина приложений (Saby AppMarket)

### Три сервиса

| Сервис | Назначение |
|--------|-----------|
| **distribution storage** | Центральный сервис: учёт артефактов сборки, ссылки на filestorage и registry |
| **app-market storage** | Только чтение — API для Saby AppMarket, без авторизации |
| **public distribution storage** | Публичная витрина для внешних (автономных) облаков |

### Основные компоненты системы

**Провайдер сборки:**
- Вход: скомпилированные ресурсы
- Формирует zip-архивы модулей и Docker-образы
- Загружает архивы → filestorage, образы → registry (Harbor)
- Передаёт метаданные версии в distribution storage

**Сервис distribution storage:**
- Принимает метаданные от провайдера сборки
- Обслуживает запросы от version-manager (Хоттабыч): ссылки, копирование, поиск
- Не хранит файлы физически, оперирует ссылками на filestorage и registry

**Очистка хранилища:**
- Периодически опрашивает облачные сервисы для списка установленных версий
- Удаляет версии, не отмеченные как используемые, с учётом давности и лимита сборок

**Реестр «Дистрибутивы»:**
- Интерфейсный модуль в Хоттабыче для ручного управления

### Взаимодействие с внешними сервисами

- **version-manager (Хоттабыч)** — основной потребитель; содержит реестр «Дистрибутивы» (UI)
- **Saby AppMarket** — мобильный клиент, получает список приложений и версий через app-market storage
- **Автономное облако** — через public distribution storage скачивает saby-пакеты для изолированной среды
- **cloud-ctrl apps** — передают информацию об установленных версиях; управляют публикацией
- **filestorage** — физическое хранилище zip-архивов модулей
- **Image Registry (Harbor)** — хранилище Docker-образов
- **datacenter** — сервис для взаимодействия с Image Registry
