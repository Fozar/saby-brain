# release-plans-registry-2026-06-12

## Summary

7 PDF-документов из подпапки «Реестр работ_Планы выпуска/Планы выпуска» — пользовательская документация по системе планов выпуска Хоттабыча (vm.sbis.ru). Включает ТД продукта, описание алгоритмов работы, структуры плана, создания нового плана, элементов управления UI, и два тестировочных документа — инструкцию и вопросы-ответы по работе тестировщиков с планами.

## Key Claims

- Планы выпуска — самостоятельное приложение `release-manager` в рабочем облаке; все Хоттабычи (тестовые и боевые) работают с одним сервисом через `release-external` (= `release-ps` + `release-bl-proxy`)
- Семь типов работ с индивидуальными параметрами запуска; для скриптов список сервисов хранится в файле `services` внутри архива
- Планировщик раз в минуту анализирует структуру плана и ограничения; четыре типа ограничений запуска
- Тестировщик фиксирует билд (нажатием «замка» когда версия есть на стенде) и ставит «Проверено» — только после этого выпускающий может выгрузить дистрибутив на prod через «покебол»
- Автообновление fix: раз в час 09–18:00, условия — нет кирпича, нет «Проверено», нет явного билда в поле; стоп-лист включает ~10 системных сервисов и основные сервисы
- Публичный API: 4 метода (`GetWorkDependencies`, `PlanList`, `UpdateWork`, `WorkList`) в объекте `ReleaseManagerAPI`; документация на wi.sbis.ru
- Контроль нагрузки плана: нагрузка = сумма весов логик активных задач; лимит настраивается параметром облака и в реальном времени

## Notes

- Документы из подпапки «Реестр работ» — это пользовательский взгляд на планы выпуска (со стороны реестра работ)
- ТД датирована изменениями 2021–2023 (авторы: Волосатов Л.В., Кузнецов А.П.)
- «Покебол» — разговорное название кнопки синхронизации дистрибутивов в панели быстрого доступа
- Тестировочные документы описывают существенно иную перспективу (QA-роль), поэтому включены в тот же батч, но не дублируют содержание ТД
