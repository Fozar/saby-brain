## Ключевые детали

- Все эндпоинты на `online.sbis.ru`; аутентификация — `/auth/service/`, прочие — `/service/?srv=1`.
- Идентификаторы участников ЭДО и ссылки (кроме `СсылкаДляКонтрагент`) — нестабильны, не хранить в ИС.
- Реквизиты формализованного документа (Дата/Номер/Тип/Подтип/Версия/ПодВерсия/Название) передавать только полным набором.
- `Идентификатор` этапа нужно сохранять — он требуется при выполнении действия над документом.
- `data.details` в ошибках логируется, но не показывается пользователю.
