# Формат протокола JSON-RPC (wasaby.Backend)

Официальная документация по внутреннему JSON-RPC протоколу СБИС 3. Описывает структуру запроса/ответа, сериализацию Record/RecordSet и **версии протокола** (ключевое для отладки).

## Ключевые инсайты

### Версии протокола (`"protocol": N`)

| Версия | Поведение |
|--------|-----------|
| 1 | Совместимость с СБИС 2. `s` и `d` — JSON-объекты (`{"поле": "тип"}`) |
| 2 | Введён тип `"Идентификатор"`, изменён `"Иерархия"`. `s`/`d` — ещё объекты |
| 3+ | `"n"` поля перенесено внутрь формата. `s` = массив, `d` = массив |
| 7 | Текущая версия веб-клиента. Все запросы с UI = `"protocol":7` |

**Веб-движок >= 3.0.15 автоматически проставляет `"protocol">=2`.**

### Protocol 1/2 — Record/RecordSet как объекты

```json
{"s": {"ИмяПоля": "Строка"}, "d": {"ИмяПоля": "значение"}}
```

- `s` — объект `{имяПоля: тип}`
- `d` — объект `{имяПоля: значение}`
- Нет полей `_type`, `f`, `tp`

### Protocol 3+ (в т.ч. 7) — массивы

```json
{
  "s": [{"n": "ИмяПоля", "t": "Строка"}],
  "d": ["значение"],
  "f": 1,
  "_type": "record"
}
```

- `s` — массив форматов `{n, t}`
- `d` — массив значений (порядок = порядку `s`)
- `f` — идентификатор формата, **начиная с 1** (не 0!)
- `_type` — `"record"` или `"recordset"`

### Поле `f` — оптимизация, не тип

`"f"` — это идентификатор формата для **дедупликации схем** в сложных вложенных структурах. Первый уникальный формат = `f:1`, второй = `f:2` и т.д. При повторном использовании того же формата поле `s` можно опустить — указать только `"f": N`.

> [!key-insight] Диагностика ошибки "Блок данных должен быть объектом"
> Эта ошибка означает: BL-метод ожидает `d` как JSON-объект (Protocol 1/2), а не массив (Protocol 3+). Либо `Фильтр` должен быть простым объектом без record-обёртки. Проверь, какой формат ожидает конкретный метод.

## Структура объекта ошибки

```json
{
  "code": -32000,
  "message": "...",
  "details": "...",
  "data": {
    "classid": "uuid класса исключения",
    "error_code": -1,
    "addinfo": null
  },
  "user_hint": "...",
  "type": "error"
}
```

- `error_code: -1` = исключение без назначенного кода
- `classid + error_code` — уникальная пара для отладки
- `details` — только в лог, не показывается пользователю
