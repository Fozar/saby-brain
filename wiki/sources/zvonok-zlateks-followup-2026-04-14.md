## Ключевые находки

### 1. «Соглашение Шрёдингера» — статус в реестре vs. в документе

В реестре партнёров (под владельцем, реферальная программа Тензор):
- Партнёр **Zlateks (Золотек/Златказс)** показывает статус **«отправлено»** (приглашение не принято).
- При открытии карточки партнёра — документ имеет статус **«принято на обслуживание»**.
- При обновлении страницы: сначала «не принято» → через секунду переключается на «принято».

> [!key-insight] Диагноз Симакова
> Документ **принят** (документная сущность имеет корректный статус). В реестре статус отображается **некорректно** — произошёл затуп (лаг) на проде, и метод обновления статуса в реестре не отработал нормально. Это объясняет расхождение.
> Доказательство: партнёр **имеет доступ** на портал (Канатьева подтвердила — они заходят через Сабинет → стрелочка → личный кабинет), а доступ проверяется по статусу документа, не реестра.

### 2. Способ входа партнёра

Канатьева учит клиентов заходить через **Сабинет → стрелочка → личный кабинет**, не через прямую ссылку. Это стандартный сценарий (проверялся в v2200).

Тесты на несанкционированный доступ (без принятия приглашения):
- По прямой ссылке на реестр офферов (`/page/partner-feed-offers`) — отпинывает на разводящую порталов. ✓
- По ссылке с реферальным хвостиком — хвостик обрезается, возвращается на разводящую. ✓
- Из реестра группы без принятия — разводящая порталов. ✓

**Вывод:** партнёр не мог попасть на портал без принятия приглашения (защита работает).

### 3. ВОЗ-ошибка на офферах

При заходе под Zlateks в раздел офферов — ошибка **«контрагент партнёра не найден»**.

Симаков: нужно понять, откуда оффер-страница дёргает статус. Скорее всего, она читает из конкретного договора по его ID (а не из реестра), и договор принят. Если читает из реестра — это архитектурная проблема (реестр может тупить → оффер не отдаётся).

### 4. Баг Земцовой (новый) — ошибка входа на новый портал

Во время сессии Земцова обнаружила, что **не может зайти на новый портал** (старый работает):
- Падает метод `account.service.login` — «внутренняя ошибка сервера, текущий клиент не зарегистрирован в системе».
- Симаков: обратиться к **Егору Платонову** за диагностикой.

### 5. Баг «создаётся в 0.05» (0.5-я версия) — нет ответа от разработки

Открытый баг в релизе 0.5 (что-то создаётся в 0.05) — Симаков писал Коле, информации о готовности фикса нет. Возможен перенос в v11.
