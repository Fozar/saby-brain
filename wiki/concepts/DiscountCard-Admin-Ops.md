# Discount Card Admin Ops

Source: `raw/Пользовательская документация (для смежных команд).md` | Domain: [[price-formation/_index]]

## Реестр типов карт на inside.sbis.ru

Административная панель для мониторинга активности использования карт по регионам. Основные пользователи — менеджеры.

**Фильтры:**
- **Активные** — выдача карт или применение на продаже за последние 30 дней
- **Ушедшие** — типы карт, по которым активность прекратилась

**Особенности:**
- Если карты не выдавались — дата создания типа считается датой последнего использования
- Если тип заблокирован, но последнее использование < 30 дней — считается что 31 день (форсированный переход в "Ушедшие")
- Если карты выдавались — дата выдачи 1-й ДК считается датой создания типа

### Стек хранения: ClickHouse

Данные агрегируются в ClickHouse для эффективной фильтрации больших объёмов (аккаунты × заведения × типы карт).

**Схема в ClickHouse:**

| Поле | Тип | Описание |
|------|-----|----------|
| `esteblishment_uuid_card_type_id` | str | `<UUID заведения>_<СДК-ИД типа карты>` |
| `region` | int | Код региона |
| `location` | int | Код населённого пункта |
| `card_type_uuid` | str | Глобальный UUID типа карты |
| `account_id` | int | Облачный ID аккаунта |
| `esteblishment_uuid` | str | UUID заведения |
| `category` | int | Категория заведения |
| `cards_count` | int | Количество выданных карт |
| `last_used` | date | Дата последнего использования |
| `created` | datetime | Дата создания 1-й ДК |

**Синхронизация в ClickHouse:**
- Используется брокер синхронизации с "фейковым" UUID (составной ключ `<UUID заведения>_<СДК-ИД типа>`)
- События: создание/удаление типа карты, изменение ТП, создание/удаление заведения, выдача карты, смена активности
- Задачи планировщика:
  - `StatCardType.ResetSync` (ночная) — выставляет признак первичной синхронизации
  - `StatCardType.Sync` (каждые 2 минуты) — первичная (`CardTypeEst.FirstSync`) или регулярная (`CardTypeEst.RegularSync`)
- Небоевые аккаунты (удалённые, заблокированные, демо) отсекаются

## Обновление образов дисконтных карт (pass-updater)

Образы **обновляются автоматически** при наличии соответствующих блоков на карте:

| Блок на карте | Триггер автообновления |
|---------------|----------------------|
| Срок действия | Изменение срока на вкладке "Описание" — массово |
| Скидка / кешбэк (бонусы) | Изменение привязанных акций или их процентов — массово |
| Баланс бонусов / покупки / посещения | Ручное изменение или после продажи — 1 образ |
| Штампики | Изменение связанной акции, её условий или накоплений при продаже |
| Реферальная ссылка (обратная сторона) | Смена владельца карты |

При переводе акции в неактивные или удалении — образы обновляются массово.

**Ручное обновление необходимо** при изменении:
- настроек гео-уведомлений
- имени клиента (при наличии блока с именем)
- настроек дизайна или выбора другого дизайна

## Apple Wallet — настройка

| Параметр | Описание |
|----------|----------|
| Team ID | Идентификатор iOS-аккаунта разработчика |
| Pass Type ID | Идентификатор приложения в аккаунте |
| Сертификаты | Pass Type ID + WWDR (Apple WWDR CA G4) |
| Цепочка | `<базовый Apple>* — Pass Type ID — WWDR` |

Домен `api` должен быть включён pass-updater и доступен извне облака (для URL, по которому iOS-устройства обращаются за образами).

В `mobile-service-internal` хранится Pass Type ID для push-уведомлений.

**Сервисы сертификатов:**
- `hsm` — хранит сертификат и ключ, подписание и расшифровка происходят здесь
- `cvs` — проверяет действительность сертификата

## Google Pay — настройка

| Параметр | Описание |
|----------|----------|
| Аккаунты | Merchant + под-аккаунт эмитента |
| ID эмитента | Из Google |
| e-mail сервисного аккаунта | Эмитента |
| Private JSON-ключ | Хранится в интеграции `gpaypasses` (сервис интеграций) |

Для публикации образов (не демо-режим) необходим запрос доступа в Google. pass-updater должен иметь доступ в интернет.
