# Реферальная система сделок

Navigation: [[price-formation/_index]] | [[ReferralProgram-Module]] | [[Loyalty-Database-Schema]]

Подсистема предоставляет прозрачный механизм учёта и тарификации сделок, создаваемых в процессе продвижения Партнёрами продуктов Владельца реферальных программ.

> [!note] Отличие от ReferralProgram-Module
> [[ReferralProgram-Module]] описывает онлайн-реферальную программу (utm-ссылки, РКО в банке). «Реферальная система сделок» — отдельная подсистема на платформе SabyNet, ориентированная на явные сделки (Leads) с расчётом вознаграждений Партнёрам.

---

## Понятийная модель

| Термин | Определение |
|--------|-------------|
| **Владелец** | Компания — собственник продукта и создатель реферальной сети |
| **Партнёр** | Участник сети, продвигает продукт Владельца за вознаграждение |
| **SabyNet** | Платформа партнёрских взаимоотношений (ЛК Владельца и Партнёра) |
| **Покупатель** | Потенциальный потребитель, которого привлекает Партнёр |
| **Реферальная программа** | Постановка цели продвижения: web-адрес + тарификация вознаграждения |
| **Участник** | Партнёр, заявивший об участии в конкретной программе |
| **Реферальный код** | Уникальный идентификатор участника в программе |
| **Реферальная ссылка** | web-адрес с реферальным кодом Партнёра в параметре |
| **Сделка (Lead)** | Результат продвижения; может завершиться успешно или нет |
| **Вознаграждение** | Начисление бонусных баллов на карту Партнёра по успешной сделке |

---

## Декомпозиция подсистем

| Подсистема | Ответственность |
|-----------|----------------|
| **SabyNet** | Партнёрские отношения, ЛК Владельца и Партнёра, приглашения |
| **Маркетинг (advert-service)** | Статистика посещений/сделок по источникам; счётчик посещений |
| **Реферальная сеть** (описываемая) | Реестр программ, статистика, виджеты, генерация ссылок, ручное создание сделок |
| **Сделки (crm)** | API создания сделок (Deal.Create) |

> [!key-insight] Вознаграждение = бонусы
> Вознаграждение Партнёра реализовано как начисление бонусных баллов по карте Партнёра, где Бонусная программа = Реферальная программа.

## API подсистемы

### sabyReferralProgram.CreateLead

Единственный публичный метод. Внешние системы и партнёры создают сделки через него.

**Параметры вызова:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| AgentGroupID | String | Идентификатор бизнес-группы |
| ReferralProgramID | Integer | ID реферальной программы |
| Params | JSON | Параметры сделки |

**Params (параметры сделки):**

| Параметр | Тип | Описание |
|----------|-----|----------|
| ClientName | String | Название организации / ФИО клиента |
| OrgInn | String | ИНН организации |
| OrgKpp | String | КПП организации |
| ContactName | String | Имя контактного лица (обязательно для орг.) |
| ContactPhone | String | Телефон контактного лица |
| ContactEmail | String | Email контактного лица |

> [!note] Решение не коробочное
> На текущий момент каждое внедрение реферальной системы в сторонние участки происходит индивидуально по проектам с согласованием тех. решений.

## Алгоритмы начисления вознаграждений

### При смене статуса сделки

Событие `Lead.StateChanged`, состояние «Завершено: Положительно»:
1. По источнику сделки определяем принадлежность к реферальной программе (программа + реф. код)
2. Определяем программу и партнёра-участника
3. Вычисляем сумму вознаграждения (с учётом индивидуальных значений), сохраняем начисление текущей датой

### Ежесуточный планировщик (посещения)

Сервис Маркетинга группирует Источники с типом `Referral_program`:
1. Внешние идентификаторы = ID реферальных программ → получаем Источники партнёров-участников
2. У Маркетинга запрашиваем кол-во посещений с начала месяца по источникам партнёров
3. Для каждого партнёра рассчитываем вознаграждение и сохраняем начисление текущей датой

## База данных

Таблицы (из [[Loyalty-Database-Schema]]):

| Таблица | Роль в реферальной системе |
|---------|---------------------------|
| **ВидЦены** | Описывает условие реф. программы: лендинг, логотип, вознаграждения за лид/посещение, тема сделки |
| **Карта** | Факт участия Партнёра в программе. Содержит индивидуальные ставки и уникальный источник (AdObject) |
| **ВидЦеныДокумент** | Начисление вознаграждения (бонусов) Партнёру. Документ=сделка (Lead) или NULL (посещение) |

### Типовые выборки (10 запросов)

1. Список реферальных программ партнёра — ВидЦены(REFERRAL_LEAD) → ВидЦеныВидКарты → Карта
2. Список партнёров в программе — ВидЦены → ВидЦеныВидКарты → Карта (Заблокировано=false)
3. Вознаграждения партнёра по сделкам — ВидЦеныДокумент WHERE Бонусы > 0 + JOIN Карта
4. Вознаграждения за посещения — ВидЦеныДокумент WHERE Документ IS NULL AND Бонусы > 0
5. Статистика по периодам — generate_series + SUM(Бонусы) по месяцам/годам
6. Совокупная статистика по программе — GROUP BY ВидЦены или Карта.Лицо
7. Проверка выданных кодов — EXISTS(Карта WHERE Эмиссия=@ВидКарты)
8. Источники и индивидуальные ставки — Карта.Атрибуты.ReferralProgram: AdObject, PricePerLead, PricePerVisitor
9. UUID реферального кода — Карта.Идентификатор по Лицо
10. Сделки с начисленными бонусами — ВидЦеныДокумент WHERE Бонусы > 0 BETWEEN DateBegin AND DateEnd

### Индексы

| Индекс | Используется в выборках |
|--------|------------------------|
| ВидЦены(Раздел) WHERE Раздел IS NOT NULL | 6 |
| Карта(Лицо) | 1, 8, 9, 10 |
| Карта(Эмиссия, ПорНомер) WHERE NOT NULL | 1, 2, 7, 8, 9 |
| ВидЦеныДокумент(ВидЦены, EffectiveDate) WHERE Документ NOT NULL | 3, 5, 10 |
| ВидЦеныДокумент(EffectiveDate) WHERE NOT NULL | 4, 5, 10 |

## Интерфейс (виджеты)

Виджеты объединены в группу `referralLead`, добавлены в [виджет-группу online](https://git.sbis.ru/online/inside).

| Виджет | Для кого | Описание |
|--------|----------|----------|
| **Офферы** | Партнёр | Таблица: оффер, баннер, посетители, заявки, продажи, % завершения, сумма вознаграждений |
| **Офферы и партнёры** | Владелец | Аналогично + разрез по партнёрам; два таба (офферы/партнёры) |
| **Станьте участником** | Партнёр | Карусель офферов, в которых партнёр ещё не участвует; кнопка «Участвовать» |

**Ограничение видимости**: нет встроенного механизма в магазине виджетов. Проверка роли происходит через `AgentContract.AccessData` (поле GroupId / Status) на уровне предзагрузки каждого виджета.

**Страницы:**
- Владелец: `owner-offers` (pagex)
- Партнёр: `participant-offers` (pagex)

## Распределение прав

| Функционал | Партнёр: доступ к участку | Партнёр: роль в группе | Владелец: доступ к участку | Владелец: роль в группе |
|-----------|--------------------------|------------------------|---------------------------|------------------------|
| Отправить заявку | Мин. «Просмотр» | «Участник» / «Менеджер» | Мин. «Просмотр» | «Участник» / «Менеджер» |
| Просмотр заявок и офферов | Мин. «Просмотр» | «Участник» / «Менеджер» | Мин. «Просмотр» | «Участник» / «Менеджер» |
| Создание/редактирование офферов | Нет доступа | Нет доступа | «Просмотр и изменение» | «Администратор» / «Модератор» |

Участок: `Бизнес/Маркетинг/Реферальные программы` (входит в роль «Менеджер» и «Администратор системы») или `Бизнес/Система лояльности`.

Метод проверки: `BusinessGroup.GetPersonRoleInAgents/1` — вызывается на предзагрузке каждой страницы.

> [!note] Версии
> При обновлении online аккаунты в рамках одной бизнес-группы (экземпляра конфигурации SabyNet «Реферальная сеть») могут быть на разных версиях.
