# Saby Service Config: Параметры vs Переключатели функционала

Два механизма изменения поведения сервисов без изменения кода.

> [!key-insight] Главное правило выбора
> **Параметры** = долгоживущие настройки поведения сервиса. **Переключатели** = временные, для вывода нового функционала; после вывода — удаляются.

## Параметры сервисов

### Что это

Набор пар (ключ → значение), загружаемый при старте сервиса. Доступ:

```cpp
Config::Instance().Get("МойПараметр")   // C++
```

```python
ConfigGet("МойПараметр")               # Python
```

Задаются в ini-файле сервиса или в "Управлении облаком" → "Настройки".

### Достоинства

- Простой и понятный механизм
- Потокобезопасны внутри рабочего процесса (не меняются пока процесс жив)
- Дёшево читать: нет сетевых запросов
- Хорошо автоматизировано задание через облако

### Недостатки

- **Требуют перезапуска сервиса** для применения изменений (или медленного rolling-restart)
- Массовый перезапуск сотен сервисов — дорогостоящая операция для ЦОД
- Нельзя задать для конкретного клиента/пользователя
- **Недоступны во фронте** и в мобильных/offline (только из ini-файлов)
- Трудно отследить устаревшие параметры (превращаются в мусор)
- Могут расходовать память при большом количестве или огромных значениях

### Именование параметров

Обязательно: **минимум 3 уровня**, разделённые точками. Пример: `Ядро.DB Conversion.Debug`.

При регистрации создаётся иерархия разделов в справочнике "Настройки":
- Корень → Подраздел → Лист

**Корневые разделы** (первый уровень) нельзя создавать самостоятельно — нужно оформить поручение на **Драгутенко А.А.** Несогласованный раздел → ошибка при регистрации параметра.

Правила названий:
- Целиком по-русски **или** целиком по-английски (смешивание запрещено)
- Первая буква раздела и первая буква после точки — заглавная
- Последний символ раздела и символ перед точкой — буква или цифра
- Описание параметра: понятно для "Эксплуатации ЦОД", с форматом и единицами измерения

### Временные параметры (в роли переключателей)

Иногда параметр используют как переключатель из соображений производительности. Обязательное соглашение:

```
Temporary.Module.Name
```

Минимум трёхуровневый. Легко найти и удалить через поиск по префиксу `Temporary.`.

## Переключатели функционала

### Что это

Пара (ключ → состояние). Ключ — строковый идентификатор ≤20 символов (строчная латиница, цифры, `-`, `_`). Два состояния: включён/выключен.

Детальный API: [[Saby-Feature-Toggles-API]]

### Достоинства

- **Не требуют перезапуска сервисов**
- Можно задать для конкретного пользователя или группы клиентов
- Доступны в **frontend, backend, мобильных и offline-приложениях**
- Можно скрывать пункты аккордеона и UI-компоненты
- Использование отслеживается в статистике

### Недостатки

- Обращение дороже: кэшируются, но могут требовать сетевой запрос (ненадёжно)
- Только two значения: включён/выключен (расширенные значения — для редких случаев)
- Изменения с лагом из-за распределённости (аналогично параметрам)
- Сложны в реализации платформы

### Описание переключателя

Критически важно. Пример плохого описания:
> "Показ демо-приложений в новом режиме"

Проблемы: непонятно что за "новый режим", некорректно описывает действие, опечатка.

Хорошее описание: что именно включается, на что влияет, в каком контексте.

## Таблица сравнения

| Критерий | Параметры | Переключатели |
|----------|-----------|---------------|
| Перезапуск при изменении | Да | Нет |
| Гранулярность | Сервис | Клиент / Пользователь |
| Доступ с фронта | Нет | Да |
| Доступ в mobile/offline | Только из ini | Да (с лагом) |
| Стоимость чтения | Дёшево (нет сети) | Дороже (кэш + возможный сетевой) |
| Время жизни | Долго | **Временно** — удалять после вывода |
| Множество значений | Да (строки) | Только bool (+ опционально строка/флаги) |
| Использование в статистике | Нет | Да |

## Когда что выбирать

```
Нужно изменить поведение для ВСЕХ без перезапуска?
  → Нет → Параметр (если производительность критична)
  → Да  → Переключатель

Нужна гранулярность до клиента/пользователя?
  → Переключатель (параметры так не умеют)

Нужен доступ с фронта или из мобилки?
  → Переключатель

Это долгосрочная настройка (навсегда)?
  → Параметр (переключатели должны удаляться)

Временный переключатель, но критична производительность?
  → Параметр с префиксом Temporary.Module.Name
```
