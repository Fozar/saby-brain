# Saby API — Объект «Навигация» (пагинация)

Списочные методы выдают результат постранично. Управление осуществляется через объект `Навигация` в параметрах запроса и ответа.

Есть два варианта: стандартный (для большинства команд) и упрощённый (только для `СБИС.СписокИзменений`).

---

## Стандартная навигация

### Запрос

В объект `Фильтр` добавляется вложенный объект `Навигация`:

```json
{
  "Фильтр": {
    "Тип": "ДокОтгрИсх",
    "Навигация": {
      "РазмерСтраницы": "25",
      "Страница": "0"
    }
  }
}
```

| Параметр | Тип | По умолчанию | Диапазон |
|----------|-----|-------------|----------|
| `РазмерСтраницы` | строка | 25 | 1–200 |
| `Страница` | строка | 0 | с 0 |

### Ответ

Результат дополняется объектом `Навигация` рядом с основными данными:

```json
{
  "Документ": [...],
  "Навигация": {
    "Страница": "0",
    "РазмерСтраницы": "25",
    "ЕстьЕще": "Да"
  }
}
```

| Поле | Описание |
|------|----------|
| `Страница` | Номер возвращённой страницы (с 0) |
| `РазмерСтраницы` | Фактический размер страницы |
| `ЕстьЕще` | «Да» / «Нет» — есть ли записи после текущей страницы |

---

## Упрощённая навигация (`СБИС.СписокИзменений`)

```json
"Навигация": {
  "РазмерСтраницы": "25"
}
```

- Диапазон `РазмерСтраницы`: 0–50, по умолчанию 25.
- В ответе: `{ "РазмерСтраницы": "...", "ЕстьЕще": "Да"/"Нет" }`.
- Нет поля `Страница` (используется курсор внутри `СБИС.СписокИзменений`).

---

## Типовой паттерн обхода всех страниц

```python
page = 0
while True:
    result = call("СБИС.СписокДокументов", {
        "Фильтр": {
            "Тип": "ДокОтгрИсх",
            "Навигация": {"РазмерСтраницы": "200", "Страница": str(page)}
        }
    })
    docs = result["Документ"]
    nav = result["Навигация"]
    process(docs)
    if nav["ЕстьЕще"] == "Нет":
        break
    page += 1
```

> [!note] Курсорная пагинация
> `СБИС.СписокЗадач` использует другой механизм — временнóй курсор через `Позиция.ДатаВремя`. Подробнее: [[Saby-External-API-Tasks]].
