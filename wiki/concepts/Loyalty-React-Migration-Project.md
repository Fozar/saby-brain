# Loyalty React Migration Project

«[Технический] Перевод страниц лояльности на React» — проект оптимизации производительности раздела Бизнес/Каталог/Скидки и акции.

## Цель и критерии

**Цель:** ускорить построение страниц лояльности с запасом по производительности (сейчас некоторые страницы на границе бюджета или за ней).

**Критерии достижения:**
- Отсутствуют ошибки производительности
- Все страницы переведены на React (online + Розница/Presto)
- Время построения страниц с таблицами уменьшено на 150–250 мс от разрешённого
- Замеры производительности организованы на постоянной основе

## Страницы в scope

### Скидки и акции (38 дней)

**UI:**
- Переход реестров мастера и продаж на React API (новые платформенные методики таблиц, отказ от WML-шаблонов)
- Перевод компонента плитки рекламы (карусель) на React
- Пересмотр библиотек компонентов: объединение конфигов/констант/фабрик → запас ~80 запросов
- Оптимизация зависимостей: PersonCard/miniCardUtils (-~1 МБ), карусель (-50 КБ, tailwind), рендер продаж (-100 КБ)

**БЛ:**
- `Promotion.GetSaleList` — порционная загрузка (мин. время выполнения без данных = 250 мс)
- `Promotion.GetList` — переход на **курсорную навигацию** (быстрее постраничной, устраняет баги неполных результатов при поиске)

### Дисконтные карты/Клиенты (20 дней)

**UI:**
- Удаление PersonCard/miniCardUtils (-~1 МБ)
- Удаление неиспользуемого WS.Core (сделать зависимость LoyaltyAnalytics/DiscountCard/Numbering динамической)
- Перевод карточки ДК на React + UI-тесты (тулбар, кнопки, верстка, ошибки)
- UI-тесты реестра типов карт и диалога создания типов

**БЛ:**
- `DiscountCard.GetListWithStats` — оптимизация + порционная загрузка
  - Ответственный: [[Тимошенко А.А.]] (Михайленко 3д, Омельяненко 1д)

### Бонусы/Клиенты (9 дней)

**UI:**
- Вынос фильтра в отдельный слайс (сейчас слайс виджета зависит от слайса списка → задержка ~300 мс)
- Замена `Bonus.ViewSettings` на `Bonus.GetBaseSettings` (~100 мс экономии)

**БЛ:**
- `Bonus.GetClientListWithStats` — порционная загрузка

### Бонусы/Покупки (7 дней)

**UI:** Переход на React API таблиц (+~50 мс)  
**БЛ:** `Bonus.GetSaleList` — порционная загрузка

### Промокоды (13 дней)

**UI:**
- Пересмотр библиотек (аналогично Акциям)
- Перевод реестра партнёрских промокодов на React

**БЛ:**
- `PromoCode.GetList` — порционная загрузка продаж + **множественная навигация** (фикс проблем с частичной загрузкой при разворачивании узлов)

### Реферальная система/Покупки (3 дня)

**БЛ:** `ReferralBonus.GetSaleList` — порционная загрузка  
(Использует общий React-компонент с Бонусы/Покупки — ускорение бесплатно)

### Прочие доработки / тех. долг (5,5 дней)

- **Предупреждение о 30+ типах карт**: если создаётся >30 типов, показывать предупреждение с инструкцией
- **Стекирование предупреждений при сохранении Акций**: сейчас показывается только одно из нескольких, нужно объединять в одно окно
- **Объединение механики валидации** акций/подсказок/наценок/промокодов: параметризовать общий код валидации

## Новое API

### Bonus.GetBaseSettings

Лёгкий метод-замена тяжёлого `Bonus.ViewSettings` для страниц Бонусы/Клиенты и Бонусы/Покупки.

```
Returns: Record {
  IsActive: bool   -- есть ли активные бонусные программы
  FranchiseRole: 1 | 2 | null  -- роль партнёра франшизы
}
```

БЛ разработчик: Матюшев (1д)

## Технологический стек (что меняется)

| До | После |
|---|---|
| WML-шаблоны + слой совместимости | tsx + React API таблиц |
| Постраничная навигация в списках | Курсорная / порционная навигация |
| Монолитные библиотеки компонентов | Укрупнённые библиотеки (констр./конфиги/фабрики) |
| `Bonus.ViewSettings` (тяжёлый) | `Bonus.GetBaseSettings` (лёгкий) |

## Ожидаемые прибытки производительности

| Страница | Ожидаемый эффект |
|---|---|
| Скидки и акции (реестры) | +50 мс (React таблицы) |
| Скидки и акции (запросы) | +80 единиц запасного бюджета (библиотеки) |
| Дисконтные карты/Клиенты | -~1 МБ трафика (сейчас 9,4 МБ при бюджете 10 МБ) |
| Бонусы/Клиенты | +300 мс (слайс) + 100 мс (API) |
| Бонусы/Покупки | +50 мс (React таблицы) |
