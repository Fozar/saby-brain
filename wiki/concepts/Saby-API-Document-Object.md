# Saby API — Объект «Документ»

Центральный объект EDI API Saby. Используется в `СБИС.СписокДокументов`, `СБИС.ПрочитатьДокумент` и других командах работы с ЭДО.

> [!note] Цветовая конвенция в документации
> - **Синие** поля — только для чтения (возвращаются сервером).
> - **Красные** поля — только для записи (передаются как аргумент).
> - **Остальные** — двунаправленные.

---

## Ключевые поля верхнего уровня

| Поле | Тип | Описание |
|------|-----|----------|
| `Идентификатор` | строка | Адресация к последней редакции |
| `Дата` | строка `ДД.ММ.ГГГГ` | Дата документа (фильтрация/сортировка) |
| `Номер` | строка | Номер документа; для отправки — уникальный |
| `Сумма` | строка | Сумма документа |
| `Тип` | строка | Тип документа (напр. `ДокОтгрИсх`) |
| `Подтип` | строка | Подтип типа документа |
| `Направление` | строка | Направление документа |
| `Состояние` | объект | Код состояния + Название + флаги |
| `Редакция` | массив | История редакций с `Актуален` |
| `Вложение` | массив | Прикреплённые файлы и ЭП |
| `Этап` | массив | Текущие этапы документооборота |
| `Событие` | массив | Аудит-лог событий |

---

## Состояние документа (`Документ.Состояние`)

```json
{
  "Код": "4",
  "Название": "Доставлен",
  "НеполнаяОбработка": "Нет",
  "Сложное": "Нет",
  "Комментарий": ""
}
```

Числовые коды состояний — в справочнике `/help/integration/api/sequence/cod/`.

---

## Участники (`НашаОрганизация` / `Контрагент`)

Оба объекта содержат:
- `GLN` — Global Location Number (приоритет при поиске).
- `СвЮЛ` — реквизиты юридического лица (ИНН 10 цифр, КПП, КодФилиала, КодСтраны ISO 3166-1).
- `СвФЛ` — реквизиты физического лица (ИНН 12 цифр, ФИО, СНИЛС).

> [!warning] Не сохранять в ИС
> Идентификаторы участников и ссылки (кроме `СсылкаДляКонтрагент`) могут изменяться оператором без уведомления.

---

## Вложение (`Документ.Вложение`)

| Поле | Описание |
|------|----------|
| `Идентификатор` | UUID вложения |
| `Тип` / `Подтип` | Тип формализованного вложения |
| `ВерсияФормата` / `ПодверсияФормата` | Из содержимого формализованного документа |
| `Файл.Ссылка` | GET-ссылка на бинарные данные |
| `Файл.ДвоичныеДанные` | Base64 (≤ 60 МБ) для отправки |
| `Файл.Хеш` | Base64 криптохэш |
| `Подпись` | Массив с `Сертификат` и `Файл.ДвоичныеДанные` |
| `Зашифрован` / `ТипШифрования` | Пользовательское / Локальное / Отсутствует |
| `Модифицирован` | Изменялось ли тело вложения перед этапом |
| `Служебный` | «Да» — служебное вложение, «Нет» — первичное |

### Подпись.Сертификат

Содержит ФИО подписанта, ИНН, Должность, ОГРНИП, Отпечаток, СерийныйНомер, Издатель, срок действия.  
Поле `Доверенность` — массив объектов МЧД (36-разрядный GUID, ИНН доверителя/представителя, срок действия).

---

## Этап и действия (`Документ.Этап`)

```json
{
  "Идентификатор": "...",
  "Название": "Подписание",
  "Служебный": "Нет",
  "Вложение": [...],
  "Действие": [
    {
      "Название": "Подписать",
      "ТребуетПодписания": "Да",
      "ТребуетКомментария": "Нет",
      "Сертификат": [...]
    }
  ]
}
```

`Идентификатор` этапа **необходимо сохранять** — он передаётся при подготовке и выполнении действия.

## Ссылки документа

| Поле | Описание |
|------|----------|
| `СсылкаДляНашаОрганизация` | Карточка в кабинете своей организации (может требовать auth) |
| `СсылкаДляКонтрагент` | Карточка в кабинете контрагента (**можно хранить в ИС**) |
| `СсылкаНаАрхив` | Архив последней редакции (документы + ЭП) |
| `СсылкаНаPDF` | PDF с впечатанными данными ЭП (пустое → PDF недоступен) |

---

## Реквизиты формализованного документа

Поля `Дата`, `Номер`, `Тип`, `Подтип`, `ВерсияФормата`, `ПодверсияФормата`, `Название` нужно передавать **все вместе** с корректными значениями. Частичная передача или неверные значения сломают отображение и PDF.

---

## Связанные документы

- `ДокументОснование` — массив документов-оснований (`Идентификатор`, `Дата`, `Номер`, `Тип`, `ВидСвязи`, `Сумма`).
- `ДокументСледствие` — массив следственных документов (аналогичная структура).
