# SBIS Record/Recordset Format

SBIS API передаёт структурированные данные в **колоночном формате** вместо привычных JSON-объектов. Ключевые поля: `f` (тип структуры), `s` (схема), `d` (данные).

## Версии протокола — критично для отладки

SBIS использует два принципиально разных формата в зависимости от версии протокола:

### Protocol 1/2 — объектный формат (старый)

```json
{"s": {"ИмяПоля": "Строка"}, "d": {"ИмяПоля": "значение"}}
```

`s` и `d` — **JSON-объекты**. Нет `_type`, `f`, `tp`.

### Protocol 3+ (текущий, включая 7) — колоночный формат

```json
{
  "s": [{"n": "ИмяПоля", "t": "Строка"}],
  "d": ["значение"],
  "f": 1,
  "_type": "record"
}
```

`s` и `d` — **массивы**. Веб-клиент всегда использует `"protocol":7`.

> [!key-insight] "Блок данных должен быть объектом"
> Эта ошибка = BL-метод ожидает Protocol 1/2 (d = объект), а ты передаёшь Protocol 3+ (d = массив). Или метод не принимает record-формат вообще.

## Поле `f` — идентификатор формата (не тип!)

`f` — номер для **дедупликации схем** в вложенных структурах. Первый уникальный формат = `f:1`, второй = `f:2`. При повторном использовании `s` можно опустить, указав только `f`. **Нумерация с 1**, не с 0. Для одиночного filter-record обычно `f:1`.

## Типы структур (`_type`)

| `_type` | `d` |
|---|---|
| `"recordset"` | массив массивов: `[[v1, v2], [v1, v2], ...]` |
| `"record"` | плоский массив: `[v1, v2, v3, ...]` |

### RecordSet — обязательные и опциональные атрибуты

| Атрибут | Обяз. | Описание |
|---------|-------|----------|
| `"_type":"recordset"` | ✓ | маркер типа |
| `"f"` | ✓ | идентификатор формата |
| `"d"` | ✓ | данные |
| `"s"` | — | формат полей (схема) |
| `"tp"` | — | имя формата записи |
| `"n"` | — | навигация (постраничный результат) |
| `"m"` | — | метаданные RecordSet |
| `"p"` | — | путь к узлу иерархии |

> [!note] Дедупликация схем
> При повторении одинакового вложенного формата указывается только `"f"`, а `"s"` опускается. Нумерация с 1. Пример: 100 записей с одинаковым вложенным Record — схема записывается один раз (`"f":2, "s":[...]`), остальные 99 — только `"f":2`.

### Record — атрибуты

| Атрибут | Обяз. | Описание |
|---------|-------|----------|
| `"_type":"record"` | ✓ | маркер типа |
| `"f"` | ✓ | идентификатор формата |
| `"d"` | ✓ | данные |
| `"s"` | — | формат полей |
| `"tp"` | — | имя формата |

## Пример recordset

```json
{
  "f": 0,
  "d": [
    [582954590, "2026-04-01", "Название задачи"],
    [582954591, "2026-04-02", "Другая задача"]
  ],
  "s": [
    {"n": "@Документ", "t": "Число целое"},
    {"n": "Дата",      "t": "Дата"},
    {"n": "Название",  "t": "Строка"}
  ]
}
```

Схема `s` — массив объектов `{"n": "ИмяПоля", "t": "ТипДанных"}`.

## Типы данных (`t`)

| Значение | Описание |
|---|---|
| `"Строка"` | UTF-8 строка |
| `"Число целое"` | integer |
| `"Число вещественное"` | float |
| `"Логическое"` | bool |
| `"Дата"` | `"YYYY-MM-DD"` |
| `"Дата и время"` | `"YYYY-MM-DD HH:MM:SS"` |
| `"UUID"` | GUID строка |
| `"JSON-объект"` | произвольный JSON |
| `"Выборка"` | вложенный recordset |
| `"Запись"` | вложенная запись |
| `{"n": "Массив", "t": "<тип>"}` | массив элементов указанного типа |
| `{"n": "Деньги", "p": 2}` | decimal с указанной точностью |

## Python-хелпер: recordset → list[dict]

```python
def sbis_rows_to_dicts(result: dict) -> list[dict]:
    schema = result.get("s") or []
    rows   = result.get("d") or []
    names  = [f["n"] for f in schema]
    if rows and isinstance(rows[0], list):
        # recordset: d = [[v, v, ...], ...]
        return [{names[i]: row[i] for i in range(min(len(names), len(row)))} for row in rows]
    # record: d = [v, v, ...]
    return [{names[i]: rows[i] for i in range(min(len(names), len(rows)))}]
```

## Фильтр и Навигация как record-объекты

При вызове списочных методов (`*.СписокПунктов`, `*.СписокПодзадач`) параметры `Фильтр` и `Навигация` тоже передаются в колоночном формате:

```json
{
  "Фильтр": {
    "d": [false, "По сотрудникам", null, 567678097, "2026-05-31", "2026-05-01", false, [], [], 1],
    "s": [
      {"t": "Логическое",  "n": "ВсеКромеВыполненных"},
      {"t": "Строка",      "n": "Группировка"},
      {"t": "Число целое", "n": "ИдОП"},
      {"t": "Число целое", "n": "ИдПлана"},
      {"t": "Дата",        "n": "ПланДатаКнц"},
      {"t": "Дата",        "n": "ПланДатаНач"},
      {"t": "Логическое",  "n": "Плитка"},
      {"t": "Строка",      "n": "РП.РазделИерархии"},
      {"t": "Строка",      "n": "СостояниеПунктов"},
      {"t": "Число целое", "n": "ФильтрПринадлежность"}
    ],
    "_type": "record",
    "f": 0
  },
  "Навигация": {
    "d": ["forward", true, 20, null],
    "s": [
      {"t": "Строка",      "n": "Direction"},
      {"t": "Логическое",  "n": "HasMore"},
      {"t": "Число целое", "n": "Limit"},
      {"t": "Строка",      "n": "Position"}
    ],
    "_type": "record",
    "f": 0
  }
}
```

Для следующей страницы передать `Position` из предыдущего ответа.
