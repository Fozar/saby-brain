# Wasaby BL CRUD Methods

Автогенерируемые методы создаются автоматически, если в объекте БЛ указана хотя бы одна таблица. Реализуют CRUD API. Нельзя создавать/удалять, но можно редактировать.

Транзакция по умолчанию: **Supported** (ведут себя как Required — сами создают транзакцию).

## Create (Создать)

Выполняет первичную инициализацию записи. **По умолчанию НЕ добавляет запись в БД** — только в память сервера.

### Основная сигнатура
| Параметр | Тип | Описание |
|---------|-----|---------|
| ИмяМетода | Строка | Имя декларативного метода; определяет формат записи |
| Фильтр | Объект | Поля, которые будут помещены в результирующую запись |

### Дополнительная сигнатура (аналог перегрузки)
| Параметр | Тип | Описание |
|---------|-----|---------|
| Расширение | Строка | `""` / имя объекта → по Object source tables; `"БазовоеРасширение"` → по Extended structure for editing |
| Фильтр | Объект | Поля для результирующей записи |

**Добавление в БД:** флаг "Insert record into a table on Create method" на объекте БЛ. При ручном вызове передать `'ВызовИзБраузера': true` для создания черновика.

## Read (Прочитать)

Читает запись по идентификатору.

### Параметры
1. **ИдО** — первичный ключ основной таблицы (целое положительное число)
2. **ИмяМетода** — декларативный или ручной списочный метод (определяет структуру ответа)
3. **Связь** — имя связи для перечитывания зависимостей

### Поведение
- `ИмяМетода = NULL` → структура по объекту БЛ, JOIN: INNER для object tables + LEFT для extension tables
- `ИмяМетода = "DeclarativeMethod"` → структура декларативного метода; в запись добавляется `Раздел$`
- Новая сигнатура: `Прочитать(ИдО, Расширение, Связь)` — перечитывает только данные, зависящие от изменённой связи

```
Прочитать( 100, "БазовоеРасширение", "Лицо1" );
```

## Update (Записать)

Создаёт новую или обновляет существующую запись. Вставка/обновление идёт по цепочке: основная таблица → расширяющие.

| Параметр | Тип | Описание |
|---------|-----|---------|
| Запись | Запись | Поля из структуры объекта БЛ |
| Проверка | Логический | `true` → проверить, не изменилась ли запись с момента прочтения |

- Если PK > 0 → `UPDATE`
- Если PK = NULL → `INSERT`

**Возвращает:** PK основной записи.

## Delete (Удалить)

Удаляет записи по массиву PK.

| Параметр | Тип | Описание |
|---------|-----|---------|
| ИдО | Массив целых | Идентификаторы удаляемых записей |

**Возвращает:** `true` всегда.

## Copy (Копировать)

Создаёт копию записи по идентификатору. Используется для клонирования объектов (например, документов) с новым PK.

## Merge (Объединить)

Объединяет несколько записей в одну (например, дублирующихся контрагентов). Одна запись становится основной, остальные перепривязываются и удаляются.

## DeleteSelected (Удалить отмеченные)

Массовое удаление записей, выбранных через интерфейс списка. В отличие от Delete, принимает не набор PK, а структуру выборки (фильтр).

## Sync (Синхронизировать)

> [!stale] Устаревший паттерн
> Метод изначально предназначался для пакетного сохранения (Create+Update+Delete за один вызов). От идеи отказались в пользу отдельных вызовов на каждое действие. Не рекомендуется к использованию.

## Общие опции методов

| Опция | Описание |
|-------|---------|
| Name | Уникальное имя в рамках объекта |
| Alias | Псевдоним метода |

## Обработчики автогенерируемых методов

Список обработчиков (задаются в Genie на вкладке Handler):

| Обработчик | Момент вызова | Метод |
|-----------|--------------|-------|
| Initialization | После Create | Изменить инициализируемый объект |
| Before Create | До Create или Update | Изменить запись до добавления в БД |
| After Create | После Create / до Update | Объект уже добавлен как черновик |
| Before Read | До Read | Получает ИдО, ИмяМетода, ИмяСвязи |
| After Read | После Read | Получает результирующую запись |
| Before Update | До Update | Изменить запись до записи в БД |
| After Update | После Update | Запись уже сохранена |
| Before Delete | До Delete | Получает массив ИдО |
| After Delete | После Delete | Записи уже удалены |
