# Markup Subsystem (Наценка)

Гибкая система надбавок к себестоимости или базовой цене товара. Укладывается в общую модель данных `ВидЦены` (Тип=32).

---

## Назначение

Ключевые задачи:
1. **Автоматизация** — система сама применяет нужную наценку по заданным правилам без ручного ввода.
2. **Гибкое ценообразование** — условия по времени, сумме, типу заказа, характеристикам клиента.
3. **Персонализация** — разные наценки для разных сегментов клиентов.
4. **Ручное применение** — кассир выбирает заранее заготовленную наценку через диалог скидок.

---

## Типы наценок

| Тип | Поведение |
|-----|-----------|
| **Автоматическая** | Срабатывает при добавлении первой номенклатуры в заказ |
| **Ручная** | Кассир добавляет вручную через диалог выбора скидок (аналогично ручной скидке) |

Если несколько наценок подходят под условия — применяется **максимальная**.

---

## Платформы применения

- Продажа в **Рознице**
- Заказ/Бронь в **Presto фронт** (онлайн/офлайн)
- Заказ на сайте **электронного меню**, странице заведения на **SabyGet**

Настройка доступна во всех розничных продуктах: Розница, Престо, Салон.

---

## Алгоритм расчёта

При любом изменении продажи Розница вызывает:

```
RetailSaleDoc.CalculateDiscount(WithMarkup=true)
  → возвращает Markup: Money (рассчитанная величина наценки)
```

Расчёт происходит **после** подарков, скидок, списания бонусов — **до** округления. Это соответствует общему порядку в [[Loyalty-Sale-Application]].

Если получена ненулевая величина → Розница добавляет/обновляет номенклатуру **Сервисный сбор** в заказе.

Наценки синхронизируются в десктопные приложения (Розница + Прesto) через существующий механизм лояльности.

---

## Специальная номенклатура «Сервисный сбор»

Автоматически создаётся при включении блока Сервисный сбор в настройках ТП.

| Параметр | Значение |
|----------|----------|
| Тип | Услуга |
| НДС | 20% (Россия), 12% (Казахстан) |
| Наименование | «Сервисный сбор» |
| Публикация | Опубликована |
| Место в заказе (гость) | Снизу (последней) |
| Место в заказе (сотрудник) | Сверху (первой) |

- Добавляется в количестве «1» при добавлении первой номенклатуры
- Пропадает если все товары удалены
- При переносе заказа на другой столик сумма пересчитывается
- Удаление заблокировано (аналогично номенклатуре доставки)

> [!key-insight] Связь с первой наценкой
> При включении блока автоматически создаётся первая наценка 5% с условием «на доставку не применяется».

## Информационная модель

### Markup (Наценка) — прикладной объект

| Системное имя | Локализация | Тип | Описание |
|---|---|---|---|
| `Name` | Название | String | Название наценки |
| `IsActive` | Используется | Boolean | Является ли наценка действующей |
| `MarkupConditions` | Условия | УсловияНаценки | При каких критериях добавляется наценка |
| `IsManual` | Вручную | Boolean | Назначается вручную кассиром |
| `Description` | Описание | String | Автогенерируемое описание |
| `MarkupProfit` | ВыгодаНаценки | ВыгодаНаценки | Величина, номенклатуры, тип применения |

### MarkupConditions (УсловияНаценки) — служебный тип

| Системное имя | Тип | Описание |
|---|---|---|
| `NomenclatureCount` | Decimal | Наценка действует при покупке N+ товаров |
| `SalesSum` | Decimal | Наценка действует при покупке на сумму N+ |

### MarkupProfit (ВыгодаНаценки) — служебный тип

| Системное имя | Тип | Описание |
|---|---|---|
| `Nomenclatures` | Номенклатура[] | Список номенклатур, на которые действует наценка |
| `Value` | Decimal | Размер наценки |
| `ExcludedNomenclatures` | Номенклатура[] | Список исключённых номенклатур |
| `ApplyType` | Enum | 0 = на номенклатуры из списка; 1 = на все, кроме исключённых |

## База данных

Наценка хранится в `ВидЦены` с **Тип=32**.

| Поле | Тип | Описание |
|------|-----|----------|
| @ВидЦены | PK | Идентификатор наценки |
| Используется | Логическое | Активность наценки |
| Название | Текст | Название наценки |
| Тип | Перечисляемое | **32** = Наценка |

> [!note] Единая модель данных
> Наценка технически реализована как скидка с типом `MARKUP_*`. Используется та же таблица `ВидЦены`, что и для [[Акции-Subsystem-Overview|акций]], скидок и бонусных программ. Модуль: `PriceFormation.Online` / `PriceFormation.Offline`.

---

## API

### PriceFormation.Online — Markup

```
PriceFormation.Online.Markup.Create(settings)
```
Создать наценку с переданными настройками (Online-вариант).

### PriceFormation.Offline — Markup

```
PriceFormation.Offline.Markup.Create(settings)
```
Создать наценку с переданными настройками (Offline-вариант).

### RetailSaleDoc.CalculateDiscount

Используется при расчёте программ лояльности в рознице. При флаге `WithMarkup`:
- Параметр `Markup: Money` — возвращает рассчитанную величину наценки.
