# Хранилище дистрибутивов — Схема БД

Схема базы данных сервиса `distribution storage` (Хранилище дистрибутивов).

## Ключевые сущности

| Сущность | Описание |
|----------|----------|
| `Продукт` | Сущность дистрибутива для объединения версий одного приложения; название обычно = имя приложения |
| `ВерсияПродукта` | Одна сборка дистрибутива (конкретный экземпляр) |
| `УстановленнаяВерсия` | Версии, защищённые от удаления (текущие в использовании) |
| `Сервис` | Соответствует `.ssrv`, собранному под определённую архитектуру |
| `Пропатченные сервисы` | Сервисы, к которым применён патч в рамках операции «Применение патча» |
| `ФайлВерсии` | Общие файлы дистрибутива, не относящиеся к сервисам |
| `Скрипт` | Архив со скриптами (логически не входит в дистрибутив, но привязан к версии) |
| `Патч` | Архив с патчами (привязан к конкретной сборке через `ВерсияПродукта`) |
| `ФайлХранилища` | Физический файл — содержит ссылку на объект в filestorage, md5, mime, размер, даты. Расширяется таблицами: `ФайлВерсии`, `Скрипт`, `Патч`, `ФайлОбновления`, `Module`, `ServicePart` |
| `Модуль` | Модуль дистрибутива. Тип: `бизнес-логика` / `интерфейс`. Соответствует `.smod` |
| `ВерсияМодуля` | Версия модуля — зависит от модуля + характеристик сборки (версия платформы, алгоритм сборки) |
| `Связь сервиса с модулями` | Таблица связи сервиса с его модулями + версия модуля для сервиса |
| `ЧастьСервиса` | Устаревшая реализация `Модуль` |
| `Образ` | Docker-образы и Helm-чарты для K8s. Может быть связан с сервисом и версией дистрибутива |
| `ФайлОбновления` | Файлы, создаваемые во время обновлений (логи) |
| `СервисХоттабыча` | Сервис «Управление версиями», хранящий файлы обновлений; разграничивает доступ к файлам разных стендов |
| `Скриншоты дистрибутива` | Скриншоты мобильных дистрибутивов |
| `Доступ клиентов к дистрибутивам` | Связь ID клиентов автономных облаков с дистрибутивами saby-пакетов |
| `Состав saby-пакета` | Из каких дистрибутивов состоит saby-пакет |

## Алгоритмы ключевых операций

### Загрузка файла (deduplication по md5)

Клиент вычисляет md5 от файла → запрашивает наличие по md5 (индекс `ФайлХранилища(md5)`). Если файл уже есть — создаётся только новая сущность-ссылка без повторной загрузки. Иначе — создаётся новая `ФайлХранилища`.

### Удаление файла (ref-counting)

Так как `ФайлВерсии`, `Module`, `Скрипт`, `Патч`, `ФайлОбновления` могут ссылаться на одну `ФайлХранилища`: при удалении ссылающейся сущности подсчитываются ссылки. При нуле — `ФайлХранилища` удаляется и делается запрос в filestorage.

### Поиск модуля по ревизии

Хранилище гарантирует уникальность тройки `(имя модуля, тип, ревизия)` → однозначно определяет содержимое. Ревизия хранится в `ModuleVersion.VersionHash`. Связь через `ServiceToModule`.

## Типовые выборки

| Выборка | Алгоритм |
|---------|----------|
| Информация о версии | Доступ по PK к `ВерсияПродукта` |
| Список версий | По FK `Продукт` с сортировкой по `Дата, Название, Сборка` |
| Дерево файлов версии | Список из `ФайлВерсии`, `Module`, `Скрипт`, `Патч` по FK `ВерсияПродукта` → объединить |
| Загрузка файла | md5 → `ФайлХранилища(md5)` → если нет, создать |
| Поиск использования файла | Ссылки на `ФайлХранилища` в зависимых таблицах перед удалением |
| Поиск модуля по ревизии | `Module(Name, Type, Revision)` — составной уникальный ключ |
| Файлы конкретного обновления | `ФайлОбновления(СервисХоттабыча, ИдОбновления)` |
| Поиск образа по ссылке | `Image(Url)` + `Image(Host)` |
| Скриншоты дистрибутива | `DistrScreenshot(DistrId)` → все записи по `DistrId` |
| Синхронизация образов репозитория | `Image` → фильтр по базовой части URL |
| Использование версии | `УстановленнаяВерсия(ВерсияПродукта)` |
| Образы версии | `ImageToVersion(DistrVersionId)` |
| Зависимости saby-пакета | `SabyPacketDependencies(DistrVersionId)` |
| Сервисы версии | `Service(DistrVersionId)` |
