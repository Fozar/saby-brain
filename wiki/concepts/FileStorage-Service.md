# FileStorage (хранилище файлов)

Подсистема SabyDisk для **долгосрочного хранения внутренних файлов** сервисов. Доступна только внутри облака. Использует резервирование, репликацию и градацию хранилищ hot/cold.

## Ключевые особенности

- Только внутри облака (нет прямого доступа извне без специального сервиса)
- Единый REST API независимо от физического хранилища
- Глобально уникальные идентификаторы файлов
- Не дублирует бинарные данные в разных подсистемах

## Архитектура

### Сервисы БЛ

| Сервис | Роль |
|---|---|
| `filestoragerest` | REST API изнутри облака |
| `fs-auth` | REST API извне (с авторизацией) |
| `fs-public` | REST API извне (без авторизации) |
| `file-storage-utility` | Фоновые задачи: перенос из hot-cache, перепаковка блоков |
| `Хранилище файлов` | Служебное API для utility; очистка временных данных |

### Системы хранения

| Хранилище | Применение |
|---|---|
| SWIFT / CEPH | Постоянные объектные хранилища |
| СБИС Архив Storage System | Внешние хранилища за пределами ЦОД |
| "Горячий" кэш (СУБД) | Временное хранение новых файлов (<5 МБ, быстрый доступ) |

### Базы данных

| БД | Тег | Содержимое |
|---|---|---|
| Хранилище файлов | `filestorage_N` | Основная (File, Metadata, Index); шардирована |
| file-storage-utility | — | Неймспейсы, объектные хранилища, MIME-типы |
| Кэш | `filestorage_cache_X` | Горячий кэш бинарных данных |
| Бэкап кэша | `filestorage_backup_X` | Резервная копия кэша |

## Структура хранения файла

Файл = **бинарные данные** + **метаданные** (имя, размер, MD5, тип) + **индекс** (физическое местоположение).

Три варианта хранения метаданных:
1. Не утилизированы → `Metadata` таблица
2. Утилизированы вместе с бинарными данными → флаг в `File`, нет в `Metadata`
3. Утилизированы отдельно → `MetadataArchive` таблица

## Жизненный цикл файла

```
Загрузка → "горячий" кэш (если <5 МБ) → фоновый перенос в SWIFT/CEPH
                                          ↓
                              Упаковка метаданных в блоки (~5 МБ)
                              Блок: версия, кол-во, список ID, MD5
```

Утилизация метаданных запускается ночью для файлов старше 2 лет.

## Блок метаданных в объектном хранилище

Группировка по: день LastModified + неймспейс + (опционально) ID клиента.
Поля записи: `FileId, Name, Size, ContentType, MD5, ...`

## GET-кэширование SWIFT

Nginx/Varnish прокси перед SWIFT:
- HIT → из кэша (локальный диск)
- MISS → из SWIFT + кэширование
