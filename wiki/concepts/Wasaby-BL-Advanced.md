# Wasaby BL Advanced Features

Расширенные возможности методов BЛ: кэширование, таймауты, защита от частых вызовов, область видимости, обработчики, произвольные методы, удалённые вызовы, работа с файлами.

## Caching (Кэширование методов БЛ)

Кэширование результатов методов БЛ для повторяющихся вызовов. Настраивается в Genie на вкладке метода.

Ключевые параметры:
- **Cache TTL** — время жизни кэша
- **Cache key** — поля, определяющие ключ кэша
- **Cache invalidation** — условия сброса кэша

Кэш хранится в Redis. Используется для тяжёлых и часто вызываемых списочных методов.

> [!note] Ограничение метода-кэширования для отчётов
> Кэширование методов сохраняет только **одну страницу**. При навигации к следующей странице снова требуется полный вызов. Для больших отчётов используйте [[Report-Prefetch-Service]] — платформенный сервис `report-prefetch-service`, который кэширует сразу `PrefetchPages` страниц.

## Timeout (Таймаут)

Два вида таймаутов, настраиваются в Genie:

### Call timeout
Предельное время ожидания ответа от сервиса-исполнителя. Отсчёт — с момента отправки запроса.

- По умолчанию: 5 минут (0 = default)
- Значение кратно 100 мс (точность сетевых таймаутов)
- При превышении — запрос удаляется из очереди, ответ: `502 Sending denial of service due to expiration of the query relevance`

**Переопределение для конкретного вызова:**
```python
# Python
sbis.EndPoint.SetTimeout(1000)  # 1000 мс

# C++
sbis::blcore::EndPoint::SetTimeout(1000);
```

**Наследование:** если метод A вызывает метод B из другого сервиса — приоритет у таймаута вызывающей стороны (A), затем собственные настройки B.

### Execution timeout
Предельное время выполнения метода в рабочем процессе сервиса-исполнителя.

## Frequent Call Protection (Antibot)

Защита от роботов/скриптов. Реализована через модуль Antibot как триггер на метод.

При превышении порога: исключение `429 Too many request` + CAPTCHA в интерфейсе.

### Настройка в Genie (флаг Frequent call protection)

| Поле | Описание |
|------|---------|
| Max calls count | Максимальное число вызовов за интервал |
| Check interval | Интервал в секундах |
| Lock time | Время блокировки в секундах |

**Пример:** `Max=10, Interval=60, Lock=1200` → max 10 вызовов/минуту, блокировка 20 минут. Redis-ключ живёт `Interval * 2 = 120с`.

### Контекст блокировки
- С аутентификацией → по `UserId`
- Без аутентификации → по `IP`

### Глобальные параметры сервиса
```ini
Безопасность.Антибот.Период = 60          ; секунды
Безопасность.Антибот.МаксимальноеЧислоВызовов = 10
Безопасность.Антибот.ВремяБлокировки = 1200
```

## Access Area (Область видимости)

Метод, который **дополнительно фильтрует** результаты другого метода. Используется для ограничения прав доступа к данным.

**Пример:** запретить менеджеру видеть сделки свыше 100 000 руб.

Создаётся в Genie как отдельный метод. Рекомендуемая доступность: `Internal`.

| Атрибут | Описание |
|--------|---------|
| Name | CamelCase, уникальный в объекте |
| Display name | Имя в UI приложения |
| Display name for list | Название ограничения для списков |
| Display name for adding | Для окна добавления |
| Accessibility | Доступность (рекомендуется Internal) |
| Deprecated | Пометка устаревшего метода + причина |

## Handlers (Обработчики)

Обработчик — особая разновидность метода, выполняемая автоматически при вызове метода БЛ. Нельзя вызвать явно с клиента.

**Зачем:** расширить базовый алгоритм CRUD/списочных/файловых методов без переписывания.

Реализация: Python или C++ (независимо от реализации самого метода).

### Параметры обработчика (вкладка Handler в Genie)

| Параметр | Описание |
|---------|---------|
| Name | Уникальное имя в рамках метода; для C++ — лексикографический порядок выполнения |
| Implementation type | Python или Native function (C++) |
| Order type | Порядок относительно унаследованных: Before parent / After parent / Replace parent |
| Handler type | Момент: до или после основного метода |

### Обработчики автогенерируемых методов

| Обработчик | Триггер | Ключевые аргументы |
|-----------|--------|-------------------|
| Initialization | После Create | Запись, Фильтр, ИмяМетода |
| Before Create | До Create / Update | Запись (изменяемая), Фильтр |
| After Create | После Create / до Update | Запись (уже в БД как черновик) |
| Before Read | До Read | ИдО, ИмяМетода, ИмяСвязи |
| After Read | После Read | Результирующая запись |
| Before Update | До Update | Запись до сохранения |
| After Update | После Update | — |
| Before Delete | До Delete | Массив ИдО |
| After Delete | После Delete | — |

> [!note] Deprecated: Параметры
> Аргумент "Параметры" устарел — всегда приходит пустой объект. Оставлен для обратной совместимости.

## Custom Method (Произвольный метод)

Метод с произвольной сигнатурой: любое число параметров, возвращает список/запись/скаляр/void.

Используется, когда стандартные типы не подходят.

**Реализация:**
- Python / C++ (Native function) / SQL Request
- Модули: BL Core (C++), BL Python (Python), Бизнес-логика (SQL)

### SQL Request
```sql
-- Именованные параметры
INSERT INTO "ТестФормата"("UUID") VALUES(:UUID)

-- Нумерованные параметры
INSERT INTO "ТестФормата"("UUID", "Целое2") VALUES($1, $2)
```
Параметры типа `Record` передаются как `hstore`. Синтаксисы взаимоисключающие.

### Работа с конфиденциальными данными

```python
# Запретить логирование результата текущего вызова
sbis.ConfidentialOutput.Enforce()

# Запретить логирование в области вложенных вызовов
with sbis.ConfidentialScope():
    sbis.EndPoint.Call("SecretMethod", ...)
```

| Ситуация | Решение |
|---------|--------|
| Конфид. данные в параметрах | Флаг "Don't log method input arguments" в Genie |
| Конфид. данные в результате | Флаг "Don't log result" + `ConfidentialOutput.Enforce()` |
| Конфид. передаётся во вложенные вызовы | `ConfidentialScope()` |

## Remote Calls (Удалённые вызовы)

Методы для вызова методов на удалённом сервисе.

### Proxy метод
Проксирует вызов на другой сервис. Настраивается в Genie.

### HTTP запрос
Позволяет сделать произвольный HTTP-запрос к внешнему сервису (REST API). Не требует Python-кода — конфигурируется в Genie.

## File Methods (Работа с файлами)

Для объектов БЛ со структурой из таблиц, содержащих поля типа "Файл", "Бинарные данные" или XML.

| Метод | Назначение |
|-------|-----------|
| Upload file method | Загрузить файл в БД |
| Upload and attach file method | Загрузить и прикрепить к объекту БЛ |
| File UUID list method | Получить список UUID файлов |
| Read file method | Прочитать файл из БД |

Методы "привязываются" к конкретному полю в структуре объекта.
