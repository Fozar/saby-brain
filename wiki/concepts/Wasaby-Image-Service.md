# Wasaby Image Service (img-remote)

Stateful-сервис для ресурсоёмких операций обработки изображений, вынесенный из локального сервиса без переработки кода.

> Сервис не имеет публичного API. Взаимодействие только через платформенный модуль `Image-BL`.

## Ключевые особенности

- **Stateful**: сессия привязана к одному внешнему BL-вызову; при создании первого изображения выбирается BL-сервис, все последующие операции вызова идут на тот же узел
- **Изоляция**: изображения из разных внешних вызовов не взаимодействуют
- **Backend**: ImageMagick (чтение/запись/редактирование)
- **No DB**: данные не персистируются между сессиями

## Использование

Использовать через модуль `Image-BL` (клиентская библиотека). Прямые вызовы на сервис запрещены.

```python
# Пример через модуль Image (C++ или Python)
# Создание/загрузка первого изображения автоматически открывает сессию на удалённом сервисе
```

Подробнее: статья «Сервисный модуль Image».

## Архитектура

```
Клиент (BL метод)
  → Image-BL (клиентская библиотека)
    → img-remote сервис (выбирается при первом обращении в рамках вызова)
      → ImageMagick
```
