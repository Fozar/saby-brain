# Wasaby BL Methods

Метод БЛ — функция, через которую осуществляется взаимодействие с данными БД или другими сущностями. Все методы объявляются в [[Wasaby-BL-Objects|объектах БЛ]].

## Реализации

- **Python** — основной язык прикладной разработки
- **Встроенная функция (C++/Native function)** — сервисный модуль BL Core
- **SQL-запрос** — сервисный модуль "Бизнес-логика"

## Таксономия методов

```
Методы бизнес-логики
├── Автогенерируемые (CRUD)          → [[Wasaby-BL-CRUD]]
│   ├── Create (Создать)
│   ├── Read (Прочитать)
│   ├── Update (Записать)
│   ├── Delete (Удалить)
│   ├── Copy (Копировать)
│   ├── Merge (Объединить)
│   ├── DeleteSelected (Удалить отмеченные)
│   └── Sync
├── Списочные                         → [[Wasaby-BL-List-Methods]]
│   ├── Декларативный (Declarative list)
│   └── Реализуемый вручную (Manually implemented)
├── Работа с файлами                  → [[Wasaby-BL-Advanced]]
│   ├── Read file method
│   ├── Upload file method
│   ├── Upload and attach file method
│   └── File UUID list method
├── Удалённые вызовы                  → [[Wasaby-BL-Advanced]]
│   ├── Proxy метод
│   └── HTTP запрос
├── Произвольный (Custom method)      → [[Wasaby-BL-Advanced]]
├── Область видимости (Access area)   → [[Wasaby-BL-Advanced]]
└── Обработчики (Handlers)           → [[Wasaby-BL-Advanced]]
```

## Стандартный контракт

Чтобы UI-компоненты работали с методами без адаптера, API метода должен соответствовать [контракту](https://link.sbis.ru/article/df7e6caf-0f3c-4d7a-8570-8377dbbdf27d). Для произвольного контракта нужен компонент-адаптер.

## Доступность (Accessibility)

| Уровень | Описание |
|---------|---------|
| Service Contract | Публичное API сервиса. Видно снаружи. |
| Internal Application Contract | Только между сервисами одного приложения |
| Internal | Только внутри сервиса |

> [!note] Конфликт имён
> Если у объекта одновременно есть списочный метод (4 параметра) с доступностью Service Contract и одноимённый метод с Internal + 1 параметр — в логах будут ошибки при вызове списочного. Это историческая особенность: списочные методы регистрируются дважды (с 4 и 1 параметром).

## Наследование и перекрытие

- Методы наследуются по иерархии объектов БЛ
- При отсутствии реализации у объекта — вызывается реализация предка
- Перекрытие по имени: если Модуль2 загружается после Модуль1 и содержит тот же `Объект.Метод` — используется Модуль2
- Автогенерируемые методы имеют приоритет над Custom при одинаковых именах
- Автогенерируемые методы сами не наследуются

## Транзакции

По умолчанию для автогенерируемых методов тип транзакции — **Supported** (ведут себя как **Required** — сами создают транзакцию).

## Исключение из проверки прав

Методы, доступные до аутентификации или без проверки прав, добавляются в исключения:
- Genie: флаг "Is always permitted" + причина
- `AllowMethods.txt` (лёгкая система прав)
- `exclude.rights` (полная система прав)
