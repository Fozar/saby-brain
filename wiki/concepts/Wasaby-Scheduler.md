# Wasaby Scheduler

Сервис выполнения задач по расписанию. Доступен в **Управление облаком → Настройки → Планировщик заданий**.

## Концепция

Задача = вызов метода БЛ (`ОбъектБЛ.Метод`) с параметрами на конкретном сервисе, по расписанию.

Задачи иерархически: семейство приложения → функциональный участок → конкретные задачи.

> [!note] Для однократных задач — используйте DWC (Сервис выполнения распределенных асинхронных бизнес-сценариев), не Scheduler.

## Создание задачи

"Штатное" создание — через **Genie** (genie.saby.ru), с использованием справочника задач.

Вручную — в Управлении облаком (Insert / кнопка "+ Задача").

### Обязательные параметры

| Параметр | Описание |
|----------|----------|
| **Название** | Краткое обозначение для поиска |
| **Описание** | Подробное описание (для других разработчиков) |
| **Сервис-исполнитель** | Имя сервиса, "Для всех БД", или "Для всех групп серверов" |
| **Метод** | `ОбъектБЛ.ИмяМетода` (формат JSON-RPC) |
| **Параметры** | JSON-параметры метода |
| **Тип запуска** | Расписание (см. ниже) |

### Типы расписания

- **Фиксированный интервал** — каждые N секунд
- **Ежедневно** — каждый день / каждые N дней в заданное время
- **Еженедельно** — по дням недели
- **Ежемесячно** — по числам месяца
- **Ежегодно** — по месяцам года

### Дополнительные параметры

| Параметр | Описание |
|----------|----------|
| **Критичная** | Ошибки = предаварийная ситуация для эксплуатации |
| **Последовательное исполнение** | Следующий запуск не стартует, пока не завершился предыдущий |
| **Итеративная задача** | Метод возвращает `{'Result': bool, 'Description': str}`; повторяется пока Result=false |
| **Интервал повторов** | Таймаут игнорирования "Последовательного исполнения" (default: 12 часов) |
| **Ожидаемое время** | Предупреждение в журнал если задача выполняется дольше (в минутах) |
| **Для каждой БД** | Выполнить для каждой базы данных сервиса (нужен модуль "Multiple databases") |
| **Для каждой схемы** | Выполнить для каждой схемы каждой БД |

## Приоритет параметров метода

При обновлении сервиса параметры из дистрибутива **не перезаписывают** параметры, введённые вручную.

Порядок приоритетов (от высшего к низшему):
1. Параметры для сервиса-исполнителя (введены вручную)
2. Параметры задачи (введены вручную)
3. Параметры из дистрибутива

Чтобы вернуть параметры дистрибутива — нужно создать ВНР: приостановить задачу → сбросить ручные параметры → восстановить после обновления.

## Управление задачами

- **Приостановка** — статус "приостановлена" в карточке (текущее выполнение не прерывается)
- **Ручной запуск** — кнопка "Запустить задачу вручную" (нужны права администратора облака)
- **История выполнения** — в карточке задачи
- **Логи** — в разделе просмотра логов

## Роль и права

Просмотр и создание задач — только роль **"Администратор облака"**.
