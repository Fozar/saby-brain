# Экспорт планов выпуска в изолированные облака

Процесс передачи планов выпуска в изолированные (air-gapped) облака — например, Туркменистан (TM) и аналоги.

## Общий процесс

Релиз-менеджеру приходит поручение на выгрузку и публикацию плана в экспортируемое облако.

**Шаги:**
1. Проверить: существует ли нужный дистрибутив (Раздел «Дистрибутивы» → фильтр «Планы выпуска»)
2. Создать дистрибутив (если не существует)
3. Первоначальное создание экспортного плана
4. Редактирование (если нужно добавить работы)
5. Выгрузка плана в хранилище
6. Закрыть поручение с указанием успешно выгруженного билда

## Создание дистрибутива для экспортируемого плана

1. Перейти на вкладку «Дистрибутивы» в Хоттабыче → фильтр «Планы выпуска»
2. По кнопке «+» заполнить название (согласовывается с ответственным за передачу)
3. Открыть созданный дистрибутив → вкладка «Аккаунты»
4. Добавить аккаунты (их сообщат исполнители работ); доступен массовый выбор и удаление

## Первоначальное создание экспортного плана

1. Ответственный РМ в реестре «Работы» отбирает работы по фильтрам: **дата + saby-пакет** + «Показывать выполненные работы» + «Завершена»
2. Отобрать все работы через ПМО («Отметить все»)
3. Нажать кнопку «Создать экспортируемый план»
4. Заполнить поля:
   - **Название** — по согласованию с заказчиком
   - **Версия** — не заполняется вручную, заполнится автоматически (`{YYMMDD} - {N} от {Дата}`)
   - **Ответственный** — руководитель отдела Релиз-менеджмента
   - **Дата** — предполагаемая дата выгрузки
   - **Дистрибутив** — ранее созданный или новый

**Структура экспортного плана (древовидная):**
```
- Дата в prod-ru облаке (Этап)
    - Тип работы (Этап)
        - Тип обновления (если есть) (Этап)
            - Работа (Пункт плана или этапы с пунктами планов)
```

После создания план появляется в корне Планов выпуска в папке «Экспортируемые».

## Редактирование экспортного плана

Если нужно добавить работы в уже созданный план:

1. В реестре отобрать нужные работы по фильтрам
2. Выделить через ПМО
3. Нажать кнопку «Добавить в экспортируемый план»
4. Выбрать нужный план из списка → «Добавить»

> [!tip] Без дублей
> Уже добавленные работы в экспортируемый план не дублируются.

> [!warning] После правок
> При внесении правок в уже выгруженный план — повторить этап «Выгрузка» чтобы в дистрибутив попали все изменения.

## Выгрузка экспортного плана

1. Релиз-менеджер нажимает кнопку выгрузки в хранилище
2. Проверить дистрибутив и автоматически сформированную версию (`{YYMMDD} - {N} от {Дата}`)
3. Нажать «Да»
4. Сообщить заказчику (ЦОД) данные плана для передачи на сторону выполнения

> [!warning] Ошибки публикации
> При ошибках — сначала перезапуск выгрузки. Если повторяется — подключать Волосатова Л.

## Скачивание и восстановление плана в изолированном облаке

Принимающая сторона:
1. Из реестра планов выпуска нажать кнопку загрузки плана
2. Выбрать доступный дистрибутив и версию
3. В процессе импорта запускаются длительные операции: загрузка и распаковка плана, загрузка из публичного в локальное хранилище дистрибутивов всех версий и образов

Загруженный план — полноценный план, готов к запуску. Эксплуатация изолированного облака может его отредактировать и добавить специфику запуска.

> [!tip] Пользовательские работы
> Для пользовательских работ доступна кнопка PDF — скачать печатную форму с описанием инструкции выполнения работы.
