# Wasaby Service Framework Overview

Сервисный фреймворк Wasaby — архитектурный слой между клиентом и базой данных, реализующий бизнес-логику в виде объектов и методов.

## Слои архитектуры

```
Клиент (JS/Web)
      ↓  HTTP / WebSocket
Сервер приложений (Application Server)
      ↓  внутренний вызов
БЛ-слой (BL Object → Method)
      ↓  SQL / ORM
База данных (PostgreSQL / MSSQL)
```

## Сервис-контракт

Сервис-контракт определяет публичный API сервиса: набор БЛ-объектов и методов, доступных извне. Контракт версионируется и служит границей между командами.

- Каждый сервис публикует свой контракт
- Внешние вызовы идут только через задекларированные точки входа
- Внутри сервиса объекты могут вызывать друг друга напрямую

## Сервер приложений

Application Server — процесс, принимающий запросы и маршрутизирующий их к нужному БЛ-объекту/методу.

- Обслуживает синхронные HTTP-запросы от клиентов
- Принимает сообщения из очереди (AMQP/RBC) для асинхронных вызовов
- Управляет пулами потоков: `rpNORMAL` (основной) и `rpLOW` (сервисный)

## Межсервисное взаимодействие

Сервисы общаются через `EndPoint`. Подробнее: [[Wasaby-BL-Calls]].

```python
# Вызов метода в другом сервисе
EndPoint('accounting-service').Document.Get(filter)
```

## Ключевые компоненты

| Компонент | Роль |
|-----------|------|
| `BLObject` | Единица бизнес-логики; иерархия наследования |
| `EndPoint` | Точка входа в удалённый сервис |
| `Method` | Атомарная операция над объектом |
| `AsyncInvoke` | Асинхронный вызов через очередь |
| `StringBuilder` | Построение запросов без аллокаций |
