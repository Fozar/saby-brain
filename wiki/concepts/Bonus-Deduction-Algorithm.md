# Bonus Deduction Algorithm

Алгоритм расчёта списания бонусов. Работает через ядро расчёта скидок (`CalcDiscount.Run`).

---

## Интеграция с ядром расчёта

`BONUS_DEC_RULE = 40`

В параметр `AvailablePriceEntities` метода `CalcDiscount.Run` поступает запись с `Type = BONUS_DEC_RULE`. В результате метод возвращает: сколько бонусов списано по каждому правилу (только по правилам с списанием > 0).

Каждый объект ВидЦены с типом `BONUS_DEC_RULE`:
- Фильтруется по своему списку "Где" + глобальному списку "Действует"
- Дополняется запрещёнными номенклатурами из глобальных настроек

---

## Алгоритм расчёта ПЛАНА списания

Все правила списания делятся на 3 группы по приоритету (1 = наивысший):

1. **Со списком акционных номенклатур** (таргетированные)
2. **Без списка** ("на весь чек", включая "на всё кроме исключённых")
3. **Глобальные настройки списания**

Для каждой группы, начиная с приоритетной:
1. Если правило не подходит → пропуск.
2. Если подходит:
   - Вычислить процент: либо фиксированный, либо по порогам.
   - Если пороги заданы и ни один не достигнут → правило пропускается.
   - Назначить процент позиции, если:
     - У позиции ещё нет назначенного процента, **ИЛИ**
     - Правило из текущей группы (той же части) и новый % **больше** прежнего.

> [!key-insight] Правила с 0% как блокировщики
> Правило списания с 0% **блокирует** менее приоритетные правила — те не смогут назначить свой процент этой позиции. Используется для явного запрета списания на отдельные товары.

---

## Алгоритм вычисления ФАКТА списания

По каждому правилу, вошедшему в план списания:

1. Вычислить фактическую сумму позиций с учётом применённых скидок.
2. Вычислить максимально возможное списание (с учётом МРЦ).
3. Вычислить списываемые бонусы = `min(% правила × фактическая сумма, максимум, баланс бонусов, ручной ввод)`.
4. Распределить между бонусными балансами.
5. Распределить по позициям (с фиксацией: с какого баланса сколько).

## Калькулятор бонусных программ

Инструмент для расчёта параметров БП перед созданием. Результат: создание/обновление БП.

### Фиксированная программа

| Параметр | Формула / По умолчанию |
|----------|----------------------|
| Кэшбэк, % | `⌊(Расход на кэшбэк% × Прибыль с чека ÷ Ср.чек) × 100⌋` |
| Лимит опл.чека, % | `⌈(Прибыль с чека × 75% ÷ Ср.чек × 100)⌉`, кратно 5 |
| Рост Ср.чека с БП, ₽ | `Ср.чек × (100 + Прогноз роста%) ÷ 100` |
| Прибыль, ₽ | `Кол-во чеков × (Рост Ср.чека − Ср.чек)` |
| Расход на кэшбэк | 10% (по умолчанию, вручную) |
| Прогноз роста | 20% (по умолчанию, вручную) |

**Приветственный бонус** зависит от типа заведения:
- Ресторан → 250 ₽
- Кафе → 200 ₽
- Пекарня / Столовая / Кофейня → 150 ₽

### Уровневая программа (до 5 уровней)

**Кэшбэк N-го уровня**:
```
((Кэшбэк_макс - Кэшбэк_1) ÷ (N - 1)) × (У - 1) + Кэшбэк_1
```

**Порог U-го уровня**:
```
Порог_1 = 0
Порог_U = Ср.чек × Эталонное_посещений_в_месяц × x × (У - 1)
```

Значение `x` по типу заведения:
- Ресторан → x = 4 месяца
- Кафе → x = 8 месяцев
- Пекарня / Кофейня / Столовая → x = 12 месяцев

**Лимит оплаты чека** — та же формула, что и в фиксированной программе.

### Действие кнопки "Применить"

1. Сохраняет лимит оплаты чека.
2. Создаёт / обновляет программу приветственных бонусов.
3. Открывает панель создания БП с рассчитанными данными.
