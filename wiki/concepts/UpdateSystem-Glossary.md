# Термины и определения — Реестр работ / Планы выпуска

Глоссарий для подсистем «Реестр работ» и «Планы выпуска» в Хоттабыче.

## Основные термины

| Термин | Определение |
|--------|-------------|
| **План выпуска** | Перечень работ на определённый день, упорядоченных в соответствии с внутренними регламентами компании |
| **Реестр работ** | Перечень работ за определённый промежуток времени (указанный в фильтрах реестра), упорядоченный по типу работы и разложенный по папкам с датами/версиями для выполнения работ на бою |
| **ВНР (Выполнить на рабочем)** | Регламент по выполнению работ на рабочем сервере |
| **ОС (Обновление сервиса)** | Регламент работ по обновлению на рабочем сервере |
| **Волна обновления** | Мажорное обновление приложений СБИС; существуют три типа волн |
| **Нулевая волна (0-волна)** | Облако и его прочие сервисы, middleware |
| **Первая волна (1-волна)** | online-master, сигнальные юниты онлайна, прочие сервисы; распределение по дням — пятница или суббота |
| **Вторая волна (2-волна)** | online-master + все online + прочие сервисы; все прочие сервисы кроме спецификаций обновляются в пятницу |
| **Предыдущая версия основных сервисов** | Версия, установленная на бою до обновления на первую волну |
| **Сервис плана выпуска** | Сервис, хранящий описания планов выпуска, запускающий планы и содержащиеся в них работы в нужных облаках, а также предоставляющий API для взаимодействия с ним из других Хоттабычей и из инструментов сборки |
| **Работа** | Единица — интерпретация документа с онлайна для Хоттабыча, которую необходимо выполнить. Может быть обновлением сервиса, выполнением скрипта, конфигурированием параметров, настройкой сервера и т.п. Все работы находятся в Реестре работ |
| **Пункт плана** | Ссылка/ярлык на работу из Реестра работ, который отображается в Планах выпуска |
| **Этап** | Пункт плана, который имеет общие свойства с работой, может быть исполнен и имеет статусы. Этапы могут быть вложенными. Пример: один день выпуска (смена дежурства эксплуатации) |
| **Ограничение запуска пункта плана** | Условие для запуска пункта плана и всех вложенных в него пунктов. Может быть: зависимости, время запуска, количество одновременно запущенных задач, интервал запуска |
| **Задача** | Экземпляр запуска работы на Хоттабыче одного из облаков системы. Набор задач работы представляет собой историю запуска работы на разных стендах |
| **Приоритет работы** | Очерёдность запуска работы при наступлении момента одновременной готовности пуска нескольких работ |
