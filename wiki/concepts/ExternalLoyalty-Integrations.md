# External Loyalty Integrations

Подсистема интеграции СБИС с внешними бонусными системами: **UDS**, **Premium Bonus (PB)** и **iikoCard**.

Доступно в продуктах розничной линейки: Presto Online/Desktop, Розница Online/Desktop, Киоск самообслуживания.

---

## Термины

| Термин | Описание |
|--------|----------|
| Тех. Скидка | Скидка без карточки и редактирования; отображается в реестре скидок |
| Тех. Бонусы | Бонусная программа без карточки; отображается в реестре начислений |
| Тех. Вид карты | Вид карты без карточки; эмиссия для карт участников внешней ЛС |
| Вебхук | Уведомление о событии (новый клиент) с UDS на внешний веб-сервер |

---

## Внешние системы

### UDS

Бонусная программа лояльности с **реферальной системой** — пользователи рекомендуют заведения друзьям и получают бонусы от их покупок.

**Клиент UDS**: ФИО, пол, дата рождения, количество доступных баллов.

**Алгоритм работы:**
1. Организация регистрируется в UDS, настраивает скидки/бонусы
2. В СБИС настраивает подключение и вебхуки о новых клиентах
3. Клиент устанавливает приложение UDS и накапливает баллы
4. При оплате клиент находит организацию в UDS и использует накопленные баллы

**Процесс на кассе:**
1. Клиент показывает шестизначный код из приложения UDS
2. Кассир вводит код → отображается ФИО, количество баллов
3. Кассир вводит количество бонусов к списанию (1 балл = 1 рубль)
4. Сумма счёта снижается на количество списанных бонусов
5. После оплаты СБИС отправляет данные о покупке в UDS

**Включение в ТП:**
1. Клиент получает api key и идентификатор организации в ЛК UDS
2. В скидках СБИС заводится «сторонняя бонусная система»
3. Автоматически создаётся интеграционная карточка UDS в онлайне

**Вебхуки UDS**: Генерируются в настройках интеграционных систем → прописываются в ЛК UDS для событий «новый клиент» / «новая операция».

### Premium Bonus (PB)

SaaS-решение для маркетинга в ресторанах, кафе, фудкортах, ритейле. Включает реферальную систему, персональные акции, рекомендации, кросс-продажи, единый профиль клиента.

**Алгоритм работы:**
1. Руководитель настраивает интеграцию (по умолчанию скидки Saby для клиентов PB — включены)
2. Кассир регистрирует гостей в PB
3. Клиент устанавливает приложение PB, копит баллы
4. Кассир вводит личный код клиента, списывает бонусы/применяет скидки
5. Presto отправляет данные о покупке в PB

### iikoCard

Бонусно-депозитная система для ресторанов. Подробнее: [[ExternalLoyalty-iiko-Integration]].

1 бонус = 1 рублю (соотношение не изменяется).

## Архитектура взаимодействия

```
Сервис интеграции  →  Online (IntegrationProxy.Get/Post)  →  Внешняя система
                                                         ↓
                            Desktop Presto/Розница (через методы Online)
```

**Карточка сервиса интеграции → Online**: При настройке/деактивации создаются тех. скидки/бонусы через `ExternalBonus.OnSettingHandler`.

**Online → Внешняя бонусная система**: Через API [Сервиса интеграции](https://wi.sbis.ru/docs/bl/objects/5952d7f6-009b-40c5-927a-fd7c457c5450/IntegrationProxy/) `IntegrationProxy.Get/Post`.

**Desktop Presto/Розница → Внешняя система**: Через методы Online, которые проксируют запросы через API Сервис интеграции.

> [!key-insight] Алгоритм расчёта бонусов UDS/PB
> При вводе кода клиента функция ценообразования делает запрос в UDS/PB. В ответ: баллы к списанию + размер скидки + данные клиента (ФИО, телефон, email, группа, номер карты). Затем проверка идентификатора в локальной БД:
> - Если найден → начисляем полученное от UDS/PB количество баллов в бонусную программу, показываем доступное к списанию
> - Если не найден → создаём клиента, связываем с продажей

## Ключевые ограничения

1. **Бонусы UDS/PB не конкурируют с другими скидками** — применяются к итоговой сумме после всех скидок и округлений
2. **Внутренняя + UDS/PB бонусная программа не суммируются** — либо одно, либо другое
3. **При применении скидки UDS/PB** начисление идёт только в UDS/PB, не в собственную программу
4. **Режим offline** — работа с бонусами UDS/PB в Presto offline и Retail offline только при наличии связи с облаком

---

## Возвраты

При возврате продажи, если есть «скидка PB»/«скидка UDS», асинхронно вызывается метод возврата баллов в PB/UDS. При офлайне данные обновляются после синхронизации с облаком. Сумма возврата рассчитывается пропорционально цене возвращаемого товара.

## Обработка ошибок

| Сценарий | Действие |
|----------|----------|
| Списание бонусов PB/UDS не прошло | Выводим ошибку, блокируем печать чека на ККТ |
| Бонусы списаны, ошибка печати ККТ | В защищённых алгоритмах откатываем списание бонусов |
| Сумма продажи < разрешённое списание | Выводим: «Списание X баллов невозможно, максимально Y» |

## База данных

**Структура для хранения внешних бонусных данных:**
- `ВидЦены` — запись бонусной программы/скидки (без карточки, тип = сторонний)
- `ВидКарты.Тип=2` — тех. вид карты для внешних участников (PB, iikoCard)
- **КодыЛиц** — связь физлица СБИС с идентификатором во внешней ИС (uid ↔ external id)

Многие сущности СБИС — «лица» (контрагенты, частные лица, номенклатуры). При обмене с внешней ИС лицу назначается идентификатор для периодической синхронизации.
