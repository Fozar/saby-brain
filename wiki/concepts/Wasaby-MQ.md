# Wasaby MQ

Две шины на базе RabbitMQ (AMQP), запущенного в k8s:

| Шина | Назначение |
|------|------------|
| **Шина серверных событий** | Облачные сервисы обмениваются произвольными событиями |
| **Шина асинхронных вызовов** | Вызов облачных методов БЛ асинхронно (AsyncInvoke) |
| **Шина клиентских событий (STOMP)** | Доставка событий клиентам (браузер) |

## Архитектура (Federated, 2 слоя)

```
БЛ-публикатор → [Proxy layer] → [Main layer] → БЛ-подписчик
                (память,        (хранение
                 высокая        очередей,
                 скорость)      диск при persist)
```

- **Proxy**: принимает события, аккумулирует при перегрузке основного слоя (≥50% памяти → вытеснение на диск)
- **Main**: обслуживает очереди сервисов; обменивается по широким каналам

## Порты кластера

| Слой | Порт | Назначение |
|------|------|------------|
| Proxy/Main | 5672 | AMQP (публикация/прием) |
| Proxy/Main | 15672 | Административная панель RabbitMQ |
| Proxy/Main | 15692 | Prometheus метрики |
| Main | 8088 | HTTP (клиентские запросы) |
| Any | 15673 | Management API (через кэш-прокси) |

## Шина асинхронных вызовов

### MessageBroker

Модуль `MessageBroker` (библиотека `sbis-msg-broker`) поставляется в SDK и включен в базовый шаблон `basic_platform`.

Два драйвера:
- **AMQP Client** — обёртка над `librabbitmq` (для сервисов в облаке)
- **EventBroker** — собственная реализация для десктопных приложений (без RabbitMQ)

### Подключение к брокеру

```
Ядро.Асинхронные сообщения.БрокерыАсинхронныеВызовыПодписка   — подключение ко всем узлам
Ядро.Асинхронные сообщения.БрокерыАсинхронныеВызовыПубликация — случайный доступный узел
```

- Публикация: случайный доступный узел → кэш соединений → при недоступности узел помечается на `Ядро.Асинхронные сообщения.ПериодВосстановленияБрокера` (default: 10 мин)
- Таймаут AMQP операций: `Ядро.Асинхронные сообщения.AMQPTimeout` (default: 3000 мс)

### Именование очередей

```
Основная:           <UUID_Облака>.<ИдБизнесЛогики>$async
Отложенные (1 мин): <ИмяОсновной>#delay60
Отложенные (1 час): <ИмяОсновной>#delay3600
```

- `UUID_Облака` — из параметра `Ядро.Асинхронные сообщения.ИдентификаторОблака`
- `ИдБизнесЛогики` — из конфигурации облака (HEX)
- Сохраняется в параметре `ОчередьАсинхронногоОбмена`

### Передача больших данных (Huge Payload Protocol)

Если аргументы AsyncInvoke превышают **100 КБ**:
1. Вызывающая сторона → загружает данные в [[Wasaby-File-Transfer|File Transfer Service]]
2. Передаёт только ссылку на данные
3. Вызываемая сторона → автоматически загружает данные перед вызовом

С версии платформы 3.18.150+. Прозрачно для прикладного кода.

## Шина серверных событий

Облачные сервисы подписываются на события и публикуют их через платформенный механизм сервисных событий.

Подписчики — группы серверов (fan-out) или единичный экземпляр сервиса.

## Конфигурация

Управление из реестра "Кластеры" на cloud.sbis.ru. Изменения применяются при следующем обновлении кластера (Helm chart).

Мониторинг очередей: **Анализ работы → Очередь загрузки → Асинхронная / Событий**

Ручной перезапуск отложенных вызовов — кнопка в реестре асинхронных очередей (аварийная ситуация).

### Состав кластера

Состав запрашивается каждые 5 минут (`RabbitMQNode.NodeListJSON`). Shovel-соединения между Proxy и Main слоями — `dynamic_shovel`.
