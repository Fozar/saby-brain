# Discount Card Subsystem Overview (СДК)

Source: `raw/Описание.md` | Domain: [[price-formation/_index]]

## Назначение

Дисконтная карта (ДК) — инструмент идентификации участника системы лояльности и предоставления выгод:
- баланс доступных бонусов и дата их сгорания
- прогресс накоплений (купи 3 — 4й в подарок)
- персональные скидки и бонусные программы

## Виды дисконтных карт

### Карта онлайна (Карта Saby)
Выдана в приложениях СБИС. Только для таких карт доступно отображение бонусов/скидок.

| Вид | Описание |
|-----|----------|
| **Персональный счёт** | Без номера, привязана к лицу. Автосоздаётся при вступлении в программу лояльности. Хранит персональные бонусы |
| **Электронная карта** | Только электронный вид, может иметь Apple Wallet (.pkpass) или Google Pay (JWT) образ |
| **Электронная-пластиковая** | Пластиковая + электронная эмиссия. Пластик более не выпускается |

### Хранимая карта (Внешняя карта)
Добавлена пользователем в Saby Get. Карты сторонних организаций.

| Вид | Описание |
|-----|----------|
| **Сетевая** | Добавлена через шаблон |
| **Произвольная** | Добавлена без шаблона |

## Сервис дисконтных карт (СДК)

Общее хранилище ДК, выданных в облаке и Desktop-приложениях, а также карт из Saby Get.

**5 частей сервиса:**
1. API-методы для CRUD хранимых карт
2. Синхронизация данных с онлайном (агрегация для Saby Get)
3. API для формирования/обновления Apple Wallet / Google Pay образов
4. API для push- и гео-уведомлений на AW-образы
5. Инструменты диагностики

## Декомпозиция подсистемы

| Сервис | Роль |
|--------|------|
| **Дисконтные карты в онлайне** | Основные механики (создание, удаление, импорт, применение на продаже) внутри одного аккаунта |
| **Сервис ДК (discount-cards)** | Общее хранилище, межаккаунтное взаимодействие (франшиза), API для SabyGet и inside-админки. Является центральным хранилищем карт и бонусных балансов для [[Franchise-Loyalty-System|франшизных сетей]] |
| **pass-updater** | Создание, обновление и доставка Apple Wallet / Google Pay образов. См. [[PassUpdater-Service]] |
| **discount-cards_diagnostic** | Диагностические и мониторинговые инструменты. См. [[DiscountCard-Diagnostic-Service]] |

## Ключевые особенности

- `Карта.Лицо` (`@Лицо`) — связь с физлицом → персона из сервиса профилей
- `ВидЦеныВидКарты` — связь карты с акциями и бонусными программами
- Бонусы хранятся на персональном счёте **и/или** на номерных картах. Суммируются для лица
- Бонусы могут: начисляться (в т.ч. отложенно), списываться, сгорать, удаляться вручную
- Номерные карты со штампованной акцией хранят прогресс по штампам

## Информационная модель

| Термин | Описание |
|--------|----------|
| **Карта лояльности** | Позволяет пользоваться системой лояльности и идентифицировать физлицо |
| **Персональная карта** | Без номера, у одного физлица одна. Хранит персональные бонусы и статистику |
| **Номерная карта** | С номером. Физлицо может иметь множество с разными выгодами |
| **Эмиссия (выпуск)** | Группа номеров для одного типа номерных карт |

## Интеграции

| Сервис | Роль |
|--------|------|
| **SabyGet** | Просмотр/создание/удаление карт, получение образов, вступление в программу лояльности |
| **file-storage** | Изображения карт и шаблонов |
| **Криптография / cvs / hsm** | Подписание AW-образов сертификатом разработчика |
| **pass-updater** | Создание и обновление AW/GPay образов |
| **Сервис рассылок** | Массовые рассылки на AW-образы |
| **Брендбук** | Настройки дизайна для карт и промокодов |
| **Webshot** | Скриншоты frontend-компонентов (баннеры штампиков) |
| **СБИС.Диск** | Загрузка/удаление файлов (иконки, баннеры, excel-файлы) |
