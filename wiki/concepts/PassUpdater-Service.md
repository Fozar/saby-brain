# Pass-Updater Service

Source: `raw/Сервис обновления образов (pass-updater).md` | Domain: [[price-formation/_index]]

Сервис `pass-updater` входит в состав web-приложения `discount-cards`. Отвечает за взаимодействие с Apple и Google для создания/обновления электронных образов ДК, промокодов, сертификатов и абонементов.

## Роль в системе

| Интеграция | Назначение |
|-----------|-----------|
| **Apple Wallet Service (AW)** | Регистрация устройств на обновление, обновление образов, отмена регистрации |
| **Apple Notification Service (APN)** | Уведомление устройств об обновлении AW-образов |
| **Google API** | Создание и обновление Google Pay образов |
| **Integration-settings** | Хранение секретного json-ключа для Google API |

## Архитектура обработки

```
Запрос → Task + Request[] → ProcessQueue
                                ↓
                    EntryPoint (бесконечный цикл)
                        → Request.Process (новая транзакция)
                        → Результат в данные запроса
                        → Processing (лог взаимодействий)
                    При ошибке: повторно в очередь (макс. 5 попыток)
                                ↓
                    ResultQueue → TaskDeliveryProcessor → DWC → СДК
```

## Сущности

| Сущность | Таблица | Описание |
|----------|---------|----------|
| **Task** (Задача) | `Task` | Бизнес-процесс. Головная сущность: порождает запросы и почтовые ящики. Содержит результат и информацию о доставке |
| **Request** (Запрос) | `Task` | Составная единица задачи. Конкретная операция взаимодействия с внешней системой |
| **MessageBox** (Почтовый ящик) | — | Хранит данные для конкретного адресата (устройство или внешняя система). Имеет TTL |
| **ProcessQueue** | `ProcessQueue` | Очередь запросов на обработку |
| **ResultQueue** | `ResultQueue` | Очередь задач, ожидающих доставки результата |
| **Processing** | `Processing` | Лог взаимодействий с внешними системами |

### MessageBox — особенности
- После доставки повторное обращение невозможно
- Если сообщение ещё не готово — адресат получает ответ "вернись позже"
- Некоторые записи имеют TTL

**Пример: обновление AW-образа:**
1. Создаётся ящик с информацией "какие образы обновить"
2. После получения устройством → создаются ящики с обновлёнными образами (1 образ = 1 ящик)
3. Устройство запрашивает образ → REST-ответ

## Задачи планировщика

| Задача | Метод | Описание |
|--------|-------|----------|
| `CancelExpiredNotifications` | `Notification.CancelExpired` | Отмена просроченных уведомлений (каждые 12 часов) |
| `PassUpdaterClearIrrelevantData` | `PassUpdater.ClearIrrelevantData` | Удаление неактуальных данных |
| `TaskDeliveryProcessor` | `Task.DeliveryProcessor` | Доставка результатов (каждые 15 с). Актуально только для Google (GPay) |

## Задержки повтора при ошибках (макс. 5 попыток)

| Сценарий | Задержка |
|----------|----------|
| PUSH-уведомление / обновление AW | 1 минута |
| Создание GPay-шаблона | 1 час |
| Обновление GPay-шаблона | 1 минута |
| Создание GPay-образа | 1 минута (3 с, если шаблон ещё не создан) |
| Обновление GPay-образа | 1 минута |

## Google Pay vs Apple Wallet

| | Google Pay | Apple Wallet |
|---|-----------|-------------|
| **Взаимодействие** | Синхронно из pass-updater | Асинхронно через APN |
| **Устройства** | Взаимодействуют с сервисами Google | Взаимодействуют с pass-updater по REST |
| **Образ** | JWT-объект в Google Wallet | .pkpass-файл, устанавливается на устройство |
| **Доставка результата** | ResultQueue → TaskDeliveryProcessor → DWC | Устройство само запрашивает обновление |

## Индексы БД

| Индекс | Выборка |
|--------|---------|
| `ProcessQueue.DateTime` | Получение запросов из очереди |
| `MessageBox.CancledNotification` | Отмена уведомлений |
| `MessageBox.ExpiredNotification` | Отмена истёкших уведомлений |

## Обновление сервиса

Сервис **нельзя** обновлять мастер-репликой: СДК записывает в таблицы pass-updater (ProcessQueue, статусы), а при мастер-реплике БД доступна только для чтения.

**Решение:** Лёгкое обновление (hot-tab) или лёгкая конвертация. При необходимости изменений БД — ВНР с минимальными блокировками.
