# Wasaby Profiles Service

Централизованное хранилище персональных данных пользователей (`profiles2`). Позволяет межсхемное взаимодействие — обмен данными между пользователями разных аккаунтов.

## Ключевые термины

| Термин | Значение |
|--------|----------|
| **Клиент** | ID клиента = номер схемы в БД Online (или `3` для Inside) |
| **Лицо** | ID записи `@Лицо` из таблицы `ЧастноеЛицо` в БД Inside/Online |
| **Пользователь** | 64-битное значение: старшие 32 бит = Client ID, младшие 32 бит = Лицо ID |
| **Персона** | UUID физического лица в сервисе профилей; однозначно идентифицирует человека вместо пары (Клиент, Лицо) |
| **Флаг «ЕстьПользователь»** | Признак наличия аккаунта у частного лица (не путать с термином «Пользователь») |

## Архитектура

Три изолированных контура:

| Сервис | Тип | Назначение |
|--------|-----|------------|
| `profiles` | ReadWrite | Сценарии с изменением данных |
| `profiles-ro-any` | ReadOnly | Критичные бизнес-сценарии без изменений |
| `profiles-ro-spec` | ReadOnly | Высоконагруженные сценарии без изменений |

### Синхронизация

Двусторонняя синхронизация с основным сервисом:
- **Из основного → profiles**: вся информация по `ЧастноеЛицо`
- **Из profiles → основной**: личные контакты + контакты из других схем

> Это позволяет избежать обращений к сервису профилей при запросе контактных данных по персоне из той же схемы.

### Поиск персоны

1. Сначала запрос в **локальной схеме** (большинство случаев)
2. Если персона не локальная → обращение к сервису профилей

## Хранимые данные

- Общая информация: ФИО, пол, город, часовой пояс, активность
- Контактные данные и подтверждённые контакты
- Опыт работы
- Отпечатки личных сертификатов физических лиц (через удостоверяющий центр)
- Метаданные аватарок (файлы — на FileStorage, раздача через Previewer)
- ID корня на СБИС Диске для хранения личных фотографий

## Типы контактов

`email`, `mobile_phone`, `home_phone`, `work_phone`, `phone`, `icq`, `skype`, `telegram`, `viber`, `whatsapp`, `vk`, `facebook`, `odnoklassniki`, `linkedin`, `twitter`, а также произвольные типы.

## Особые режимы

- **Демо-режим**: отдельный сервис со своей базой данных
- **ФЛ-режим**: модуль `Сервис Профилей ФЛ` для `my.sbis.ru`

## Права доступа

| Роль | Доступ |
|------|--------|
| Текущий пользователь | Полный доступ к своим данным |
| Менеджер отдела кадров | Доступ к данным пользователей своего аккаунта |
