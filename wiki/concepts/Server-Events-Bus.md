# Шина обмена серверными событиями

Сервис для обмена произвольными событиями между облачными сервисами БЛ. Построен на RabbitMQ.

## Архитектура (федеративная, 2 слоя)

Идентична [[Async-Calls-Bus]]:

```
БЛ-источник → AMQP → [Proxy Layer] → [Main Layer] → БЛ-подписчик
```

| Слой | Роль |
|---|---|
| **Proxy** | Буферизация; данные в ОЗУ (>50% → диск) |
| **Main** | Очереди сервисов; управляющий http-сервер (15673) |

## Отличие от async bus

- Async bus: вызовы БЛ-методов (request-reply паттерн)
- Server events bus: произвольные события (pub-sub паттерн)
- Инфраструктура RabbitMQ идентична: обменники по именам событий, durable/transient очереди

## Сценарий доставки

Подписчиком обычно выступает **группа серверов** (вся группа = одно целое). Экземпляр из группы принимает событие случайно.

Режим доставки при подписке:
- **Без гарантии** (по умолчанию): событие теряется если адресат недоступен
- **Гарантированная** (`durable` в Python / `SetIsDurable` в C++): событие сохраняется в БД брокера и доставляется после рестарта

## Гарантии доставки

```
Два сценария:
1. Прямая доставка: Источник → Брокер → Подписчик
2. Агрегация: Источник → Сервис агрегации → Брокер → Подписчик
```

> [!note] Сервис агрегации не поддерживает гарантированную доставку.

## Публикация из одного процесса

Можно публиковать и обрабатывать событие в одном рабочем процессе — брокер не участвует.

## Управление очередями в UI

`cloud.sbis.ru` → Анализ работы → Очередь загрузки → Очередь событий
