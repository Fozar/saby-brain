# iiko/iikoCard Integration

Интеграция СБИС (Presto/Розница) со сторонней бонусной системой **iikoCard**. Реализована для клиента УК «Алендвик», ООО.

---

## Что такое iiko

iiko — специализированная ERP для автоматизации учёта в ресторанах, кафе, барах, службах доставки. Состоит из двух компонентов:

- **iikoCard** — бонусно-депозитная система, отвечающая за лояльность
- **iiko** — модуль обмена данными с учётной системой (выгрузка/загрузка справочников и документов)

---

## Предварительные требования

1. Настроить интеграцию с **iiko** для загрузки каталога товаров (CSV или API)
2. Создать интеграционную карточку **iikoCard** на сервисе интеграций

**Базовые требования работы:**
- Регистрация покупателей в iikoCard происходит **строго за рамками СБИС**
- Одно и только одно подключение к iiko И одно к iikoCard
- Идентификация клиента — **только по карте iiko** (телефон/ФИО запрещены)
- Лояльность СБИС и iikoCard **не смешиваются** (либо одно, либо другое)
- Desktop Presto/Розница работает только **при наличии связи с облаком**

---

## Алгоритм продажи

1. Кассир сканирует карту iikoCard клиента
2. СБИС запрашивает в iikoCard: доступные к списанию бонусы + сумма скидки
3. Если покупатель не найден в СБИС → создаётся с привязкой к iikoCard id (таблица `КодЛица`)
4. Кассир видит доступный баланс бонусов, вводит сумму к списанию
5. Скидки/бонусы пропорционально распределяются по позициям (тех. скидка + тех. бонусная программа)
6. СБИС создаёт заказ в iiko и фиксирует продажу

**Если задан список бонусных программ в настройке интеграции** → списание разрешено только по этим программам.

---

## Известные проблемы

> [!gap] Временная задержка обновления баланса
> Между закрытием заказа в СБИС и фактическим проведением бонусной операции в iiko баланс может не обновляться **несколько минут**. Теоретически позволяет списывать одни и те же бонусы дважды. При списании: заказ создаётся, запрос на закрытие выполняется успешно, но заказ реально не закрывается и бонусы не списываются (риск уйти в минус).

**Зависимость программы от точки питания**: в разных точках сети могут быть разные программы обслуживания.

---

## Обработка ошибок (специфика iiko)

В отличие от UDS/PB у iikoCard особая обработка:

- **Ошибка при фиксации продажи в iikoCard** → СБИС продажа **не закрывается**, пользователю выводится ошибка с вариантами действий:
  - Исправить замечания iikoCard (например, изменить количество списываемых бонусов)
  - Отменить работу с iikoCard в данной продаже
- Все ключевые взаимодействия с web API iikoCard сохраняются в **служебную историю продажи** (недоступна пользователям, доступна технологам/разработчикам)

---

## Отладка лояльности iikoCard

1. После добавления клиента в продажу в служебной истории появляется ссылка на временный файл на СБИС.Диске
2. Открыть ссылку → скачать файл отладки
3. Перейти в IikoCard → Справка → Анализ лояльности
4. Вставить содержимое файла → получить список акций с пометками (сработала/не сработала/почему)

> [!key-insight] Срок хранения файла отладки
> На боевых серверах — **24 часа**. На тестовых стендах — меньше. Задаётся глобально в настройках облака для всех временных файлов на СБИС.Диске.

## API

**Цепочка вызовов СБИС:**
```
CRMClients.Suggest → ExternalIntegration.Find → Uds.GetCustomer
```
Через IntegrationProxy: `IntegrationProxy.Get/Post`

**iikoCard WEB API методы:**

| Метод | Назначение |
|-------|-----------|
| `POST /api/1/access_token` | Авторизация / создание сессии |
| `POST /api/1/organizations` | Информация по точке питания (ТП) |
| `POST /api/1/loyalty/iiko/customer/info` | Идентификация участника по номеру карты |
| `POST /api/1/order/create` | Формирование продажи, связывание покупателя, установка бонусов как оплаты |
| `POST /api/1/loyalty/iiko/calculate` | Расчёт скидок и бонусов в продаже |
| `POST /api/1/order/close` | Фиксация продажи |

Публичное API: [API Внешних бонусных программ](https://online.sbis.ru/shared/disk/7b00b912-87a2-4866-ac6f-8e935bf2d207)

## Схема базы данных

**Ключевые таблицы:**
- `КодЛица` — идентификатор покупателя iikoCard (связь физлица СБИС с iikoCard id)
- `ВидЦены` — тех. скидка (без карточки) для отображения в интерфейсе
- `ВидКарты.Тип=2` — тех. вид карты для карт участников лояльности iikoCard

**Сервисные взаимодействия:**
```
Карточка сервиса интеграции → Online → iikoCard (через IntegrationProxy)
                                ↑
              Desktop Presto/Розница (через методы Online)
```

При настройке/деактивации интеграции → `ExternalBonus.OnSettingHandler` создаёт тех. скидки/бонусы.

## Архитектура интерфейса

**Карточка интеграции** — настройка подключения к iikoCard.

**Декомпозиция модулей:**
