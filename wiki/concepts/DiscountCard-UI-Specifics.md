# Discount Card UI Specifics

Source: `raw/Особенности работы интерфейса.md` | Domain: [[price-formation/_index]]

## Реестр типов карт

Основной компонент: платформенная плитка `Controls/tile:View` с кастомным шаблоном (название, превью вида карты, статистика).

**Пустое представление** (нет созданных типов): показывается форма создания, скрываются кнопки в шапке и тулбаре.

### Источники данных

#### 1-я группа методов (параллельно при открытии страницы)
| Метод | Назначение |
|-------|-----------|
| `DiscountCardType.GetListSimple/4` | Список типов карт (название, превью, активный/неактивный) |
| `Bonus.IsActive` | Есть ли бонусные программы → выделяет место под бонусный баланс в плитке |
| `LoyaltyProgram.GetFranchiseList/1` | Список франшиз → показывает кнопку синхронизации если клиент владелец |

#### 2-я группа методов (последовательно)
| Метод | Назначение |
|-------|-----------|
| `DiscountCardType.GetListSimple/4` | Лимит 1. Если нет записей — нужно пустое представление |
| `DiscountCardType.Create/2` | Создаёт черновик, если GetListSimple не вернул записи |

#### После первого построения (клиент)
| Метод | Назначение |
|-------|-----------|
| `DiscountCardType.GetListWithStats/1` | Статистика (бонусный баланс, новые карты за период). Вызывается после построения — может работать долго |

### Действия в реестре
| Метод | Действие |
|-------|---------|
| `DiscountCardType.CanDelete` | Проверить, можно ли удалить (нельзя если есть выданные карты) |
| `DiscountCardType.Update` | Деактивировать (если нельзя удалить) / активировать (если уже деактивирован) |
| `DiscountCardType.Delete` | Полное удаление (если возможно) |
| `DiscountCardType.UpdateName` | Переименование из диалога |

### Сохранение в пустом представлении (последовательно)
| Метод | Назначение |
|-------|-----------|
| `SalesPoint.Read/2` | Информация о ТП → заполняет блоки "Телефон" и "Адрес" на обратной стороне дизайна |
| `DiscountCardType.Init/1` | Сохраняет черновик с данными (Название, ТП, дизайн) |

## Создание типа карты

Открывается в диалоговом окне. Поля: название, точки продаж (обязательно), упрощённая настройка дизайна (контроллер брендбука `BrandbookCore/controller`).

Предзагрузчик диалога вызывает `DiscountCardType.Create/2`.
Тот же компонент используется при создании из пустого представления.

## Настройка типа карты — Мастер-деталь

Основа каждой страницы настройки: `Controls/masterDetail:Base`.
- **Мастер**: список страниц + статистика по типу + фильтр по периоду (возле "Клиенты")
- **Деталь**: разные компоненты в зависимости от страницы

Навигация по страницам: `Page/base.NavigationController.navigate`

Статистика (после первого построения, может работать долго):
- `DiscountCard.GetListWithStatsTotals/1` — количество карт, покупки, сумма, баланс бонусов
