# Wasaby Cloud Sync Broker

**Облачный брокер синхронизации** — сервис записи фактов изменений для синхронизации данных между облаком и офлайн-приложениями (мобильными, плагинами).

## Ключевая идея

Брокер хранит не сами изменения, а **факты изменений** (что изменилось и когда).

Потребитель при синхронизации спрашивает: **"что изменилось с момента последней синхронизации (timestamp)?"** — и получает список изменённых сущностей.

## Архитектура синхронизации

```
BL записывает факт изменения → Sync Broker
                                      ↓
                          Немедленное STOMP-уведомление
                                      ↓
                          Офлайн-приложение просыпается
                                      ↓
                          Запрашивает: "что изменилось с T?"
                                      ↓
                          Sync Broker возвращает список фактов
```

## Реактивная подсистема

Как только факт записан, брокер **немедленно отправляет STOMP-уведомление** подключённым офлайн-приложениям. Это позволяет:
- Не опрашивать сервер по таймеру
- Синхронизироваться сразу при наличии изменений
- Включать в уведомление полные данные изменения (избегает дополнительный round-trip)

## Отличие от прямой передачи данных

| Подход | Хранит | Масштабируемость |
|--------|--------|-----------------|
| Прямая передача | Сами данные | Плохо (большие объёмы) |
| Sync Broker | Факты изменений | Хорошо (компактно) |
