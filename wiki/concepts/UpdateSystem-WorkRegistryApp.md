# Приложение Реестр работ — интерфейс и алгоритмы

Реестр работ — основное и единственное место хранения работ, выполняемых Хоттабычем и сотрудниками ЦОДа.

> [!key-insight] Реестр vs Планы выпуска
> В реестре хранятся **РАБОТЫ**, в планах выпуска — **ПУНКТЫ ПЛАНА** (ярлыки, ссылающиеся на конкретную работу). При открытии пункта плана из любого плана — всегда открывается одна и та же оригинальная работа.

## Структура реестра

Все работы сгруппированы по типу. Внутри каждого типа — группировка по версии и **дате выполнения на бою** (дата плана выпуска, в который включена работа).

## Варианты отображения

| Режим | Описание |
|-------|---------|
| **Дерево** | Стандартный вид. Папки Тип-Версия-Дата. Легко найти работу, увидеть список по типу на день. Сортировка внутри папок — по алфавиту |
| **Список** | Линейное представление. Быстрая сортировка по колонкам: Название/Версия/Дата/Тестировщик |
| **Дерево без версии** | Coming soon. Папки Тип-Дата (без версии) |

## Элементы управления

| Кнопка | Действие | Требует ПМО |
|--------|----------|-------------|
| Добавление работы (+) | Выбор типа работы → карточка для заполнения | Нет |
| Изменение даты | Изменение даты выбранных работ | Да* |
| Выгрузка в Excel | Выгрузка выбранных работ и папок | Да* |
| Комментарий | Подписать комментарий у выбранных | Да* |
| Удаление | Удаление из реестра (из всех планов!) | Да* |
| Строка поиска | Поиск по Названию/Автору/Тестировщику/Версии/Дате/Ссылке | Нет |
| Фильтры | Настройка отбора | Нет |

### Боковая панель (по порядку)
- Переключение между основным и twin-Хоттабычем
- Переход в cloud
- Переключение режима отображения
- История реестра работ (кто и что делал)
- Включение/Выключение ПМО
- Копировать ссылку на текущий настроенный реестр

## Правила изменения даты

- Если дата плана **ещё не завершилась** → работа переносится в новый план, из старого пропадает
- Если дата плана **в прошлом** (наступило следующего дня 00:00) → работа остаётся в старом плане И добавляется в новый

## Удаление работы

> [!warning] Внимание
> Удаляя работу, нужно понимать что она будет удалена из **всех планов**, в которые была включена.

## Фильтры реестра

### Быстрый фильтр

Только выбор стенда. По умолчанию в РУ-облаке — PROD.

### Расширенный фильтр

| Поле | По умолчанию | Описание |
|------|-------------|---------|
| Облако (Стенд) | Prod | Стенд для отбора статусов и принадлежности к облаку |
| Сервис | — | Устаревшее, скоро будет удалено |
| Период выполнения | Неделя | Сегодня / Вчера / Неделя (с пн по вс) / Месяц / Произвольный |
| Скрывать/показывать выполненные | Скрыты | Отбор по статусу «Выполнено» для выбранного стенда |
| Наличие работ в облаке | Не выбран | Принадлежность к облаку (стенду) |
| Статус | Не выбран | Автоматические и пользовательские статусы |
| Критичность | Не выбран | Наличие/отсутствие флага критичности |
| Тип работы | Не выбран | По существующим типам работ |
| Регион | Не выбран | RU / KZ / UZ / TM |

## Основные алгоритмы работы

### Добавление работы

1. Нажать «+»
2. Выбрать тип работы
3. Вставить ссылку, заполнить поля
4. Проставить дату выполнения на бою
5. Сохранить карточку работы
6. Работа добавляется в план на указанную дату в папку «по умолчанию»

### Изменение даты у текущих работ

1. Найти работу (по ссылке или другим способом)
2. Выделить через ПМО
3. Нажать «Изменить дату»
4. Указать нужную дату и сохранить

### Выгрузка в Excel

1. Настроить фильтр
2. Выбрать через ПМО работы и папки
3. Нажать «Выгрузка в Excel»
4. Сохранить полученный файл
