# Wasaby — Управление облаком (cloud-ctrl)

Сервис: [cloud.sbis.ru](https://cloud.sbis.ru/)

Основные разделы:
- **Приложения** (Структура облака) — реестр всех объектов облака
- **Очередь** (Текущая загрузка сервисов) — мониторинг активных сервисов
- **Клиенты** — управление клиентами
- **Пользователи** — управление пользователями облака

---

## Приложения (Структура облака)

Иерархия объектов облака:

```
Семейство web-приложений
└── Web-приложение
    ├── Web-сервис
    │   └── Бизнес-логика (узел сервиса)
    │       └── База данных
    ├── Multitenancy-сервис
    │   └── Tenant-группа
    └── Multiversion-сервис
```

### Фильтрация приложений

- По серверам, версии, ответственному, дистрибутиву
- По стендам разработчиков, сайтам, аутентификации, блокировке
- По Kubernetes, REST API, репликам
- По умолчанию: без тестовых стендов

### Пометки на записях

| Иконка/цвет | Значение |
|-------------|----------|
| Зелёный | Все узлы активны |
| Жёлтый | Часть узлов неактивна |
| Красный | Все узлы неактивны |
| Пунктир | Нет сайтов |
| Мигающий красный флажок | Конфигурация диспетчера устарела |

Цвет обновления: серый=полная конвертация, зелёный=лёгкое, голубой=лёгкая конвертация, красный=мастер, оранжевый=реплика, жёлтый=посхемное.

### Операции

- **Обновление конфигурации диспетчера** — для выбранных сайтов
- **Остановка/Перезапуск** — запуск/остановка/перезагрузка BL сервисов с прогресс-баром
- **Выгрузка CSV** — список сервисов (без тестовых стендов)
- **Аналитика** — агрегация по версии/ответственному
- **Drag'N'Drop** — перемещение с учётом иерархических правил

### Описание в k8s

1. Активировать флаг "Развернут через Kubernetes"
2. Заполнить: Кластер Kubernetes, Пространство имени, Экземпляр объекта
3. На сервисах заполнить вкладку "Узлы сервиса" → "Контейнер" и "Helm"
4. Обновить через Хоттабыч

## Очередь (Текущая загрузка сервисов)

Путь: cloud.sbis.ru → Оркестратор облака → Очередь

### Общая статистика

Три колонки:
1. Очередь — запросы, ожидающие исполнения
2. Служебная очередь — запросы в служебном пуле
3. Название / системное имя / URL сервиса или tenant-группы

### Пулы процессов

- **Основной пул** — синхронные (интерфейсные) запросы, ожидается быстрый отклик
- **Служебный пул** — асинхронные, задачи планировщика, долгие фоновые задачи

Счётчики: `работают / стоят / в резерве`

Параметр `Ядро.Сервер приложений.ЧислоДополнительныхПроцессов` — доп. процессы при перегрузке.

### Детальная статистика

- Узлы сервиса, рабочие процессы, потоки, корутины
- Фильтрация: по узлу, сессии, методу, длительности исполнения
- **Критическая ошибка** = красный цвет узла; нарушает нормальную работу (DB disconnect, Redis disconnect)

Интервал обновления: 1с / 5с / 10с / 30с / 1мин / 5мин

## Клиенты

Клиент — лицо (организация, частное лицо), зарегистрированное в системе, со своей конфигурацией приложения и изолированными данными.

Пользователь — лицо, подчинённое клиенту. Права пользователя ≤ прав клиента.

Подробнее: [Аккаунты и пользователи](https://link.sbis.ru/sabyauth/knowledge?...)
