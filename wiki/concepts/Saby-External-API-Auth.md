# Saby External API — Аутентификация и сессии

Внешний JSON-RPC API Saby (СБИС) требует сессионной аутентификации. После входа все запросы идут с заголовком `X-SBISSessionID`.

## Ограничения

- Не более **300 вызовов в минуту** для методов аутентификации — при превышении блокировка IP на 20 минут.
- Не более **5 активных сессий** на пользователя одновременно.
- Вход по ЭП завершает все прочие активные сессии автоматически.
- Сессия остаётся активной ещё **1 минуту** после вызова `СБИС.Выход`.

## Endpoint'ы

| Назначение | URL |
|---|---|
| Аутентификация (логин/пароль) | `https://online.sbis.ru/auth/service/` |
| Аутентификация по сертификату | `https://online.sbis.ru/auth/service/` |
| Все прочие методы API | `https://online.sbis.ru/service/?srv=1` |

## Методы аутентификации

### По логину и паролю — `СБИС.Аутентифицировать`

```json
POST https://online.sbis.ru/auth/service/
Content-Type: application/json; charset=UTF-8

{
  "jsonrpc": "2.0",
  "method": "СБИС.Аутентифицировать",
  "params": {
    "Параметр": {
      "Логин": "user@example.com",
      "Пароль": "secret",
      "НомерАккаунта": "optional — если несколько кабинетов"
    }
  },
  "id": 0
}
```

**Ответ (успех):**
```json
{ "jsonrpc": "2.0", "result": "000b8ee4-000b8ee5-00ba-a40615cd1a61453a", "id": 0 }
```

`result` — строка, идентификатор сессии.

### По сертификату (ЭП)

1. `СБИС.СписокСертификатовДляАутентификации` — получить список допустимых сертификатов.
2. `СБИС.АутентифицироватьПоСертификату` — войти; ответ содержит зашифрованный ID сессии (структура `ContentInfo/EnvelopedData`).
3. Расшифровать ID сессии с помощью СКЗИ (например, КриптоПро).

## Использование сессии

Передавай `X-SBISSessionID` в заголовке каждого HTTP-запроса:

```http
POST https://online.sbis.ru/service/?srv=1 HTTP/1.1
Content-Type: application/json-rpc;charset=utf-8
X-SBISSessionID: 000b8ee4-000b8ee5-00ba-a40615cd1a61453a
```

**Исключения** (без заголовка): `СБИС.Аутентифицировать`, `СБИС.АутентифицироватьПоСертификату`, `СБИС.СписокСертификатовДляАутентификации`.

## Завершение сессии — `СБИС.Выход`

```json
POST https://online.sbis.ru/auth/service/
X-SBISSessionID: <id>

{ "jsonrpc": "2.0", "method": "СБИС.Выход", "params": {}, "id": 0 }
```

Ответ: `{ "result": null }`. Сессия деактивируется через 1 минуту.

При неверном/устаревшем `X-SBISSessionID` возвращается ошибка 401 → нужно повторить аутентификацию.

## Информация о текущем пользователе — `СБИС.ИнформацияОТекущемПользователе`

```json
POST https://online.sbis.ru/service/?srv=1
X-SBISSessionID: <id>

{
  "method": "СБИС.ИнформацияОТекущемПользователе",
  "params": { "Параметр": { "ДопПоля": "СписокВнешнихПровайдеров" } }
}
```

**Ответ:**
```json
{
  "Пользователь": {
    "Фамилия": "Алексеев",
    "Имя": "Алексей",
    "Отчество": "Алексеевич",
    "ВнешнийПровайдер": [
      { "Имя": "Yandex", "Ссылка": "https://online.sbis.ru/auth/connect_to_provider?provider=Yandex" }
    ]
  }
}
```

`ДопПоля: "СписокВнешнихПровайдеров"` — условно-обязательный; без него поле `ВнешнийПровайдер` не возвращается.

## Типовой flow интеграции

```
1. POST /auth/service/ → СБИС.Аутентифицировать → получить sessionId
2. Все запросы: X-SBISSessionID: sessionId
3. При 401 → повторить шаг 1
4. По окончании работы → СБИС.Выход
```

## SMS-подтверждение (2FA)

Если аккаунт требует подтверждения, `СБИС.Аутентифицировать` вернёт ошибку c `classid 1FA000001002` и в `data.addinfo`:

- `Телефон` — маскированный номер для SMS
- `Идентификатор` — временный ID (TTL 5 минут)
- `МетодОтправкиКодаПодтверждения` → `СБИС.ОтправитьКодАутентификации`
- `МетодПроверкиКодаИсключения` → `СБИС.ПодтвердитьВход`
- `ИдентификаторСессии` — временный ID сессии

## Ошибки аутентификации

Полный справочник: [[Saby-API-Error-Codes]].

| classid | Смысл |
|---|---|
| `1FA000001000` | Rate limit (≥300 вызовов/мин), HTTP 429 |
| `1FA000001001` | Неверный логин/пароль, HTTP 500 |
| `1FA000001002` | Требуется SMS-подтверждение, HTTP 500 |
| `1AA000001000` | Нефатальная ошибка аутентификации — можно повторить |
