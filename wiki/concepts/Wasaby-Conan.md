# Conan — C++ Package Manager

**Conan** — пакетный менеджер для сборки сторонних библиотек C++ в экосистеме Tensor/Wasaby. Используется для управления зависимостями в C++ компонентах платформы.

## Принципы

- **1 библиотека = 1 рецепт = 1 репозиторий** — каждая сторонняя библиотека имеет отдельный репозиторий с рецептом
- Сборка происходит на Jenkins; готовые пакеты публикуются на conan-server
- На сборочных машинах нет доступа в интернет — все зависимости берутся с внутреннего сервера
- Версии пакетов привязаны к версии платформы: если приложение `24.1234`, пакеты должны быть `24.1200`

## Структура репозитория рецепта

```
conan-recipes/
  recipes/
    <libname>/
      conanfile.py       # рецепт сборки
      test_package/      # тесты пакета
```

## Добавление нового пакета

1. Написать `conanfile.py` с рецептом сборки
2. Написать тесты в `test_package/`
3. Локальная проверка через `local_build.py`
4. Создать MR в `conan-recipes`
5. CI на Jenkins соберёт и опубликует пакет на conan-server

## Локальная разработка

Перед работой нужно установить зависимость:

```bash
py -3.11-32 -m pip install requests
```

Запуск локальной сборки:

```bash
python local_build.py
```

Скрипт `local_build.py` находится в репозитории `conan-recipes` и позволяет проверить рецепт без Jenkins.

## Версионирование

Версия пакета = версия платформы (major.minor):
- Платформа `24.1234` → Conan-пакеты `24.1200`
- Версия патч-части (`1234` → `1200`) сбрасывается/округляется до стандартного значения
