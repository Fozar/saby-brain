# SBIS Internal API Methods

Внутренние методы SBIS (в отличие от внешнего [[Saby-External-API-Tasks|СБИС.СписокЗадач]]) — это методы платформенных объектов (`ПунктПлана`, `СлужЗап`, `Документ` и др.), доступные через тот же JSON-RPC транспорт.

> [!key-insight] Два вида task API в SBIS
> `СБИС.СписокЗадач` — внешний API, простой, для интеграций. `ПунктПлана.СписокПунктов` / `СлужЗап.*` — внутренний API, богатые поля, используется веб-клиентом. Данные форматируются через [[SBIS-Record-Format|колоночный формат]].

**Транспорт:** POST `https://online.sbis.ru/service/?srv=1` + `X-SBISSessionID: <session_id>`

## Справочник методов

| Метод | Назначение | Ключевые параметры |
|---|---|---|
| `ПунктПлана.СписокПунктов` | Список задач плана за период | `Фильтр.ИдПлана`, `ФильтрПринадлежность` (1=только я), даты |
| `ПунктПлана.ПрочитатьДляУчастника` | Детали пункта плана | `ИдО`, `ИмяМетода="ПунктПлана.Список"` |
| `ПунктПлана.AttachList` | Вложения пункта плана | `Фильтр` (catalogId) |
| `СлужЗап.СписокПодзадач` | Подзадачи документа | `Фильтр.Надзадача` (id родителя), `Навигация` |
| `СлужЗап.ПрочитатьДляУчастника` | Детали задачи | `ИдО`, `ИмяМетода="СлужЗап.Список"` |
| `Документ.ПолучитьИерархиюСвязанныхДокументов` | Связанные документы | `ИдО`, `Фильтр.Route` ("up"/"down"), `Фильтр.ИмяОбъекта` |
| `Документ.ПолучитьДанныеОДокументе` | Краткие данные документа | `docRec` |
| `Документ.Timeline` | Лента событий документа | `document`, `settings` |
| `Проект.ПрочитатьДляУчастника` | Детали проекта | `ИдО` |
| `ЛицоДокумента.СписокИсполнителейПунктаПлана` | Исполнители пункта | `Фильтр.ИдДокумента` |
| `ДокументСообщение.КоличествоСообщений` | Счётчик сообщений | `documents` (UUID) |
| `СБИС.ПрочитатьДокумент` | Полный документ по ID | `Документ.Идентификатор` |

## Паттерн ПрочитатьДляУчастника

Единый паттерн чтения объектов: применяется к `ПунктПлана`, `СлужЗап`, `Проект`.

```json
{
  "jsonrpc": "2.0",
  "method": "ПунктПлана.ПрочитатьДляУчастника",
  "params": {
    "ИдО": "<string_id>",
    "ИмяМетода": "ПунктПлана.Список",
    "ДопПоля": { "ИмяОбъекта": "ПунктПлана" }
  },
  "id": 0
}
```

Варианты:
- `ПунктПлана.ПрочитатьДляУчастника` + `ИмяМетода: "ПунктПлана.Список"`
- `СлужЗап.ПрочитатьДляУчастника` + `ИмяМетода: "СлужЗап.Список"`
- `Проект.ПрочитатьДляУчастника` + `ИдО` (без ИмяМетода)

## ПунктПлана.СписокПунктов

Список задач плана по периоду. Ответ в [[SBIS-Record-Format|колоночном формате]].

### Поля фильтра (d-массив, 10 элементов)

| Индекс | Поле | Описание |
|---|---|---|
| `[0]` | `ВсеКромеВыполненных` | bool — `false` = показать всё |
| `[1]` | `Группировка` | `"По сотрудникам"` |
| `[2]` | `ИдОП` | int\|null — узел иерархии, `null` = корень |
| `[3]` | `ИдПлана` | int — ID плана (напр. `567678097`) |
| `[4]` | `ПланДатаКнц` | date `"YYYY-MM-DD"` — конец периода |
| `[5]` | `ПланДатаНач` | date `"YYYY-MM-DD"` — начало периода |
| `[6]` | `Плитка` | bool — `false` = список |
| `[7]` | `РП.РазделИерархии` | `str[]` — `[]` |
| `[8]` | `СостояниеПунктов` | `str[]` — `[]` |
| `[9]` | `ФильтрПринадлежность` | int — `0`=все, `1`=только я исполнитель |

### Ключевые поля ответа (по индексу в `d`)

| Индекс | Поле | Описание |
|---|---|---|
| `[0]` | `@Документ` | ID документа |
| `[2]` | `Раздел@` | `true` = строка-раздел (сотрудник), не задача |
| `[9]` | `Номер` | Номер задачи |
| `[10]` | `ТипДокумента` | Тип (`СлужЗап`, `ПунктПлана`, ...) |
| `[16]` | `СвязьДокументовПлан.ПланДата` | Плановая дата |
| `[39]` | `ДокументСледствие.ДокументРасширение.Срок` | Срок |
| `[41]` | `РП.ПроцентВыполнения` | `{Выполнено:N, Готово:N, Подсказка:"..."}` |
| `[42]` | `РП.Название` | Название задачи (или ФИО для раздела) |
| `[49]` | `РП.ПланВремя` | Плановое время (часы) |
| `[50]` | `РП.ФактВремя` | Фактическое время (часы) |
| `[55]` | `РП.ДочерниеПункты` | Список ID дочерних пунктов |
| `[64]` | `РП.ИдПроекта` | ID проекта |
| `[65]` | `РП.НазваниеПроекта` | Название проекта |
| `[88]` | `РП.Просрочен` | Просрочено (bool) |
| `[93]` | `Ид` | String ID пункта плана ← использовать в API-запросах |

> [!key-insight] Строки-разделы
> Строки с `Раздел@=true` — заголовки групп сотрудников, не задачи. Фильтровать перед обработкой.
