# Шина обмена асинхронными вызовами

Сервис для асинхронного вызова облачных методов БЛ между сервисами. Построен на RabbitMQ + AMQP.

## Архитектура (федеративная, 2 слоя)

```
БЛ-публикатор → AMQP → [Proxy Layer] → [Main Layer] → сервис-подписчик
```

| Слой | Роль |
|---|---|
| **Proxy (проксирующий)** | Лёгкие соединения, буферизация при перегрузке main |
| **Main (основной)** | Очереди всех сервисов; передаёт задачи подписчикам |

Внутри proxy данные хранятся в ОЗУ. При >50% памяти — вытеснение на диск.

## Именование очередей

- **Основная**: `<UUID_облака>.<ИдБизнесЛогики>$async`
- **Отложенные**: `<основная>#delay60` (1 мин), `#delay3600` (1 ч)
- Для локальных стендов: `<хост>:<порт><каталог>`

## Huge Payload Protocol

При размере аргумента > 100 КБ:
1. Данные выгружаются в [[File-Transfer-Service]]
2. В запросе передаётся ссылка
3. Принимающая сторона автоматически загружает данные до вызова метода

## Подключение к брокеру (sbis-msg-broker)

- **Подписка**: соединение сразу со всеми узлами кластера
- **Публикация**: случайный доступный узел; кэш соединений; недоступный помечается на 10 мин (настраиваемо)
- AMQP timeout по умолчанию: **3000 мс** (`Ядро.Асинхронные сообщения.AMQPTimeout`)

## Драйверы

| Драйвер | Применение |
|---|---|
| **AMQP Client** | Cloud-сервисы (обёртка над librabbitmq) |
| **EventBroker** | Desktop-приложения (нет доступа к cloud RabbitMQ) |

## Управление очередями в UI

`cloud.sbis.ru` → Анализ работы → Очередь загрузки → Асинхронная очередь:
- Сервис, Задержка (сек), Очередь, Отложено, Выгруженные в amqp-tools
- Массовый перезапуск отложенных вызовов — кнопка в реестре

## Порты

| Слой | Порт | Назначение |
|---|---|---|
| Proxy | 5672 | AMQP от БЛ |
| Proxy | 15672 | Admin панель |
| Proxy | 15692 | Prometheus |
| Main | 5672 | AMQP между слоями |
| Main | 15672 | Admin панель |
| Main | 15692 | Prometheus |
| Main | 8088 | HTTP для облака |
