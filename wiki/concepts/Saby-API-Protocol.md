# Saby API — Протокол, структура запросов и типы данных

Описывает транспортный уровень внешнего API Saby: HTTP-структура, JSON-RPC 2.0, типы данных, форматы строк.

---

## Эндпоинты

| Назначение | URL |
|-----------|-----|
| Аутентификация | `https://online.sbis.ru/auth/service/` |
| Все прочие команды | `https://online.sbis.ru/service/?srv=1` |
| Бинарные данные (GET) | URL из поля `Файл.Ссылка` (обычно `disk.sbis.ru`) |

Сервер работает в UTC+3 (Москва). Все даты и время в ответах — московские.

---

## HTTP-запрос (POST)

### Обязательные заголовки

```http
POST https://online.sbis.ru/service/?srv=1 HTTP/1.1
Content-Type: application/json-rpc;charset=utf-8
X-SBISSessionID: <session-id>
Content-Length: <bytes>
User-Agent: YourSystemName/0.1.0
Accept: */*
```

- `Content-Type`: рекомендуется `utf-8`; поддерживается и `windows-1251`.
- `X-SBISSessionID`: идентификатор сессии из ответа [[Saby-External-API-Auth|аутентификации]].
- `User-Agent`: название и версия вашей системы (обязательно).

## JSON-RPC 2.0

Saby использует [спецификацию JSON-RPC v2.0](https://www.jsonrpc.org/specification). Все запросы — в кодировке UTF-8.

### Объект запроса

```json
{
  "jsonrpc": "2.0",
  "method": "СБИС.ПрочитатьДокумент",
  "params": {
    "Документ": {
      "Идентификатор": "e1756445-f1be-4562-b5cf-4549968b5f4f"
    }
  },
  "id": 0
}
```

| Поле | Описание |
|------|----------|
| `jsonrpc` | Всегда `"2.0"` |
| `method` | Название команды (кириллица) |
| `params` | Объект параметров команды |
| `id` | Строка или целое число — возвращается в ответе |

### Объект ответа (успех)

```json
{
  "jsonrpc": "2.0",
  "result": { ... },
  "id": 0
}
```

`result` не имеет фиксированного формата. Если метод вернул null → `"result": null`.

### Объект ответа (ошибка)

```json
{
  "jsonrpc": "2.0",
  "error": {
    "code": -32000,
    "message": "Сервис оператора временно недоступен.",
    "details": "Подробное описание (логируется, не показывается пользователю)",
    "data": {
      "classid": "00000000-0000-0000-0000-1FA000001001",
      "addinfo": null
    }
  },
  "id": 0
}
```

| Поле | Описание |
|------|----------|
| `code` | Целочисленный код (`-32000` для всех Saby-ошибок) |
| `message` | Текст пользователю |
| `details` | Подробности для лога (не для UI) |
| `data.classid` | Расширенный классификатор → [[Saby-API-Error-Codes]] |

## GET-запрос (бинарный контент)

```http
GET https://disk.sbis.ru/disk/api/v1/<uuid>?object=simple_file_sd&...
Content-Type: application/json-rpc;
X-SBISSessionID: <session-id>
Host: disk.sbis.ru
```

Ответ: `Content-Type: application/octet-stream`, бинарные данные файла.

## Типы данных

Saby JSON поддерживает три базовых типа:

### Строка

```json
"Название": "Фактура № 829766305 от 17.04.15 на сумму 1 026 996.48р."
```

- Ключ в кавычках, разделитель `:`, значение в кавычках.
- Допустимо экранирование `\`.

### Объект

```json
{
  "Идентификатор": "",
  "Имя": "Иван",
  "Отчество": "Иванович",
  "Фамилия": "Иванов"
}
```

Неупорядоченное множество пар «ключ: значение» в фигурных скобках.

### Массив объектов

```json
"Этап": [{"Вложение": "", ...}]
```

Массив в квадратных скобках. Допускается передавать одиночный объект вместо массива из одного элемента:

```json
"Этап": {"Вложение": "", ...}
```

## Форматы и ограничения строк

| Ограничение | Значение |
|-------------|---------|
| Максимальная длина строки | 255 символов |
| Регистрозависимость значений | Да (`"ВременныйКод" ≠ "ВРЕМЕННЫЙКОД"`) |
| `ДвоичныеДанные` — максимальный размер файла | 70 МБ |
| `Файл.Имя` — недопустимые символы | Недопустимые символы имён файлов Windows/Linux |

Строки должны содержать только символы, допустимые для JSON-RPC (RFC 4627) и XML.
