# SabyGet Landing Page & Data Architecture

## Разводящая страница

**URL**: [sabyget.ru](https://sabyget.ru/)

Разделена на 2 области:
- **Слева** — заведения текущего региона, сгруппированные по категориям (Рестораны, Салоны, Магазины…), отсортированные по популярности
- **Справа** — карта заведений в выбранном регионе

![[e20a67b67cc5784a31e5812261b5034c_MD5.avif|683]]

### Компоненты страницы

| Компонент | Описание |
|-----------|----------|
| Шапка с регионом | Приветственный баннер + поиск; настраивается в Get Saby |
| Карусель заведений | Популярные заведения по категории; вход: регион |
| Карусель курсов | Ближайшие курсы; данные от «Обучающей системы» |
| Карусель мероприятий | Ближайшие события; данные от «Обучающей системы» |
| Список категорий | Категории заведений в регионе; клик → подборка |
| Карта заведений | Все заведения региона на карте |

### Основной принцип работы (flow)

```
Пользователь открывает sabyget.ru
  ↓
Page/base:Controller загружает данные
  ↓
Определение региона (приоритеты):
  1. ID в URL: sabyget.ru/city/[city_id]
  2. Кука sgCityData
  3. Геолокация пользователя
  4. Fallback: Москва
  ↓
Locality.GetProperties(city_id) → полная информация о регионе
  ↓
Сохранение города в кuke sgCityData
  ↓
Establishment.MainPage(city_id) → каруселей + категории
```

**Ключевые BL-методы:**
- `Locality.GetProperties` — данные региона ([API](https://fix-cloud.sbis.ru/services-info/page/card/sabyget-site-ru/Locality/GetProperties/1))
- `Establishment.MainPage` — основной контент страницы ([API](https://cloud.sbis.ru/services-info/page/card/ext13_u266/Establishment/MainPage/1))
- `EduSabyGet.MainCarousel` — курсы и мероприятия ([API](https://cloud.sbis.ru/services-info/page/card/Groups/EduSabyGet/MainCarousel/4))

## Алгоритм сортировки данных

### Сортировка каруселей заведений

**Вес категории** вычисляется как сумма по всем заведениям выборки:
```
Вес = Избранное × 1 + Клиент × 10 + Затраты × 100
```
где:
- `Избранное` = 1, если заведение в избранном пользователя
- `Клиент` = 1, если заведение принадлежит аккаунту пользователя
- `Затраты` = затраты пользователя в заведении

**Порядок сортировки каруселей:**
1. Тип (0=общая, 1=ТЦ) — возрастание
2. Вес категории — убывание
3. Порядковый номер — возрастание
4. ID категории — возрастание

**Условия показа карусели:** категория с включённой опцией отображения + минимум 4 заведения.

### Сортировка списка категорий

**Порядок сортировки:**
1. Вес категории (с учётом подкатегорий) — убывание
2. Порядковый номер (родительской категории при наличии) — возрастание
3. Тип (0=общая, 1=ТЦ) — возрастание
4. Раздел (наличие подкатегорий / отсутствие надкатегорий) — убывание
5. Порядковый номер внутри раздела — возрастание
6. ID категории — возрастание

## Архитектура сервисов — Главная страница

![[e20a67b67cc5784a31e5812261b5034c_MD5.avif|683]]

Пользователь → **Главная страница** (4 карусели + категории):

| Блок | Данные идут через |
|------|-------------------|
| Карусель заведений | `sabyget-service` → `referral` (бонусные программы), `user-service` (данные пользователя), `discount-cards` → `theming` (стиль карт) |
| Категории | `sabyget-service` → `central-statistics-interface` + `socnet` |
| Карусель курсов | `socnet` |
| Карусель мероприятий | `sabyget-service` → `socnet` (группы заведений) → `Profile Service` |

**Ключевой сервис**: `sabyget-service` — центральный оркестратор всех запросов страницы.

## Карты — Архитектура данных

![[7c4a64d48efde069bd6906759b766b67_MD5.avif|683]]

Пользователь → Страница «Карты» → **Реестр карт** → `sabyget-service`:
- → `referral` — бонусы по картам
- → `discount-cards` — сами карты → `theming` (настройки стиля)
- → `user-service` — данные пользователя

## Бонусы — Архитектура данных

![[4e1f3cd88820b50f97beda50f7570693_MD5.avif|683]]

Пользователь → Страница «Бонусы» → два параллельных потока:
- **Карты** → `sabyget-service` → `referral` (бонусные программы заведений), `discount-cards` (карты) → `theming`
- **Бонусы** → `referral` → `discount-cards` (бонусы), `sabyget-service` (заведения) → `socnet` (группы заведений)

## Промокоды — Архитектура данных

![[3da9cdb5c512327e25f588211d7080fa_MD5.avif|683]]

Пользователь → Страница «Промокоды» → **Реестр промокодов** → `sabyget-service`:
- → `discount-cards` — промокоды → `theming` (настройки стиля)
- → `user-service` — данные пользователя

## Карта сервисов SabyGet

```
sabyget-service     — центральный оркестратор (розовый)
referral            — бонусные программы и реферальная система (зелёный)
discount-cards      — карты и промокоды (синий)
user-service        — данные пользователя (розовый)
theming             — настройки стиля карт (фиолетовый)
central-statistics-interface — статистика категорий (розовый)
socnet              — курсы, мероприятия, группы заведений (жёлтый)
Profile Service     — профили заведений для мероприятий (фиолетовый)
```
