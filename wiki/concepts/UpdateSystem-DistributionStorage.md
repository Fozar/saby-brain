# Хранилище дистрибутивов

Сервис для приёма, учёта и раздачи дистрибутивов всех типов, используемых «Управлением версиями».

## Задачи

- Приём, учёт и предоставление по ссылкам дистрибутивов приложений всех типов
- Копирование дистрибутивов и образов между стендами, поиск модулей, сравнение сборок, удаление неиспользуемых артефактов
- Публикация утверждённых версий и образов в публичный контур, контролируемая выдача автономным (изолированным) облакам
- Предоставление специализированного API для мобильного магазина приложений (Saby AppMarket)

## Термины

| Термин | Определение |
|--------|-------------|
| Дистрибутив | Именованное файловое представление приложения, подлежащего установке |
| Версия дистрибутива (сборка) | Конкретный экземпляр: набор zip-архивов для разных типов серверов + xml-файлы с метаданными |
| Saby-пакет | Специальный тип дистрибутива, описываемый `.sabypack` JSON-файлами (тип `saby-packet-meta`) |

## Архитектура

```
Хранилище дистрибутивов
├── File Storage (сырые архивы и xml-метаданные)
├── API layer (SBIS BL)
│   └── SabyPacket.Upload() — загрузка saby-пакетов
├── Saby-пакеты генератор
│   ├── Хранилище мета-описаний (saby-packet-meta дистрибутивы)
│   ├── Парсер .sabypack (JSON → внутреннее представление)
│   └── Сборщик коллекций пакетов
├── Подсистема прав — контроль доступа к скачиванию
└── Сервис public distribution storage — публичный контур
```

## Генерация saby-пакетов

API: `SabyPacket.Upload()`

Компоненты:
- **Хранилище мета-описаний**: дистрибутив типа `saby-packet-meta`, содержащий `.sabypack` JSON-файлы описания пакетов
- **Парсер**: извлекает описания пакетов → приводит к внутреннему представлению
- **Источники версий из облака**: список версий приложений, кластеров, пакетов данных (запросы к API управления облаком)
- **Вычисление версий и диапазонов**: для Saby-версий (формат `24.NNNN-NNN`) → диапазон `>=текущая_версия <следующая_мажорная`; для остальных (кластеры) → точная версия
- **Версионирование пакета**: версия = max из версий входящих приложений (без кластеров/пакетов данных); номер сборки = инкремент относительно уже существующих сборок с такой же версией
- **Валидация**: домены должны существовать в облаке; типы зависимостей из допустимого перечня; нет дублей
- **Сохранение**: json → zip → filestorage → новая запись `DistrVersion` типа «saby-пакет»
- **Обработчик событий**: при добавлении файла к версии → анализ типа: `saby-packet` → обновляет таблицу зависимостей; `saby-packet-meta` → валидация всех `.sabypack` в архиве

## Публикация дистрибутива

Публикация = перевод дистрибутива из внутреннего контура в публичный. Инициируется из `cloud-ctrl`.

API: `DistrVersion.Publish(DistrName, Version)` — доступен только внутри контура.

Компоненты:
- **Копирование образов в публичный registry**: перенос Docker-образов через Datacenter; после копирования в таблице `Image` заполняется поле `PublicHost`
- **Установка флага публичности**: `Public = True` в записи версии
- **Heartbeat**: периодические сигналы в `cloud-ctrl` о том, что операция выполняется (нужно при долгом копировании больших образов)
- **Callback**: после завершения (успех/ошибка) → вызов заданного метода в `cloud-ctrl`

Всё в единой транзакции: при ошибке копирования или обновления БД → публикация не фиксируется.

## Загрузка из публичного хранилища (автономными облаками)

API: `DistrVersion.Copy()` (инициатор) / `DistrVersion.GetFullStructure()` (источник, public distribution storage).

Алгоритм:
1. Локальный distribution storage автономного облака → запрос к public distribution storage
2. Проверка аутентификации (OAuth) и прав доступа: версия должна быть публичной (`Public = True`) + клиент в списке разрешённых
3. Public distribution storage формирует структуру:
   - Для файлов модулей → временные защищённые ссылки (`mIsProtected = True`) в filestorage
   - Для образов → публичный адрес из `PublicHost` (не внутренний URL)
4. Локальный ХД скачивает файлы по временным ссылкам → загружает в локальный filestorage; образы копируются из публичного registry в локальный через Datacenter
5. Идентификаторы и URL в структуре заменяются на локальные → создаётся независимая локальная версия дистрибутива

## Копирование дистрибутива между стендами

1. `distribution storage (prod)` запрашивает полную структуру у `distribution storage (fix)`
2. Получает ссылки на файлы и образы
3. Скачивает файлы из `filestorage(fix)` → загружает в `filestorage(prod)`
4. Через `datacenter(prod)` запускает перенос образов из `registry(fix)` в `registry(prod)`
5. Получает новые ссылки → сохраняет структуру с обновлёнными ссылками в `distribution storage (prod)`

## Подсистема прав

Зона доступа: `VM_Repositories`

| Роль | Права |
|------|-------|
| `Cloud User` | Просмотр хранилища дистрибутивов |
| `Distribution Storage Operator` | Создание, изменение, удаление дистрибутивов и сборок |
| `Version Manager Administrator` | Всё что Distribution Storage Operator (общая роль администратора «Управления версиями») |

## Облачные параметры

### Очистка (группа Управление версиями.Хранилище дистрибутивов.Очистка)

| Параметр | Тип | Описание |
|---------|-----|---------|
| `Период очистки старых версий` | число дней | Через сколько дней удаляются неиспользуемые сборки |
| `Период хранения сборки` | число дней | Сколько дней хранятся сборки, не попавшие в N свежих |
| `Число сборок одной версии` | число | Сколько сборок одной версии сохранять |
| `Переопределение` | JSON | Переопределение параметров для конкретных дистрибутивов: `{"distr_name": {"Период очистки...": N, "Число сборок...": N, "Период хранения...": N}}` |
| `Исключения` | массив строк | Дистрибутивы, исключённые из очистки |
| `Срок хранения скриптов` | число дней | Срок хранения архивов скриптов |

### Хранилище файлов (Управление версиями.Хранилище файлов)

| Параметр | Описание |
|---------|---------|
| `Имя хранилища` | namespace в filestorage для файлов дистрибутивов |
| `Мобильные дистрибутивы.Имя хранилища` | namespace для мобильных дистрибутивов |
| `Сайт` | Системное имя сайта для загрузки/отдачи дистрибутивов |
| `Мобильные дистрибутивы.Сайт` | Системное имя сайта для мобильных дистрибутивов |
| `СрокЖизниФайла` | секунды; TTL файла при загрузке в filestorage (для самоочистки при ошибке сохранения) |

### Задачи планировщика

- **Очистка репозиториев** — ежедневное удаление устаревших неиспользуемых дистрибутивов
- **Очистка образов** — ежедневный анализ текущего хранилища образов

## Сервис public distribution storage

Публичная «витрина» для внешних (автономных) облаков; надстройка над основным distribution storage с **общей БД**.

Живёт в публичном домене `registry.saby.ru`, недоступен из внутреннего контура.

Файлы не копируются при публикации — генерируются **персональные временные ссылки** в момент запроса.

**Ключевые API:**

| Метод | Описание |
|-------|---------|
| `SabyPacketList` | Получение списка saby-пакетов |
| `SabyPacketComposition` | Полный состав пакета (дистрибутивы и их версии) |
| `DistrVersionGetFullStructure` | Структура дистрибутива со всеми ссылками на файлы и образы |

## Схема БД (ключевые сущности)

| Таблица | Описание |
|---------|---------|
| `Продукт` | Дистрибутив — объединяет версии одного приложения |
| `ВерсияПродукта` | Одна сборка дистрибутива |
| `УстановленнаяВерсия` | Установленные версии, защищённые от удаления |
| `Сервис` | ssrv-файл под конкретную архитектуру/ОС |
| `ПропатченныеСервисы` (`PatchedServices`) | Сервисы, на которые применён патч в рамках операции |
| `ФайлВерсии` | Общие файлы дистрибутива, не относящиеся к сервисам |
| `Скрипт` | Архив скрипта (привязан к версии дистрибутива) |
| `Патч` | Архив патча (привязан к версии дистрибутива) |
| `ФайлХранилища` | Физический файл: ссылка в filestorage + метаданные (размер, mime, md5, дата) |
| `Модуль` | Модуль дистрибутива (BL или интерфейс), соответствует `.smod` |
| `МодульВерсии` (`ModuleVersion`) | Версия модуля — отдельная сущность, зависит от платформы и алгоритма сборки |
| `СвязьСервисаСМодулями` (`ServiceToModule`) | M:M — сервис ↔ версия модуля |
| `ЧастьСервиса` (`ServicePart`) | Устаревшая реализация `Модуль` |
| `Образ` (`Image`) | Docker-образ / helm-chart; поле `PublicHost` — адрес в публичном registry |
| `ФайлОбновления` | Логи и файлы, создаваемые во время обновлений |
| `СервисХоттабыча` | Сервисы «Управления версиями», хранящие файлы обновлений (разграничение доступа по стендам) |
| `DistrClient` | Связь ID клиентов автономных облаков с дистрибутивами saby-пакетов |
| `SabyPacketDependencies` | Состав saby-пакета (из каких дистрибутивов состоит) |

**Ключевые индексы:**
```
ФайлХранилища(md)                          -- дедупликация по MD5
Module(Name, Type, Revision)               -- уникальность модуля перед загрузкой
ModuleVersion(VersionHash, Name, Type) UNIQUE
ServicePart(Revision), ServicePart(md)     -- дедупликация частей сервиса
ServiceToModule(Module), ServiceToModule(Service)
ВерсияПродукта(Продукт, Дата DESC, Название DESC, Сборка DESC NULLS LAST)
Image(Url)                                 -- поиск образа при загрузке дистрибутива
ФайлОбновления(СервисХоттабыча, ИдОбновления)
```

**Алгоритм дедупликации файлов:** перед загрузкой клиент вычисляет md5 → запрашивает хранилище по индексу `ФайлХранилища(md)`. Если файл есть — создаётся только ссылка на существующий `ФайлХранилища`, без повторной загрузки.

**Удаление файла:** при удалении любой ссылающейся сущности — подсчёт ссылок на `ФайлХранилища`. Если ссылок нет → запись `ФайлХранилища` удаляется → запрос в сервис «ХранилищеФайлов».

## JSON-RPC API (сервис `distr-storage`)

Взаимодействие через стандартный протокол СБИС JSON-RPC. Endpoint: `sbisEndPoint('distr-storage')`.

### `Продукт.Список/`

Получение списка дистрибутивов.

| Параметр фильтра | Тип | Описание |
|-----------------|-----|---------|
| `СтрокаПоиска` | Строка | Часть названия дистрибутива |

Возвращает: `@Продукт` (ID), `Название`.

```python
Фильтр = sbis.Record({'СтрокаПоиска': 'standard'})
ep = sbis.EndPoint('distr-storage')
result = sbis.BLObject('Продукт', ep).Список(None, Фильтр, None, None)
```

### `ВерсияПродукта.Список/`

Список версий (сборок) дистрибутива.

| Параметр фильтра | Тип | Описание |
|-----------------|-----|---------|
| `Продукт` | Число | ID дистрибутива |
| `НазваниеПродукта` | Строка | Название дистрибутива |
| `Версия` | Строка | Часть версии со сборкой и датой |
| `ФильтрПоВерсии` | Строка | Версия без сборки и даты |
| `Тип` | Число | 0 — обычный, 1 — для update.sbis.ru, 2 — пакет данных |

Возвращает: `@ВерсияПродукта`, `Название`, `Сборка`, `Дата`, `Продукт`, `Установлена`, `ПолноеНазвание`, `ЕстьМодули`, `ЕстьПатч`, `Тип`, `PatchedServices: StringArray`.

```python
Фильтр = sbis.Record({'НазваниеПродукта': 'admin.sbis.ru'})
ep = sbis.EndPoint('distr-storage')
result = sbis.BLObject('ВерсияПродукта', ep).Список(None, Фильтр, None, None)
```

### `ВерсияПродукта.ДеревоФайлов/`

Дерево файлов версии дистрибутива. Поддерживает полную и постраничную загрузку.

| Параметр фильтра | Тип | Описание |
|-----------------|-----|---------|
| `ВерсияПродукта` | Число | ID версии |
| `Раздел` | Строка | Фильтр по разделу (ИдРодителя) |
| `Название` | Строка | Фильтр по имени файла |
| `Рекурсивно` | Логическое | Загрузить всё дерево целиком |

Возвращает: `Ид`, `ИдОбъекта`, `Тип` (модуль/файл версии/скрипт/патч), `Название`, `Раздел`, `Раздел@` (является папкой), `ФайлХранилища`, `Ссылка` (URL для скачивания), `Размер`, `Дата`, `Mime`, `md` (md5), `Применен` (для патчей), `Описание` (для патчей/скриптов), `ОС` (для модулей), `Архитектура` (для модулей).

```python
# Корень версии ID=123
Фильтр = sbis.Record({'ВерсияПродукта': 123})
ep = sbis.EndPoint('distr-storage')
result = sbis.BLObject('ВерсияПродукта', ep).ДеревоФайлов(None, Фильтр, None, None)

# Полное дерево
Фильтр = sbis.Record({'ВерсияПродукта': 123, 'Рекурсивно': True})
result = sbis.BLObject('ВерсияПродукта', ep).ДеревоФайлов(None, Фильтр, None, None)

# Конкретный раздел (с пагинацией)
Фильтр = sbis.Record({'ВерсияПродукта': 123, 'Раздел': 'module_'})
ep = sbis.EndPoint('distr-storage', auth_data=sbis.AuthByLogin('viewer'))
lst = sbis.BLObject('ВерсияПродукта', ep).ДеревоФайлов(None, Фильтр, None, sbis.Navigation(0, 100, True))
```

## sabycloud-transfer-script

Python-скрипт для выгрузки дистрибутивов из облака Тензора и загрузки в локальное хранилище автономного облака.

**Режимы работы:**
1. Скачивание: скрипт на сервере в сети Тензора → скачивает дистрибутивы по переданному списку → создаёт единый файл
2. Загрузка: клиент монтирует файл → запускает скрипт в режиме загрузки в локальное хранилище

**Взаимодействует с:** Хранилище дистрибутивов, Хранилище файлов, Harbor (контейнеры Тензора), Gitea (контейнеры клиента).

GIT: `https://git.sbis.ru/orchestration/admin`

## Организация кода

- **Состоит из модулей:** distribution-storage, public-distribution-storage
- **Зависимости:** `DiskClient`, `Workflow Coordinator Client`, `PermissionChecker`
- **Сторонние библиотеки:** не используются
