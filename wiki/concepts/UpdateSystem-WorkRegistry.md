# Система обновлений: Реестр работ и Планы выпуска

Подсистема Хоттабыча для планирования и отслеживания всех работ по обновлению продуктивной среды СБИС/Saby.

## Назначение

Служба эксплуатации ежедневно запускает десятки (а во время выпуска релизов — сотни) обновлений, скриптов, патчей и пакетов данных. Реестр работ и Планы выпуска решают задачу координации этих запусков, снижая риск человеческого фактора.

**Реестр работ** — основное и единственное место хранения работ, выполняемых Хоттабычем и сотрудниками ЦОД. Работы структурированы по типу, версии и дате выполнения.

**Планы выпуска** — перечень работ на определённый день, упорядоченных по внутренним регламентам. Содержат не сами работы, а **пункты плана** — ярлыки/ссылки на работы из Реестра.

> [!key-insight] Разделение Работа / Пункт плана
> Работа хранится в Реестре ОДИН РАЗ. Пункт плана — это ссылка. При изменении работы в реестре она автоматически обновляется во всех планах, где добавлена.

## Роли пользователей

| Роль | Задачи |
|------|--------|
| Релиз-менеджеры | Настройка, формирование планов, запуск работ на бою |
| Дежурные ЦОД | Запуск работ, контроль выполнения, ручные статусы, отчёты |
| Тестировщики | Проставление билдов и статусов по своим сервисам (отмашка о готовности) |
| Сборщики тест-стендов | Запуск и выполнение работ на тестовых стендах |

## Структура

```
Реестр работ
├── [Тип работы] (папка)
│   ├── [Версия] (папка)
│   │   └── Работа (карточка)
│   └── ...
└── ...

Планы выпуска
├── Экспортируемые (папка)
│   └── [Экспортный план]
├── [YYYY-MM-DD] (Этап-день)
│   ├── [Тип работы] (Этап)
│   │   └── Пункт плана (ссылка → Работа)
│   └── ...
└── ...
```

## Волны обновления

Три типа волн мажорного обновления:

- **Нулевая волна** — облако и middleware-сервисы
- **Первая волна** — online-master, сигнальные юниты онлайна (пятница/суббота)
- **Вторая волна** — online-master + все online (все прочие сервисы, кроме спецификаций — пятница)

**Предыдущая версия основных сервисов** — версия на бою до обновления на первую волну.

## Регионы

Каждая работа имеет флаги региона:
- **RU** — Россия
- **KZ** — Казахстан
- **UZ** — Узбекистан
- **TM** — Туркменистан

Принадлежность к облаку определяется по: документу-основанию → наличию приложения в облаке → ручной установке.
