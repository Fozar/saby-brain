# Wasaby Internationalization (i18n / l10n)

Три уровня:
- **i18n** — оборачивание строк в `rk()` в исходном коде
- **l10n** — перевод в JSON-словарях
- **r18n** — региональная адаптация (форматы дат, валют) через аспекты регионализации

## Функция rk()

Единая для Python и C++. Параметры: строка → контекст (необяз.) → число (для склонений).

### Python

```python
from sbis import rk
# или: import sbis; sbis.rk(...)

# Базовый перевод
LogMsg(rk('Функция checkout устарела', 'Мой функционал'))

# С плейсхолдерами (НЕ склеивать строки!)
raise Error(rk('Ошибка: "{0}" с кодом {1}').format(message, code))
raise Error(rk('Ошибка: "{message}" с кодом {code}').format(message=msg, code=code))

# Со склонением
rk('plural#запись(-и,-ей)', count)
```

### C++

```cpp
LogMsg(rk("Функция checkout устарела"_sv, "Мой функционал"_sv));

// Плейсхолдеры (те же правила)
// В аргументах rk() — только строковые константы, не переменные
rk("ООО \"Тензор\""_sv)  // кавычки экранировать
```

## Правила локализации

| Правило | Неправильно | Правильно |
|---------|------------|-----------|
| Не склеивать строки | `rk('Ошибка') + ': ' + rk('…')` | `rk('Ошибка: {0}').format(x)` |
| Не создавать статические константы с rk() | `class A: msg = rk('…')` | `def m(self): return rk('…')` |
| Знаки препинания — внутри rk | `rk('Ошибка') + '!'` | `rk('Ошибка!')` |
| Нет пробелов/спецсимволов по краям | `rk('  Текст  ')` | `rk('Текст')` |
| Нет `\n` внутри ключа | `rk('\nТекст')` | `"\n" + rk('Текст')` |

## Контекст перевода

Одна и та же строка может переводиться по-разному в зависимости от контекста. Контекст задаётся вторым аргументом `rk()`:

### Python / C++

```python
rk('Отзыв', 'Клиентский')   # → "Review" (отзыв клиента)
rk('Отзыв', 'Отпуск')       # → "Recall" (отозвать из отпуска)
```

```cpp
rk("Отзыв"_sv, "Клиентский"_sv);
```

### Шаблоны (TypeScript/UI)

```
{[Клиентский@@Отзыв]}
```

Валидатор **СТАН** проверяет согласованность контекстов — одна строка с разными контекстами должна давать разные переводы.

## Роли и системные секции

### Локализация ролей

Название роли переводится через словарь с префиксом `ИмяРоли@@`:

```json
{
  "ИмяРоли@@Бухгалтер": "Accountant",
  "ИмяРоли@@Директор": "Director"
}
```

Комментарии к роли (не само название) — без контекста.

### Локализация системных секций

Названия участков системы (sections) с префиксом `ИмяУчасткаСистемы@@`:

```json
{
  "ИмяУчасткаСистемы@@Розница/ККТ": "Retail/CKT",
  "ИмяУчасткаСистемы@@Бухгалтерия": "Accounting"
}
```

### Ограничения и уровни доступа

Строки ограничений/уровней доступа — контекст не нужен (переводятся без `@@`).

## Словари переводов

**Формат файла**: JSON, ключ → перевод.

```json
{
   "Что нового?": "What's new?",
   "Клиентский@@Отзыв": "Review",
   "Отпуск@@Отзыв": "Recall",
   "plural#рубль(-я,-ей)": "рубль|рубля|рублей|рубля"
}
```

Синтаксис:
- `@@` — разделитель контекста и ключа: `"<Контекст>@@<Ключ>": "<Перевод>"`
- `plural#` — склонение по числу; формат: `ключ|форма2|форма5`
- Для plural-ключей обязательны словари на **все** языки включая русский

**Именование файлов**:
- `[lang].json` — общий (напр. `en.json`, `ru.json`)
- `[lang]-[COUNTRY].json` — региональный (`en-GB.json`, `en-US.json`)

**Расположение словарей**:
- Интерфейсный модуль: `[модуль]/lang/[язык]/.json`
- Сервисный модуль (БЛ): `[модуль]/lang/` → один общий словарь (Python + C++ + пакеты данных)

**Требования к словарям**:
- Всегда должен существовать общий `[lang].json`; regional только если перевод отличается
- Все ключи региональных словарей должны быть в общем
- Нет пустых строк, нет HTML-тегов; отступ — 3 пробела; после `:` — пробел

## Локаль, язык, страна

Формат: `язык-СТРАНА` (ISO 639-1 + ISO 3166-1)

Примеры: `ru-RU`, `ru-KZ`, `en-US`, `en-GB`

Язык хранится в cookie `lang` и в профиле пользователя. При смене языка cookie обновляется и страница перезагружается; распространение на все приложения — задержка до 2 минут.

Получение языка на БЛ:
- Авторизованный пользователь → язык из профиля (через cookie `lang`)
- Без аутентификации → язык системы

## Подключение i18n к сервису

Сервисный модуль `i18n` из SBIS SDK. По умолчанию уже подключён через `basic_platform.s3srv`.
