# Franchise Loyalty System (Франшиза)

Source: `raw/Описание.md` | Domain: [[domains/price-formation/_index]]

## Назначение

Функционал франшизы в системах лояльности предоставляет комплексное решение для повышения конверсии и возвращаемости клиентов **не для одного заведения, а для всей франшизной сети**.

**Три ключевые задачи:**
1. Распространение единых условий программ лояльности и правил получения выгод по всей сети.
2. Хранение и использование единого бонусного баланса клиентов в рамках всей франшизной сети.
3. Проксирование клиентов для создания единой базы клиентов.

---

## Терминология

| Термин | Определение |
|--------|-------------|
| **Франшиза** | Группа SabyNet с определёнными свойствами, задаваемыми на уровне сервиса соцсети |
| **Владелец** | Аккаунт-создатель франшизной сети; задаёт правила лояльности |
| **Участник** | Аккаунт, вошедший в сеть франшизы по приглашению Владельца |
| **Точка продаж** | Конкретное торговое заведение Участника, добавленное во франшизу |

> Термин «Группа соцсети» используется в описаниях общей механики SabyNet; «Франшиза» — в контексте специфической бизнес-логики.

---

## Концепт решения

1. Владелец создаёт **«Франшизную сеть»** через функционал SabyNet.
2. Приглашает Участников, указывая набор их точек продаж.
3. Отношения организуются через SabyNet: приглашения/договоры, личные кабинеты, ролевая конфигурация.

### SabyNet — базовые возможности для франшизы

- Личный кабинет для Участников и Владельца с возможностью конфигурации по роли.
- Механизм приглашения с возможностью заключения договора.
- API определения роли пользователя/аккаунта/лица в контексте бизнес-группы.

---

## Настройка единых правил лояльности

При создании новой франшизной сети в реестрах акций и промокодов Владельца автоматически создаются **папки, одноимённые группе соцсети** — в них помещаются все франшизные сущности соответствующего типа (акции, бонусные программы, промокоды).

### Гранулярность применения акции

Владелец может назначить акцию на:
- Всю франшизу
- Конкретных Участников сети
- Конкретные точки продаж Участников

### Синхронизация данных

1. Владелец нажимает **кнопку синхронизации** (дублируется в каждом реестре настроек лояльности).
2. Вызывается метод БЛ, подготавливающий весь набор настроенных данных.
3. Для всех Участников, находящихся во франшизе в данный момент, вызывается метод получения подготовленных данных и записи в локальную БД.

> **Важно:** Признак «отправленности» отдельных сущностей **не хранится**. При каждой синхронизации ранее отправленные сущности лояльности **полностью перетираются** — поиск соответствия ведётся по UUID акции/карты. Это обеспечивает целостность данных, но не позволяет узнать, какая акция у какого Участника нуждается в обновлении.

Участник может **просматривать** настройки лояльности в своих реестрах, но **не редактировать** их.

## Бонусные балансы и карты

Для единых бонусов используется **СДК (Сервис Дисконтных Карт)** как хранилище карт (включая персональные) и бонусных балансов.

- Любое движение бонусов (начисление, списание, сгорание) фиксируется на СДК.
- Любой аккаунт-участник франшизной сети может получить **актуальный баланс** каждого клиента.

См. детальную архитектуру → [[DiscountCard-Subsystem-Overview]]

## Единая база клиентов

Все способы присоединения к программам лояльности ведут на **аккаунт Владельца**, даже если переход происходит из-под аккаунта Участника.

**Реализовано через:**

| Точка входа | Поведение |
|-------------|-----------|
| **Анкета выдачи дисконтных карт** | Создаёт клиента и карту сразу в аккаунте Владельца; при необходимости отправляет созданную карту Участнику |
| **Кнопка «Присоединиться» в SabyGet** | Ведёт на присоединение к Владельцу; клиент и карта создаются в аккаунте Владельца |

## Декомпозиция по сервисам

### SabyNet

- Механизмы взаимоотношений Владелец ↔ Участники (приглашения, договоры, расторжение).
- Реализация личных кабинетов Владельца и Участника.
- API определения роли пользователя/аккаунта/лица в контексте бизнес-группы.

### Лояльность в облаке (`PriceFormation.Online`)

- Настройка и отображение франшизных сущностей систем лояльности.
- Расчёт и применение скидок и бонусов на продажах.
- Синхронизация данных между аккаунтами.

### СДК (`discount-cards`)

- Хранение всех франшизных (и не только) карт.
- Хранение и фиксация операций с бонусами.
- Вычисление франшизных бонусных балансов.

---

## Ограничения

> **Текущее ограничение:** Системы лояльности поддерживают только **одну франшизную сеть** для Владельца и для Участников. Нельзя быть Владельцем двух франшиз.
