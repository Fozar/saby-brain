# Патчи в Wasaby

Патч — инструмент экстренного исправления файлов в дистрибутиве без сборки нового дистрибутива.

> [!key-insight] Когда использовать патч
> Только для срочных аварийных правок. Патч применяется **на все сервисы** внутри приложения (нельзя выбрать конкретный сервис). Изменения должны быть сохранены в репозитории через отдельный MR.

## Что можно патчить

| Можно | Не рекомендуется |
|-------|-----------------|
| `*.orx`, `*.py` (БЛ без компиляции) | Бинарные файлы (`*.so`, `*.dll`, `*.dylib`) |
| `*.js`, `*.ts`, `*.tsx`, `*.tmpl`, `*.wml`, `*.css`, `*.less` | Файлы описания БД (`.s3db`, `.s3cfg`, `.dpack`) |
| Картинки, текстовые файлы | Скомпилированные библиотеки |
| Файлы описания модулей | |

## Патч на модуль интерфейса (4 шага)

### Шаг 1: Получение ревизии

Из файла `meta.info` в дистрибутиве (Jenkins → артефакты или Хоттабыч → Дистрибутивы):
- Ключ `rev` — номер ревизии (хэш коммита) для данной сборки

### Шаг 2: Переключение на ревизию

```sh
git checkout <rev>
# Создать ветку: <версия>/bugfix/<автор>/<номер задачи>
```

### Шаг 3: Merge Request на GitLab

- MR создаётся на **git.sbis.ru** (не online.sbis.ru!)
- Из ветки в rc-ветку, на основе которой собран дистрибутив
- Конфликты: выбирать в пользу новых изменений

### Шаг 4: ВНР

Задача "Выполнить на рабочем" (ВНР) на online.sbis.ru:
- Тип работы: "Патч на интерфейс"
- Прикрепить ссылку на MR
- **После запуска ВНР нельзя изменять MR!**

### Шаг 5: Доброска в репозиторий

После выполнения патча — MR в следующую версию (изменения должны войти в текущую и все последующие).

## Патч на модуль бизнес-логики (3 шага)

### Шаг 1: Подготовка файлов

Способ 1: Загрузка с сетевого диска `\\sbis-dev.corp.tensor.ru\PROGRAMMERS\СБиС_3\Тестирование\`

Способ 2 (рекомендуется): Git-репозиторий
```
├── Модули бизнес-логики/
│   └── <ModuleName>/
│       └── <изменённые файлы>.py
```

### Шаг 2: ВНР

- Тип работы: "Патч на бизнес-логику"
- Архив именуется: `p_<фиопользователя>_<номервнр>.zip`
- Кодировка имён файлов: **CP866**
- Архиватор: 7-Zip (стандартные средства Windows зависают)

### Шаг 3: Доброска в репозиторий

Аналогично интерфейсному патчу.

## Версионирование

Формат версии с патчами: `<версия>-<сборка>.<число патчей>`

Пример: `24.5220-60.3` — 3 патча на сборку 60.
