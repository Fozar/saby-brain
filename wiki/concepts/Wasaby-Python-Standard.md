# Wasaby Python Dev Standard

Стандарт разработки на Python в Wasaby Framework, основан на PEP-8 с исключениями.

## Отличия от PEP-8

| Правило | В Wasaby |
|---------|----------|
| Длина строки | Максимум **120 символов** (PEP-8: 79) |
| CamelCase в методах | **Допустим** для методов объектов бизнес-логики (BLObject) |

Пример допустимого CamelCase для BL:
```python
sbis.BLObject("Account").Invoke("ListOfCertificatesAndApplications", myfilter)
```

## Правила оформления

- **Отступы**: 4 пробела на каждом уровне
- **Пустые строки**: 2 строки вокруг блоков верхнего уровня (функции/классы), 1 строка между методами класса
- **Импорты**: в начале файла; каждый пакет на отдельной строке; абсолютные пути предпочтительны; wildcard (`from module import *`) **запрещён**
- **Кавычки**: строки — любые, многострочные — всегда `"""`

## Правила именования

| Объект | Формат | Пример |
|--------|--------|--------|
| Пакеты и модули | строчные, подчёркивание допустимо | `my_module` |
| Классы | CamelCase | `MyClassName` |
| Исключения | CamelCase + суффикс `Error` | `ValidationError` |
| Функции/методы | snake_case | `my_method_name` |
| Аргументы метода | snake_case; первый аргумент = `self` / `cls` | |
| Непубличные методы | `_snake_case` (один `_`) | |
| Приватные методы | `__snake_case` (два `__`) | |
| Константы | UPPER_SNAKE_CASE | `MAX_OVERFLOW` |

## Комментарии

- Назначение файла — в начале каждого файла
- Все классы и методы — с комментариями
- Однострочный: начинается с `#`, **2 пробела** перед `#`, **1 пробел** после
- Для временных «плохих» решений — ссылка на задачу/ошибку
- API документируется через docstrings (PEP-257)

## Code Review — числовая оценка

**Итоговая оценка = 5 − штраф Radon − штраф Pylint − штраф за грубые ошибки**

### Штраф Radon (цикломатическая сложность)
```bash
radon cc -nc -as <path>   # блоки уровня B+
radon cc -nd -as <path>   # блоки D+
```
- D+ < 20% от B+ → −1 балл
- D+ < 30% от B+ → −2 балла
- D+ ≥ 30% от B+ → −3 балла

### Штраф Pylint
- Оценка ≥ 8 → 0 баллов
- 5 ≤ оценка < 8 → −0.5 балла
- оценка < 5 → −1 балл

### Грубые ошибки (−1 балл каждая)
- Более 5 строк кода в Python-методе в `.orx`-ресурсе
- SQL-запросы чтения из БД **в цикле**
- Подфункции (функции внутри функций), кроме lambda-выражений — −0.5 балла

## Частые антипаттерны

| Антипаттерн | Описание | Решение |
|-------------|----------|---------|
| **Божественный объект** | Файл или класс > 1000 строк | Разбить модуль на несколько / декомпозировать класс |
| **Глубокая вложенность** | Вложенность > 5 уровней | Уменьшить вложенность (early return, вынос в методы) |
| **Целиком статичный класс** | Все методы `@classmethod`/`@staticmethod` | Заменить модулем с функциями и публичным API |
| **Абстрактный класс с пустыми публичными методами** | Обход статичного класса | То же решение — модуль с функциями |
| **Класс-хелпер** | Публичные методы не используют состояние экземпляра | Заменить модулем с функциями |

> [!key-insight] Подфункции ≠ декомпозиция
> Подфункция имеет ту же область видимости, что и внешняя функция — это не снижает реальную сложность, а лишь маскирует её для Radon. Используйте настоящие методы класса или отдельные функции.
