# Сервис параметров (parameters)

Высокопроизводительный сервис хранения именованных значений, привязанных к разным типам области видимости.

## Понятие параметра

**Параметр** = имя (key) + значение (string) + тип области видимости (scope).

Если имя вида `"префикс.суффикс"` — `"префикс"` является **пространством имён** (вложенность не поддерживается).

## Типы области видимости (ScopeType)

| Код | Тип | Примеры |
|---|---|---|
| 0 | **GLOBAL** | Системные настройки "для всех" |
| 1 | **USER** | Истории ввода, фильтры реестров |
| 2 | **PROFILE (PERSON)** | Настройки персоны |
| 3 | **ACCOUNT (CLIENT)** | Настройки бухучёта аккаунта |
| 4 | **DEVICE** | Настройки уведомлений, логирование |
| 5 | **USER_DEVICE** | Настройки пользователя на устройстве |

## Архитектура

1. **Шардированная БД** — высокая производительность, горизонтальное масштабирование
2. **Отслеживание времени обращения** — параметры удаляются при отсутствии обращений
3. **Клиентский модуль** — API для работы из других сервисов

## Время жизни параметров

- По умолчанию: **30 дней** без обращений (`Parameter Service.Config.Время жизни параметра`)
- Боевой сервис: **180 дней**
- Параметр **Permanent** — бессрочное хранение (описание через `.psk` в Genie)

## Права доступа

- Из браузера можно получить только параметры своей учётной записи
- GLOBAL и ACCOUNT нельзя записывать с клиента (настраивается флагом `ЗапретитьЗаписьГлобальныхПараметров`)

## parameters-constants

Модификация сервиса parameters для параметров с периодом действия:
- Хранит историю (нет автоматической очистки)
- Дополнительное API: отбор параметров, действительных в заданный момент времени

## Декларативное описание параметров

Нужно если требуется **бессрочное хранение** или **синхронизация в offline-приложениях**.

Используется словарь ключей `.psk` в Genie (создаётся через контекстное меню модуля).

Поля PSK-записи:
- **Name** — имя параметра
- **Scope type** — Client / User / Person
- **Flags** — Permanent (бессрочно), Synchronizable (синхронизация)
- **Sync direction** — только из облака / из облака и обратно

Синхронизация в мобильное приложение: репозиторий `middleware`, проект `ParameterServiceMobile`.
