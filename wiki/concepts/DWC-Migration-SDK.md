# DWC Migration — Перевод СДК с событий на DWC

Navigation: [[price-formation/_index]] | [[DiscountCard-Subsystem-Overview]] | [[DWC-Distributed-Workflow-Coordinator]]

Проект переводит взаимодействие между Онлайном и Сервисом Дисконтных Карт (СДК) с событийной модели на DWC-задачи. Цель — устранить проблемы с очередями, порядком обработки и управлением нагрузкой.

> [!key-insight] Суть изменения
> Вместо событий (`online.loyalty-card.*`) Онлайн ставит **DWC-задачи** напрямую на вызов методов-обработчиков СДК. Каждый сценарий клиента занимает не более 1 воркера СДК. Приоритет контролируется через ограничение ресурсов DWC.

---

## Проблемы текущей схемы (события)

1. **Очереди**: массовые изменения (деактивация БП на аккаунте с тысячами карт) → тысячи событий → очереди на СДК. [Прецеденты](https://online.sbis.ru/opendoc.html?guid=db2ec46a-4b2f-46b4-b8a1-be6b8118d42b&client=3)
2. **Неправильный порядок**: событие создания карты может обработаться после события начисления приветственных бонусов → потеря данных. [Пример](https://online.sbis.ru/opendoc.html?guid=9ad181e5-31f8-43a0-adc1-db1de2bce06b&client=3)
3. **Нет управления нагрузкой**: невозможно приоритизировать критичные сценарии перед некритичными.

---

## Цели и критерии достижения

| Цель | Критерий |
|------|----------|
| Нет очередей при массовых изменениях | Нет очередей при откреплении БП от типа карт (>300 000 карт) |
| Нет потери данных при неправильном порядке | Приветственные бонусы начисляются при задержке 30с в обработчике создания карты |
| Управление нагрузкой на СДК | Новые карты создаются при нагрузке БД >85%; удаление карт блокируется, пока нагрузка не упадёт <75% |

**Потенциальный круг пользователей**: ~73 000 аккаунтов (все, кто работает с картами или промокодами).

---

## Концептуальное решение

- 1 событие → 1 DWC-сценарий с 1 задачей
- Массовые изменения → 1 DWC-сценарий с N задачами (по частям)
- Одинаковые сценарии разных клиентов выполняются параллельно; каждый занимает ≤ 1 воркер СДК

**Приоритеты через resource limits DWC:**

| Приоритет | Условие запуска |
|-----------|----------------|
| Высокий | Всегда |
| Средний | Нагрузка БД СДК < 85% |
| Низкий | Нагрузка БД СДК < 75% |

## Сценарии приёмки (демонстрации)

**Карты**: нет регресса + приветственные бонусы при задержке 30с + новые карты создаются при загрузке >85%, удаление блокируется до <75%.

**Типы карт**: нет регресса + нет очередей при откреплении БП от типа карт (300 000+ карт).

**Выпуски промокодов**: нет регресса.

**Промокоды**: нет регресса + если СДК недоступен при изменении лимита применения франшизного промокода, лимит изменится при восстановлении сервиса.

**Настройки бонусов**: нет регресса.

**Тех. долг**: в истории реестров Акций/Промокодов видны Группы; диагностика выгружает расшифровку баланса.
