# Wasaby BL List Methods

Списочные методы возвращают набор записей по заданным параметрам. Два подтипа: декларативный (конфигурируется в Genie без кода) и реализуемый вручную (Python/C++/SQL).

## Общие входные параметры

Оба типа принимают одинаковую сигнатуру:

| Параметр | Тип Python | Описание |
|---------|-----------|---------|
| ДопПоля | — | Набор дополнительных полей к результату (расширяет Result source fields) |
| Фильтр | — | Параметры фильтрации выборки |
| Сортировка | `SortingList` | Массив `ColumnSortParameters`; перекрывает сортировку из опций |
| Навигация | `Navigation` | Постраничная или по курсору |

### Сортировка (SortingList)
```python
from sbis import SortingList, ColumnSortParameters, SortOrder

sorting = SortingList()
sorting.AddItem(ColumnSortParameters(
    fieldName="Дата",
    order=SortOrder.soDESCENDING,  # DESC
    nullPolicy=True  # NULL в конце
))
```

### Навигация (Navigation)
```python
from sbis import Navigation

# Постраничная
nav = Navigation(Limit=50, Page=0, HasMore=False)

# По курсору
nav = Navigation(Position=cursor_value, Direction="forward", Limit=50)
```

`HasMore=True` → в ответе флаг "есть следующая страница"; `HasMore=False` → общее количество записей.

## Declarative List (Декларативный)

Создаёт SQL-запрос к БД без написания кода — только конфигурация в Genie. Требует модуль "Бизнес логика" из SBIS SDK.

### Ключевые опции в Genie

| Опция | Описание |
|-------|---------|
| Result source fields | Поля по умолчанию (если не все нужны) |
| Return all columns | Возвращать все колонки таблицы |
| Filter type | Тип фильтра: Free / Linked records / Hierarchical |

### ДопПоля

Расширяет `Result source fields` без дублирования методов:
```
Result source fields = ["@Товар", "Наименование", "Описание"]
ДопПоля = ["Цена", "Количество"]
→ Результат: 5 полей
```

Передача ДопПоля не нужна, если `Return all columns = true`.

## Manually Implemented List (Реализуемый вручную)

Используется, когда декларативного не хватает. Реализация: Python, C++ (Native function) или SQL.

| Реализация | Модуль |
|-----------|--------|
| C++ | BL Core |
| Python | BL Python |
| SQL | Бизнес-логика |

## Filter Types (Типы фильтра декларативного метода)

### Free (Произвольный)
Наиболее общий. Любые параметры фильтрации. Каждый параметр настраивается через "Диалог редактирования параметров фильтрации".

### Linked records (Связанные записи)
Для выборки по связи Многие-к-одному. Автоматически добавляет 2 параметра:
1. `ИмяПоляСвязи` — поле связи в основной таблице (Value category: Constant)
2. `Идентификатор` — PK родительской записи

### Hierarchical structures (Иерархические структуры)
Для таблиц с иерархией. Автоматически добавляет 5 параметров:

| Параметр | Описание | По умолчанию |
|---------|---------|------------|
| Название (узел) | Идентификатор узла иерархии; если не передан → корень | корень |
| ИмяИерархии | Поле связи иерархии (напр. "Раздел") | Constant |
| РазворотСкрытыхУзлов | Доступность вложенных в скрытые узлы | Нет |
| Разворот | Вид разворота | Без разворота |
| ВидДерево | Что разворачивать (узлы+листья/только узлы/только листья) | С узлами и листьями |

## Расширенные паттерны

Подробно в [[Wasaby-BL-List-Advanced]]:
- Стандартные параметры фильтрации: `ИдО` / `СписокИдО` / `Раздел`
- Навигация по курсору + порционная загрузка (`iterative` + `nextPosition`)
- Множественная навигация (мультирутовый обход)
- `ListIterator` — массовая отметка записей
- `ShowMarked` — показать отмеченные с путями
- `MoveToFolder` — перенос отфильтрованных записей в папку
- `Sum.ByMethod` — суммирование через DWC
- `TranslitListCall` — поиск с учётом раскладки
- `CoreUtils.ListWithParents` — хлебные крошки в иерархическом поиске

## Безопасность и лимиты

> [!key-insight] Уязвимость без лимита
> Вызов без навигации с произвольным количеством записей — DoS-уязвимость (отдел безопасности).

**Best practices:**
1. Без навигации метод должен: выдавать ошибку ИЛИ 0 записей ИЛИ внутренний лимит (≤ 100)
2. Используйте навигацию по курсору (предпочтительнее постраничной)
3. С клиента всегда передавать навигацию с лимитом ≤ 100

Настройка ограничения: `Ядро.Сервер приложений.МаксимальныйРазмерВыборкиСписочныхМетодов` в "Управление облаком". Ограничение только для вызовов с клиента, межсервисные вызовы не затрагивает.
